# Strata — Production-Readiness QA Report

**Cycle:** `ralph/qa-production-readiness` · **Started:** 2026-05-30 · **Verdict:** **PENDING** (locked until US-013)

This is the living evidence document for the QA production-readiness cycle. The
release go/no-go decision is made against the contents of this file, not vibes.
It is updated per-story; the final verdict is set in US-013.

Scope backends: **TiKV + memory** (canonical CI default), **RADOS/ceph**,
**S3-compatible (MinIO)**. Cassandra stays first-class in code but is gated off
the per-PR CI gate (see `.github/workflows/ci.yml::integration`).

**Source of truth = CI on Linux** (FR-4). Many failure modes (cgo runtime,
darwin-only `gosigar` sysctl, race timing) do not reproduce on a local
macOS/lima box. Numbers below tagged `CI (local env-blocked)` are measured on
the Linux CI coverage job, not the author's laptop.

### Baseline reading notes

- **`internal/meta` 16.5% / `internal/meta/storetest` 0.0%** — these are
  interface + shared-contract-harness packages; coverage lands when the
  contract suite runs against a backend (memory/tikv), not when `go test` runs
  the package in isolation. Not a real gap.
- **`internal/meta/cassandra` 4.6%** — Cassandra integration tests are gated
  off the default tag (testcontainers + `integration` build tag). The 4.6% is
  the non-integration surface only; the backend is exercised under
  `-tags integration` locally, not on the per-PR gate.
- **`internal/data/memory` 6.5% / `internal/data/manifestpb` 0.0% /
  `internal/pprofutil` 0.0%** — thin or generated/test-fixture packages.
  Recorded as baseline; not regression targets for this cycle.
- **`internal/data` 56.0% / `internal/lifecycle` 56.0%** — meaningful logic
  with the lowest real coverage among hot-path packages. Candidate residual
  risks (durability dimension, US-011/US-012).
