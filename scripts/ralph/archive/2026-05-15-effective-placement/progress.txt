- **Learnings for future iterations:**
  - The smoke script's "drain succeeds at admin level even when stuck_single_policy>0" property is server-side semantic — the gateway never gates the drain POST on impact analysis; only the UI gates Submit via `<ConfirmDrainModal>`'s complianceLockedCount > 0 check. The PRD AC line "drain Submit returns 409 stuck buckets exist" describes the UI gate, not the server. Smoke validates the server side (stuck_single_policy classification, 503 DrainRefused on mid-drain PUT, deregister_ready=false until fixed); Playwright validates the UI side (Submit disabled).
  - Per-row `placement_mode_override` is set by the server in `suggestedPoliciesForBucket` (US-003 effective-placement) — the strict-stuck bucket's index-0 suggestion is ALWAYS the Flip-to-weighted shortcut with `placement_mode_override: "weighted"`. The client (BulkPlacementFixDialog) just threads that field into the PUT body via `resolveModeOverride(choice)`. No client-side mode-decision logic — the truth lives on the server.
  - When spoofing PUT /placement in a Playwright e2e, mirror the server's drain-impact cache invalidation by dropping the matching stuck-bucket entry from the closure-state array (and decrementing stuckSingleChunks accordingly) so the parent modal's TanStack refetch sees stuck=0 and unlocks Submit. The drain-transparency.spec.ts already established this pattern; we reused it.
  - Two-route patterns coexist in the spoofed admin API: `/admin/v1/buckets/*` (the bucket detail GET) and `/admin/v1/buckets/*/placement` (the placement triple). Playwright route matching is glob+regex; place the more-specific route LAST so the placement route handler doesn't get masked by the bucket detail catch-all. The spec defines placement first then BucketDetail second to ensure correct precedence.
  - `pnpm typecheck` uses `tsc -b --noEmit` which respects project references. When a spec file (`web/e2e/*.spec.ts`) references types from `web/src/api/*`, TS picks them up via the spec's tsconfig — but Playwright's `import type` of `Route` from `@playwright/test` must be value-import-compatible because it's also used at runtime in route handler signatures. Keep `import { type Route }` in the named import list rather than a separate `import type { Route }`.
  - CLAUDE.md cluster-state-machine section is the single source of truth for picker semantics; adding a new dimension (per-bucket mode) belongs INSIDE that section as a sub-block, not as a new top-level heading. Future PUT-routing changes should land their semantic notes there too — the section is already the operator's mental model anchor.
  - The Hugo theme is a git submodule (`docs/site/themes/`) — running `make docs-build` without submodule init fails silently with `theme not found`. Existing developer/CI environments already have it initialized; new contributors need `git submodule update --init`. Not a regression risk in this cycle but worth flagging if a fresh clone hits the wall.
