## Codebase Patterns

(seed entries for this cycle. Replace/remove as US-001..US-019 land.)

- data.Backend interface (internal/data/backend.go) is the contract; three
  in-tree implementations now: memory (test), rados (primary), s3 (this
  cycle, US-001+). All four ops are stream-shaped: Put(reader, size),
  Get/GetRange returning io.ReadCloser, Delete. New backend impls live in
  internal/data/<name>/ — mirror the existing s3 package layout.
- Native shape ONLY: one Strata object = one backend S3 object via the
  backend's own multipart upload. Never split a Strata object into N
  4-MiB-chunks-as-S3-objects — request count amplification kills it.
  Manifest.Chunks is empty when BackendRef is set (US-008).
- VersionId is the defensive correctness lever (US-002, US-004, US-008,
  US-010). Capture from SDK response on every PUT/Complete; pass back on
  every Delete. Three-state semantics: '' (backend without versioning, or
  versioning-off — plain delete), 'null' (versioning suspended — versioned
  delete with literal 'null'), <uuid> (versioning enabled — versioned
  delete bypasses delete-marker creation). Never branch on backend
  capability — let the SDK return what it returns and pass through.
- aws-sdk-go-v2 brings ~6 transitive deps. Acceptable for a P1 first-class
  feature. Pin via go.mod; track upstream releases manually (no
  auto-update cron exists).
- Streaming PUT uses feature/s3/manager.NewUploader — single-shot or
  multipart transparently per object size. Memory bound by part-size
  (default 16 MiB) * upload concurrency (default 4) = 64 MiB peak.
- Operator pre-creates the backend bucket. Strata refuses to start if the
  bucket does not exist or is not writable. Strata does NOT own the
  backend bucket lifecycle (auto-create would leak ownership).
- Backend object key format: <bucket-uuid>/<object-uuid> — UUIDv4 first
  4-8 hex chars give AWS S3 enough entropy for automatic prefix
  partitioning, per-bucket grouping is forensic bonus
  (`aws s3 ls s3://<bb>/<bucket-uuid>/`).
- Cross-region operation (Strata in region X, backend bucket in region Y)
  is an operator concern. Document in docs/backends/s3.md (US-017); do
  NOT add startup region-mismatch detection or enforcement.
- Backend versioning on Strata-managed bucket = anti-pattern but supported
  defensively via VersionID. If operator enables versioning on an existing
  bucket Strata wrote to with empty VersionID rows, those rows leak on
  delete (plain DeleteObject creates delete-marker). Document; do not
  policy.
- Lifecycle bidirectional mapping (US-014) is best-effort. Strata's own
  lifecycle worker keeps running for transition classes the backend does
  not support natively; for natively-supported classes (STANDARD_IA /
  GLACIER / DEEP_ARCHIVE), Strata translates to backend lifecycle and
  skips those objects.
- ROADMAP maintenance rule (CLAUDE.md): every commit closing a ROADMAP.md
  item flips the bullet to strikethrough Done format in the SAME commit,
  with closing SHA inline (or `(commit pending)` with a follow-up SHA
  edit). Every commit discovering a new gap adds a new entry under the
  right severity section in the same commit.

  + internal/meta/tikv/tikvtest/cluster.go. Build tag `integration`.

- HeapInuse-based memory bound assertion: sample HeapInuse during the
  upload via a 20 ms ticker goroutine, take the max, assert peak delta
  < body size. With prod defaults (16 MiB * 4 = 64 MiB pool) HeapInuse
  drifts up to ~100 MiB on docker-localhost uploads (Go's GC sawtooth
  + sync.Pool retention of recently-used part buffers), masking the
  bound — use a tightened test config (PartSize=5 MiB, Concurrency=2 →
  10 MiB pool) to keep the assertion provable. Don't rely on HeapAlloc
  delta — same problem (sync.Pool keeps the buffers live).

- SDK retry config: awsconfig.WithRetryMode(aws.RetryModeAdaptive) +

  429 + connection errors) match PRD US-006 — no need for a custom
  IsErrorRetryable. RetryMode constants live in github.com/aws/aws-

- SDK retry log lines are emitted at Debug classification (see

  o.Logger with a thin filter that re-emits any "retrying request"
  format-prefixed line through slog.Warn. The line carries
  service+operation+attempt; delay is NOT in this format string —
  AC partial on the delay detail.
- Per-op timeout pattern: opCtx + uploadCtx helper methods on Backend
  centralise the WithTimeout/WithCancel choice. Get/GetRange must
  NOT defer cancel — body read happens AFTER the SDK call returns
  and is bound to the same ctx; use cancelOnCloseReader so Body.Close
  releases the context (and the per-op timer). Side effect: opTimeout
  bounds the entire Get lifecycle including body streaming, so
  operators with slow-link large GETs should bump
  STRATA_S3_BACKEND_OP_TIMEOUT_SECS.

  Default adaptive backoff schedules ~3 s between attempts, so a
  three-attempt test runs in ~6 s wall-clock.

  op returns, retry.GetAttemptResults(metadata) returns the per-attempt
  result slice — len(results) - 1 == retry count. Wire via
  o.APIOptions = append(o.APIOptions, instrumentStack) on s3.Options.

- data.WithBucketID(ctx, id) / data.BucketIDFromContext(ctx) (US-009
  internal/data/context.go) thread the Strata bucket UUID through the

  uploadPart, uploadPartCopy, copyObject) — DON'T forget when adding new
  PUT sites or s3-backend objects lose the per-bucket grouping affordance.
- s3api orphan-cleanup pattern (US-009): every callsite that previously
  did `s.enqueueChunks(ctx, m.Chunks)` for a deleted/aborted manifest now
  calls `s.enqueueOrphan(ctx, m)` instead. enqueueOrphan branches on
  manifest shape: BackendRef-shape → synchronous s.Data.Delete (s3
  backend object); chunks-shape → existing GC chunk-deletion queue (rados).
  Add new orphan-cleanup paths via enqueueOrphan, not enqueueChunks.
- s3.Backend's data.Backend bridge (US-009): PutChunks counts bytes via a
  countingReader wrapper around the input io.Reader because manager.
  UploadOutput does not surface the streamed length, then builds Manifest
  with Class+Size+ETag+BackendRef and empty Chunks (1:1 invariant).
  GetChunks rejects chunks-shape manifests with errors.ErrUnsupported —
  s3 backend serves only BackendRef shapes; cross-feeding never happens
  but the defensive check prevents silent fallthrough on a misconfigured
  gateway. Delete on chunks-shape is a no-op (rados handles its own).
- data.MultipartBackend (US-010) is the OPTIONAL capability surface for
  backends that support 1:1 multipart pass-through. Gateway type-asserts
  s.Data.(data.MultipartBackend) at every multipart entry-point and falls
  through to the chunk-based PutChunks path when the assertion fails —
  rados/memory require no plumbing. Handle encoding (s3 backend uses
  <key>\\x00<uploadID>) is opaque to meta + gateway; only the backend that
  produced the handle decodes it. New surface mirrors the SDK's multipart
  contract: CreateBackendMultipart returns handle, UploadBackendPart
  returns per-part ETag, CompleteBackendMultipart returns BackendRef-shape
  Manifest, AbortBackendMultipart is idempotent on NoSuchUpload.
- meta.Store.CompleteMultipartUpload (US-010) trusts a gateway-supplied
  obj.Manifest when non-nil and skips its own chunks-shape assembly. The
  gateway-supplied manifest carries BackendRef from the backend's own
  CompleteMultipartUpload; storing it preserves the 1:1 invariant from
  US-008. Only one of (Manifest.Chunks, Manifest.BackendRef) is populated
  per object — store layer never auto-promotes one to the other.
- Multipart pass-through (US-010) requires meta state to span the upload
  lifecycle: BackendUploadID on meta.MultipartUpload (set at initiate,
  read at every UploadPart and at Complete/Abort) + BackendETag on
  meta.MultipartPart (set at UploadPart, replayed in CompleteBackendMultipart).
  Cassandra schema is additive ALTER TABLE (multipart_uploads +
  multipart_parts), single-column-per-row — old rows decode with empty
  string for the new columns, gateway treats empty as "not pass-through".
- AWS SDK v2 quirk: s3types.NoSuchUpload IS a typed error (matches
  errors.As). Use the typed-error pattern (var nsu *s3types.NoSuchUpload;
  errors.As(err, &nsu)) for idempotent abort — same convention as
  s3types.NoSuchKey for GetObject.
- For multipart-pass-through unit tests without MinIO, an in-process
  http.Handler that mimics the S3 multipart XML (InitiateMultipartUpload-
  Result + ETag header per part + CompleteMultipartUploadResult) hooked
  via http.Client.Transport keeps the SDK's serialization and request
  signing on the SDK side. Bridge with httptest.NewRecorder -> rec.Result()
  inside RoundTrip; nil-check req.Body (CreateMultipartUpload requests
  carry no body and the io.Copy(io.Discard, nil) panics).
- Compose profiles (US-011): every service that's stack-specific carries
  `profiles: [<stack>]`. Cassandra is shared meta and stays unprofiled —
  always runs. Ceph stack = ceph-backend profile (ceph + gateway + gc +
  lifecycle). S3 stack = s3-backend profile (minio + init-minio +
  strata-s3). Default `docker compose up` runs cassandra alone — operators
  must pick a profile to activate gateway/data-plane. `make up-all` /
  `run-gateway` invoke COMPOSE_CEPH; `make up-s3-backend` invokes
  COMPOSE_S3. `make down` passes both --profile flags + -v so all
  volumes (incl strata-minio-data) clear cleanly.
- Compose `service_completed_successfully` is the right depends_on
  condition for one-shot bootstrap containers (init-minio bucket creator).
  Healthcheck-gating doesn't fit because the bootstrap exits — gate on
  exit status instead, then dependent services start once the bootstrap
  reports success.
- /readyz endpoint (cmd/strata-gateway/main.go) is the gateway's container
  healthcheck target. The Open() boot-time writability probe (US-005) is
  the actual readiness gate; /readyz just returns 200 once the HTTP
  goroutine starts serving (backend was already validated). Don't add
  per-request backend checks here — too expensive and would mask transient
  S3 503s as gateway-readiness failures. Healthchecks elsewhere (minio,
  cassandra) follow the same pattern: gate on transport-level readiness,
  not deep liveness.
- mc against a running compose stack (US-012): `docker run --rm --network
  docker_default -e MC_HOST_strata=http://USER:PASS@minio:9000
  --entrypoint mc minio/mc:latest find strata/<bucket> --type f`. Alias
  seeded via MC_HOST_<name> env var — no stateful `mc alias set` step
  needed. `--type f` excludes prefix-only directory entries so `wc -l`
  yields a real object count. The compose project prefix derives from the
  -f file's parent dir (deploy/docker → `docker_default` network +
  `docker_strata-minio-data` volume). Override via COMPOSE_NETWORK /
  COMPOSE_VOLUME env knobs in scripts/smoke-s3-backend.sh if you switch
  to `-p <name>`.
- smoke.sh end-state invariant (US-012): every PUT path has a matching
  DELETE, so a successful smoke run leaves zero remaining objects in the
  backend. That property doubles as the end-to-end orphan-cleanup signal
  for the s3 backend (1 assertion = `mc find ... | wc -l == 0`) — no
  separate synthetic test needed. Future smoke additions that PUT without
  DELETE must add explicit cleanup or this assertion regresses.
- Manifest.SSE (US-013) records the encryption disposition per-object as
  {Mode, Algorithm, KMSKeyID} so the GET path branches on what was
  actually written, not on the current backend config. Mode value
  vocabulary {"passthrough", "strata", "both"}; Algorithm vocabulary
  {"AES256", "aws:kms"} or empty (strata mode). manifest.proto field 12
  is now allocated to the SSEInfo message; sub-fields live in SSEInfo's
  own field-number space (mode=1, algorithm=2, kms_key_id=3). New SSE
  states must extend SSEInfo, not Manifest's reserved range.
- s3 backend SSE wiring (US-013): Open() validates SSEMode against the
  three-value whitelist; b.applyPutSSE / b.applyMultipartSSE attach
  ServerSideEncryption headers (AES256, or aws:kms when SSEKMSKeyID is
  set) on PutObject + CreateMultipartUpload only — UploadPart inherits
  from the session per S3 protocol so the per-part path stays SSE-free.
  PutChunks + CompleteBackendMultipart call b.manifestSSE() to stamp the
  Manifest. New backend handlers (UploadPartCopy native passthrough,
  COPY) must add the same applyPutSSE / manifestSSE pair or the
  invariant breaks.
- Gateway SSE response headers (US-013): s3api.getObject surfaces
  x-amz-server-side-encryption (+ -aws-kms-key-id when present) from
  o.Manifest.SSE so clients see per-Strata-object SSE metadata on GET.
  When Manifest.SSE.Algorithm is empty (strata mode, or legacy pre-US-013
  manifests) no header is set — preserves the "no SSE" wire shape that
  pre-US-013 clients expect.
- Destructive CI variant pattern (US-012): `make smoke-s3-backend` is
  safe for repeated local runs (SKIP_DOWN=1 default; non-destructive).
  CI uses `make smoke-s3-backend-ci` which sets SKIP_DOWN=0 and runs the
  `make down` volume-cleanup assertion at the end. Don't fold the
  destructive teardown into the default target — operators want to poke
  the stack after a passing smoke run.
- Optional capability surfaces (data.MultipartBackend US-010,
  data.LifecycleBackend US-014) live in internal/data with primitive args
  to stay cycle-free. Pattern: define interface + flat input types in
  internal/data, do per-feature parsing/translation in the gateway layer,
  type-assert at the gateway entry-point and fall through to the
  chunk/worker path when assertion fails. Don't put internal/lifecycle (or
  any package importing internal/data) types in the interface signature —
  importing internal/lifecycle into internal/data/s3 cycles via worker.go.
- Lifecycle bidirectional translation (US-014): Strata-stored lifecycle is
  the source of truth; backend lifecycle is derived state, re-pushed on
  every PutBucketLifecycle and cleared on Delete. Translation failure logs
  WARN and does NOT fail the user request — a rollout from rados → s3
  with stale IAM still serves PUT lifecycle 200; worker keeps owning
  everything until backend permissions are fixed. Backend filter prefix is
  always <bucket-uuid>/<userPrefix> so an empty Strata prefix doesn't
  expire ProbeKey or sibling Strata-bucket objects sharing the backend
  bucket.
- CORS bidirectional mirror (US-015): same source-of-truth pattern as
  lifecycle, with one difference — CORS in S3 is bucket-scoped (no
  per-prefix CORS protocol), so data.CORSBackend has no bucketPrefix arg.
  Multi-Strata-bucket-per-backend-bucket deployments see last-writer-wins
  on the backend mirror; Strata-stored config remains per-Strata-bucket
  and authoritative for the gateway response. NoSuchCORSConfiguration is
  not a typed SDK error — match via strings.Contains on err.Error(),
  same shape as NoSuchLifecycleConfiguration handling.
- s3api GET-side union pattern (US-015): when the backend exposes the
  optional Get half of a capability surface (CORSBackend.GetBackendCORS),
  the gateway's GET handler unions Strata-stored XML blob with backend
  rules, with Strata winning ID conflicts. Implementation: parse Strata
  blob, build a set of Strata IDs, append backend rules whose ID is not
  in the set, re-marshal as XML. When backend has no extra rules, return
  Strata blob verbatim (preserves operator's original XML formatting).
  Backend errors log WARN and fall back to Strata blob.
- Per-bucket admin flags (US-016 BackendPresign) live on meta.Bucket
  directly + a SetBucket<Flag> method on meta.Store. Cassandra adds via
  additive ALTER TABLE buckets ADD <col> boolean — NULL on legacy rows
  decodes as Go zero value (false) so existing buckets default to off.
  GetBucket + ListBuckets SELECT must include the new column when added;
  forgetting to update both surfaces silently returns zero-value flags
  on every lookup. Pattern works for any boolean per-bucket toggle.
- New CI e2e variant pattern (US-018): every new e2e variant gets its
  own job in .github/workflows/ci.yml — parallel, no `needs:` on the
  existing e2e job (independent failure surfaces). Each job builds the
  gateway image inline (don't try to share with docker-build), runs
  `make up-<stack>` then `make smoke-<stack>-ci` (the destructive `-ci`
  variant where it exists is a strict superset of the non-CI smoke),
  collects `docker logs` per container into ci-logs/, uploads via
  `actions/upload-artifact@v4` with `name: e2e-<stack>-logs` and
  retention-days 14. timeout-minutes per AC. When adding a new
  alternative-data-backend or alternative-meta-backend stack, follow
  this template — one job, one artefact, parallel-runnable.
- US-016 presign passthrough: data.PresignBackend (PresignGetObject(ctx,
  *Manifest, expires) → URL) is the optional capability surface. The s3
  backend implements via awss3.NewPresignClient(b.client) — local URL
  computation, no network round-trip, so tests can use a stub
  http.RoundTripper. Forward Manifest.BackendRef.VersionID to the
  GetObjectInput so versioning-enabled backends resolve to the same
  version Strata recorded at write time. Default expires (when caller
  passes 0) = SDK default 15 minutes. Gateway-side passthrough decision
  lives in s3api.maybeBackendPresignRedirect: bucket flag on AND
  X-Amz-Signature query param present AND data.PresignBackend assertion
  AND Manifest.BackendRef present ⇒ 307 Temporary Redirect with Location
  + audit slog.Info(presign_passthrough=true). Any miss falls through to
  in-process serve. Backend mint error logs Warn and serves bytes — never
  surfaces the failure to the client. HEAD bypasses passthrough by design
  (presign HEAD is rare; gateway-side response is cheap).

## 2026-04-30 — US-010
- Added optional data.MultipartBackend interface (internal/data/backend.go)
  + data.BackendCompletedPart input type. Gateway type-asserts at each
  multipart entry-point so rados/memory backends keep working unchanged.
- Implemented MultipartBackend on *s3.Backend in new internal/data/s3/
  multipart.go: CreateBackendMultipart issues SDK CreateMultipartUpload
  at <bucket-uuid>/<object-uuid> (bucket id read from data.WithBucketID
  ctx, same convention as PutChunks); UploadBackendPart streams the
  part to SDK UploadPart and returns the per-part ETag (quote-stripped);
  CompleteBackendMultipart issues SDK CompleteMultipartUpload and builds
  a BackendRef-shape Manifest (Chunks empty per US-008 invariant);
  AbortBackendMultipart is idempotent on NoSuchUpload (typed error via
  errors.As). Backend handle = key + "\\x00" + SDK uploadID; opaque to
  callers.
- Schema-additive on meta types: MultipartUpload.BackendUploadID +
  MultipartPart.BackendETag — both string, both omitempty in JSON via
  zero-value short-circuit (no struct json tags needed since those types
  aren't marshalled). Cassandra schema gets two ALTER TABLE statements
  (multipart_uploads.backend_upload_id, multipart_parts.backend_etag);
  Insert/Select/List queries on those tables now read+write the new
  columns — empty strings on legacy rows decode cleanly. Memory store
  carries new fields by struct copy (no codec changes).
- meta.Store.CompleteMultipartUpload now honours a gateway-supplied
  obj.Manifest (preAssembled = obj.Manifest != nil): when set, store
  trusts the supplied manifest (BackendRef shape) and only computes
  totalSize defensively when obj.Size == 0. Memory + Cassandra share
  this contract.
- Wired s3api multipart endpoints (internal/s3api/multipart.go):
  initiateMultipart calls mb.CreateBackendMultipart on the
  data.MultipartBackend type-assertion (with data.WithBucketID(ctx, b.ID)
  for the <bucket-uuid> prefix) and persists the handle on the meta row;
  cleanup-on-meta-failure aborts the backend session. uploadPart routes
  to mb.UploadBackendPart when mu.BackendUploadID != ""; saves
  BackendETag (= per-part backend ETag) on the meta row, ETag mirrors
  backend ETag for the wire response. completeMultipart calls
  mb.CompleteBackendMultipart with backendETag from each persisted part,
  stamps obj.Manifest with the BackendRef-shape result + totalSize, then
  delegates to s.Meta.CompleteMultipartUpload (which now skips its own
  chunks-shape assembly per the preAssembled path). abortMultipart
  fetches mu.BackendUploadID before AbortMultipartUpload, then calls
  mb.AbortBackendMultipart so the backend session is terminated even
  though the meta-side abort returns no chunks-shape manifests for
  pass-through parts.

    surfaces AES256/aws:kms; strata mode surfaces nothing. Saves
    a round-trip to the backend just to inspect the SSE header that
    Strata already knows from its own write.

    needs a key for conflict detection. Used rule ID — non-empty IDs
    deduplicate across (Strata, backend); empty-ID backend rules always
    append (no collision). Operators who mix manual backend-side rules

  - When Strata blob covers everything backend reported (zero rules to
    append), maybeMergeCORS returns ok=false and the caller writes the

    operator's original XML formatting (whitespace, attribute order)
    which is otherwise lost through xml.Unmarshal → xml.Marshal.
  - Preflight-on-presigned-URL AC (US-016 territory) doesn't ship in this
    story — today's Strata preflight handler still serves OPTIONS. When
    US-016 lands, presigned-URL clients will hit the backend directly,
    and the backend's CORS mirror (this story) will be the rule set
    that applies.

  - The auth middleware validates SigV4 query parameters on presigned
    requests (internal/auth/middleware.go validatePresigned). By the
    time getObject runs, the signature is verified — IAM allow GET is
    implicit. PRD AC "Strata pre-checks before issuing the URL" needs
    no additional handler-level check today (Strata has no per-object
    ACL eval beyond signature). Future ACL/Policy work should add a
    pre-check inside maybeBackendPresignRedirect.

    that Open() adds to the regular client — presign URLs are signed
    locally and never executed by the SDK. The s3 backend reuses
    b.client (which carries those middlewares) but they're a no-op on
    the presign code path.

    Range header on the original presigned GET is replayed against
    the backend, so partial-content semantics still work post-
    passthrough. Never use 302 here — 302 lets the client downgrade
    to GET and drop headers.
  - meta.Bucket lookups happen on every object request — adding fields
    to Bucket has zero per-bucket cost beyond the SELECT column. But
    Cassandra schema migrations require coordinating GetBucket SELECT,
    ListBuckets SELECT, and the alterStatements list. Forgetting any
    of the three silently returns zero-value flags.
  - Stub PresignBackend tests need a Backend constructed via Open()
    with SkipProbe=true and a stub http.RoundTripper (the Open() probe
    would otherwise round-trip a PutObject + DeleteObject before
    returning). Endpoint set to a fake URL; the URL host shows up in
    the minted presign URL so tests can assert on it.
  - s3api harness fakes need to fake BackendRef on the manifest after
    PutChunks (the inner memory backend produces chunks-shape
    manifests; the gateway requires BackendRef to mint a backend
    URL). Strip BackendRef back off before delegating GetChunks /
    Delete to the inner memory backend so it can find the chunks.
