## Codebase Patterns
- Operator one-shot tools (rewrap, rebuild-index) are `strata admin <name>`
  SUBCOMMANDS (single-binary invariant), NOT new binaries. They connect
  DIRECTLY to backends via `buildRewrapMetaStore(cfg,logger)` (meta) +
  `buildBenchDataBackend(cfg,logger)` (data) — NOT the gateway HTTP client. The
  admin package pulls darwin-incompatible deps (gosigar) so it cannot build on
  local macOS; verify with `CGO_ENABLED=0 GOOS=linux GOWORK=off go build
  ./cmd/strata/admin/`. Dispatch in run.go's top-level `rest[0]==` chain (for
  no-subcommand verbs) + the usage banner.
- RADOS data-plane primitives live in TWO places: always-on shared shape +
  sentinel in main module `internal/data/rados/` (no go-ceph, builds CGO-free),
  real librados impl in the separate `cephimpl/` Go module. Mirror `rados.New`:
  a main-module dispatcher type-asserts `data.Backend` to a capability interface
  and returns `data.ErrRADOSNotCompiled` when it's not the librados backend.
  Keep reusable helpers (rate limiters, parsers) in the main module so cephimpl
  needs no extra deps. Hermetic gate: `GOWORK=off go mod graph | grep -c go-ceph`
  must stay 0.
- Local macOS CANNOT compile cephimpl (cgo + Xcode license + gosigar darwin).
  Verify main-module logic with `CGO_ENABLED=0 GOWORK=off go test ./internal/data/rados/`;
  cephimpl integration legs are CI-only (`make test-rados`). CI (Linux) = truth.
- Chunk OID format is `fmt.Sprintf("%s.%05d", uuid.NewString(), idx)` →
  `<uuid>.<>=5 digits>`. `rados.ParseChunkOID`/`IsChunkOID` is the inverse filter.
- Per-PUT data-tier dimensions ride the data-plane ctx, NEVER the
  `data.Backend.PutChunks(ctx, io.Reader, class)` signature (carries no
  bucket/key/version/mtime). Add a `With*/...FromContext` pair in
  `internal/data/context.go`, read it in the backend `PutChunks`; the gateway
  stamps the ctx via the `dataCtxForPut*` helpers.
- To make a chunk and its meta row agree on `version_id`, the HANDLER must
  pre-decide it (`meta.NewVersionID()`, only for `VersioningEnabled`;
  Disabled/Suspended → `meta.NullVersionID`) and pre-set `Object.VersionID` +
  `Object.Mtime` BEFORE PutChunks — both backends honour a pre-set non-empty
  VersionID. Otherwise version_id is minted by PutObject AFTER PutChunks.
- WEB CONSOLE wiring (full stack for a new operator action): (1) Go handler in
  `internal/adminapi/<name>.go` on the COOKIE-authed `/admin/v1/*` surface (NOT
  the SigV4/IAM-root s3api `/admin/*` surface the CLI/curl uses — the console
  can't sign). Mirror the s3api handler's meta calls + `s3api.SetAuditOverride`.
  Register in `server.go::routes()`. (2) TS api wrapper in `web/src/api/<name>.ts`
  + re-export from `web/src/api/client.ts`; add a query key in `web/src/lib/query.ts`.
  (3) Page in `web/src/pages/<Name>.tsx`; route in `web/src/App.tsx`; nav item in
  `web/src/components/layout/nav.ts`. (4) Playwright spec in `web/e2e/<name>.spec.ts`
  — the e2e gateway runs MEMORY backend with NO workers, so async-job endpoints
  sit queued forever; spoof every endpoint via `page.route()` to drive the
  queued→running→done state machine deterministically (pattern: reshard-progress.spec.ts).
  GOTCHA: the console is `//go:embed all:web/dist` at COMPILE time — `pnpm build`
  BEFORE running e2e (webServer does `go run` which re-embeds current dist).
  GOTCHA: CSS `capitalize` is visual-only — DOM text stays lowercase, assert
  `/running/` not `'Running'`. GOTCHA: hold the `running` state for several poll

  - internal/meta/memory imports github.com/gocql/gocql (TimeUUID helper) → pulls gocql transitively. Do NOT import meta/memory from the cephimpl module (separate go.mod) — it would force gocql into cephimpl/go.mod and break the go-ceph/main-module dep split. Use a tiny in-test fake embedding meta.Store instead.
  - cephimpl integration tests run UNDER the workspace: `make test-rados` runs plain `go test -tags integration` (go.work active), NOT GOWORK=off. So a cephimpl test may import any main-module package that resolves through the workspace; only the standalone `make vet` leg uses GOWORK=off (and it's skipped locally without librados).
  - reconcile.ScannedChunk.Size feeds rebuild.readChunk/restore.readChunkBytes via a single-chunk manifest with GetChunks(m,0,ref.Size). A scanner that leaves Size=0 silently zero-reads every chunk — the S3 scanner sets it from native object size; the RADOS scanner needed WithSize.
  - SSEAlgo is the gateway SSE disposition string ("AES256"/"aws:kms"/""), same value multipart's mu.SSE uses — matches data.SSEAlgorithm* constants. It's an algorithm LABEL only, never key material.
