- **Three discovered failure modes — each addressed independently**:
  1. **Period reconcile bug** (PRD US-005 original scope). `zone create --rgw-zonegroup=X --master --default` doesn't reliably add the zone to the zonegroup's `zones[]` array on Ceph squid 19.2.x. Subsequent `period update --commit` reads the empty zones[] and commits a period whose zonegroup has no zones → RGW daemon refuses to serve. Fix: explicit `zonegroup add --rgw-zone` + `zonegroup modify --master` before `period update --commit`. Idempotent — re-runs are no-op.
  2. **Pool pre-create chicken-and-egg**. `radosgw-admin period update --commit` internally calls `librados::Rados::pool_create` on `default.rgw.{control,meta,log}`; on the single-OSD memstore lab those return EOVERFLOW (34, "Numerical result out of range") which blocks the period commit's zone propagation step. Cycle 1 then ships an empty period; every subsequent operation cascades. Fix: pre-create the three RGW backing pools via `ceph osd pool create` before `period update --commit`. None are dot-prefixed so the plain CLI accepts them.
  3. **Memstore non-persistence ↔ mon DB persistence**. The lab OSD uses `osd_objectstore = memstore` — RAM-only, lost on every container restart. But the mon DB (`/var/lib/ceph/mon`) is volume-persisted and keeps a pgmap pointing at osd.0's expected pgs. Cycle 2's empty OSD comes up, mon insists "these 57 pgs should be on you", PGs sit `stuck inactive, last acting []`, every librados client call blocks indefinitely (`radosgw-admin realm get` was the visible victim — observed stuck in `futex_wait_queue` for >60s). Fix: reset `/var/lib/ceph/{mon,osd,mgr,tmp}` + `/etc/ceph/ceph.conf` + `ceph.client.admin.keyring` at the start of every ceph container boot (`BOOTSTRAP_RESET=1` default). Memstore + reset matches the storage tier's non-persistence semantic; cycles run "fresh ceph each time" which is the only honest behavior for a memstore lab. Set `BOOTSTRAP_RESET=0` for opt-out (tests that need cross-restart persistence — would need bluestore for real, but the opt-out is there for cleanness).
- **Additional safety net**: `radosgw_retry()` wrapper in rgw-entrypoint.sh — 30s `timeout` × 3 attempts around every radosgw-admin invocation. Defense-in-depth for transient OSD slow-ops during the first few seconds after a memstore boot. Rarely fires after fix #3 but cheap to keep.
