### Versioning null literal (US-007 specifics)
- meta.Object.VersionID is `text` type in cassandra. Literal `"null"` (4 chars) sorts AFTER all UUID strings under default `text` clustering (UUIDs are hex+dashes, max ASCII 'f' = 0x66; 'n' = 0x6E > 'f'). Result: latest null shows on top of any earlier UUID-versioned ancestors when versioning was off — desired behaviour for `?versionId=null` resolution.
- Suspended-bucket new PUT must atomically `DELETE old null + INSERT new null` — wrap in LWT batch for cassandra; pessimistic-txn for tikv; mu.Lock for memory.
- Storetest contract entry exercises round-trip; lockstep update across all 3 backends.

### Continuation token format (US-006)
