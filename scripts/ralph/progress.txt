- **Learnings for future iterations:**
  - **Compare-and-set (CAS) is the user-facing rename for both `LWT` and `pessimistic txn`.** US-005 already established this for `scaling.md`. Used again here for `placement-rebalance.md` — the rebalance-worker race-loss wording reads cleaner as "compare-and-set lost the race" than spelling out either Cassandra's LWT semantics or TiKV's pessimistic-txn shape, AND the grep gate keeps both `LWT` and `gocql` out of user-facing prose.
  - **Component refs read as `<package>.<Identifier>`, never `internal/<path>/<file>.go::<Func>`.** `placement.PickCluster` / `rebalance.ShardedFanOut` / `serverapp.ClusterReconcile` are grep-friendly for an engineer who wants to chase the impl, AND user-language clean. The trailing `::FunctionName` paste-from-stack-trace form was the worst offender in `placement-rebalance.md` — it leaks IDE-clipboard noise into operator docs.
  - **`bucket_stats` is grep-banned in user-facing sections — even when discussing the schema.** The Cassandra table NAME is operator-meaningful (it's queryable via `cqlsh`), but in /best-practices/ prose use "bucket usage stats" / "bucket-usage stats table". `/architecture/` may name the table directly under US-009's carve-out. `quotas-billing.md` still carries `bucket_stats` references — that file is US-007 scope and intentionally untouched this story.
  - **`metric NAMES survive grep — only descriptive prose changes`.** `strata_rebalance_cas_conflicts_total` contains lowercase substring `cas` that does NOT trip the grep gate (case-sensitive `LWT`, not `lwt`). Counter / histogram names are external API per Prometheus dashboard contract; rewriting them would break operator alerts. Pattern locked: sanitize the surrounding sentence, leave the metric identifier verbatim.
  - **`gocql.QueryObserver` parenthetical was the only `gocql` token in /best-practices/.** Removing the `(gocql.QueryObserver)` parenthetical in `tracing.md` L26 cleared the entire section's `gocql` count — the rest of the file references the abstraction ("query observer") without naming the gocql interface. Keep this pattern: when refactoring out gocql/goceph mentions, the interface concept usually survives as a noun phrase ("query observer", "RADOS observer").
