## Codebase Patterns
- `internal/s3api/race_test.go` is the existing race harness — drives mixed PUT/GET/DELETE/multipart concurrency against in-memory backend. `internal/s3api/race_integration_test.go` is the build-tag-gated integration variant against testcontainers Cassandra. Move shared workload helpers (raceFixture, newRaceClient, raceWorkload) into `internal/racetest/` package; existing tests become thin shims.
- `cmd/` currently holds two production binaries: `strata` (gateway + workers) and `strata-admin` (one-shot ops, currently rewrap). New `cmd/strata-racecheck/` is a developer/CI tool — explicitly NOT counted as a regression on the binary-consolidation goal. Note this in `tasks/prd-binary-consolidation.md` Non-Goals.
- The `make build` target wraps `go build ./...` — adding a new binary needs no Makefile edit, but `make build`-implied artefacts (bin/strata-racecheck) should drop into bin/.
- Race harness drives gateway over HTTP, so `cmd/strata-racecheck` builds without `-tags ceph` — pure HTTP, no librados linkage. Important for portable CI image.
- SigV4 path: harness MUST use static credentials with `STRATA_AUTH_MODE=signed` (NOT `optional`). Streaming SigV4 (`x-amz-content-sha256: STREAMING-AWS4-HMAC-SHA256-PAYLOAD`) is a separate code path with chain HMAC validation; include both at ~50/50 weight per US-003 to exercise both signing flavors.
- `internal/auth/streaming.go` chain HMAC validation (US-022 of prior cycle) — workload should hit it via streaming PUTs to catch regressions in `computeChunkSignature` + `prevSig` chaining.
- Cassandra LWT under sustained concurrent versioning flips is the highest-risk hot path. Workload weights versioning flips at ≥10% of all ops per Technical Considerations.
- Workers (gc, lifecycle, replicator, access-log, notify, inventory, audit-export, manifest-rewriter, quota-reconcile, usage-rollup) are not active during race-soak — single-region, no replication, healthy stack only. Chaos / fault-injection / replication-races are out of scope (future P2).
- `make up-all` brings up the full developer stack: cassandra + ceph + strata + prometheus + grafana. Memory footprint exceeds 7 GB on idle. The cycle adds `make up-all-ci` (compose override `deploy/docker/docker-compose.ci.yml`) that caps Ceph osd_memory_target=1Gi, Cassandra MAX_HEAP_SIZE=2G, drops Prometheus/Grafana, RF=1 — fits ubuntu-latest budget (2 vCPU / 7 GB RAM / 14 GB SSD).
- `/readyz` probe wires up via `internal/health` — readiness gate for both the harness driver script and the CI smoke step. Ceiling 10 min on cold runner pulls.
- Nightly CI runs on free `ubuntu-latest` runners (public repo → unlimited minutes). 90 min job timeout. 03:00 UTC daily cron. NOT triggered on pull_request.
- Report shape: JSON-lines events (`op_started`, `op_done`, `inconsistency`, `summary`). One Markdown summary (`report/SUMMARY.md`) embedded into `$GITHUB_STEP_SUMMARY`. Artefact retention 30 days.
- Verifier oracles: (1) read-after-write — every successful PUT records expected (bucket, key, version_id, etag, size); verifier GETs latest, ETag/size mismatch = `inconsistency` event; (2) versioning monotonicity — after EnableVersioning flip, subsequent PUTs produce monotonically increasing version_id per key; (3) listing convergence — after DeleteObjects batch, deleted keys do NOT reappear in ListObjects within configurable grace window (default 5s).
- Concurrency cap: harness refuses --concurrency > 64 (empirical OOM ceiling on 7 GB ubuntu-latest). Default 32. Plausible nightly throughput 200-500 ops/sec — goal is consistency-violation detection, not benchmarking.
- Baseline file (`docs/racecheck/baseline-2026-XX.md`) is a human-readable record, NOT a test fixture. Nightly runs do not assert against it. Regression-pinning is explicitly out of scope.
- ROADMAP close-flip rule: same commit flips P1 entry to `~~**P1 — Race harness as a real test, not a gate.**~~ — **Done.** ... (commit `<pending>`)` per CLAUDE.md Roadmap maintenance. Closing-SHA backfill follow-up commit on main per established pattern.

- racetest event sinks (US-002): `EventSink` interface + nopSink/writerSink/fileSink. ReportPath beats EventWriter when both are set. fileSink wraps writerSink with a buffered writer so the 64 KiB bufio buffer absorbs the per-op churn at ~500 ops/sec. Concurrency-safe via single mutex on the underlying io.Writer.
- racetest SigV4 wiring (US-002): aws-sdk-go-v2 v4.NewSigner is the signer. Body is hashed with sha256 (matches the gateway's verifier). Empty AccessKey returns nil signer; anonymous flow stays unchanged so existing in-process tests keep working without auth setup.
