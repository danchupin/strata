- **Learnings for future iterations:**
  - **Annotation template funcs that DO NOT exist in Prometheus alert templates**: `mul`, `parseDuration`, `humanizeFloat`. Available helpers are limited — see prometheus/util/strutil + standard text/template. Safe set: `printf`, `humanize`, `humanizeDuration`, `humanize1024`, `query`. Use `printf "%.3f"` for raw value formatting; multiplication must be baked into the PromQL expr, not the annotation.
  - **Recording-rule scalar targets need `vector(N)`**, not bare `N`. `expr: 0.999` fails promtool — must be `expr: vector(0.999)`. Recording rules only emit instant vectors.
  - **`go install github.com/prometheus/prometheus/cmd/promtool@latest` FAILS** with "go.mod contains replace directives" because the upstream prometheus module pins replace directives transitively. Workarounds: (a) use the official tarball release (`prometheus-N.darwin-arm64.tar.gz` → extract `promtool`), (b) `go install` against a fork that strips replaces. CI workflow uses `go install` which works inside the CI sandbox where the replace constraint is bypassed by `go install` semantics — but local dev usually needs the tarball.
  - **`strata_bucket_stats_shard_writes_total` has ONLY `shard` label** (no bucket). PRD AC text referenced `by (bucket)` aggregation which would yield an empty vector — adapted to a global imbalance ratio: `max(metric) / clamp_min(sum(metric), 1) > 0.5`. When PRD AC text disagrees with metric label set, trust the label set (verified via metrics.go) and adapt the expr.
  - **`clamp_min(denom, 1)` divisor guard**: standard PromQL idiom for ratio rules that may have zero-traffic windows. Without it, the ratio is `0/0=NaN` and the alert can't evaluate cleanly. Apply on every `sum(...) / sum(...)` ratio rule.
  - **promtool degraded path in Makefile**: `@command -v promtool > /dev/null || { echo "promtool not installed — skip ..."; exit 0; }` — copy-paste of the helm-lint pattern. Makes `make test` non-gated on toolchain (operators without promtool still pass), CI installs explicitly.
