## Codebase Patterns
- Multi-cluster data backends already exist: RADOS via `STRATA_RADOS_CLUSTERS` (+ `STRATA_S3_CLUSTERS` for S3-over-S3) parsed into `internal/data/rados/Backend.clusters map[string]*radosCluster` and `internal/data/s3/Backend.clusters map[string]*s3Cluster`. The Backend already routes per-storage-class via `STRATA_S3_CLASSES` (`resolveClass`). This cycle adds per-BUCKET routing via a new `meta.Bucket.Placement map[string]int` (weight by cluster). The router is hash-mod stable so a chunk's home cluster never depends on iteration order.
- Bucket blob-config helpers live in `internal/meta/cassandra/store.go::setBucketBlob/getBucketBlob/deleteBucketBlob` (reuse pattern from lifecycle/CORS/policy/ownership-controls). Reuse for `Placement` and `cluster_state` rather than custom CRUD per endpoint.
- `meta.Store` contract test infra: `internal/meta/storetest/contract.go::Run(t, factory)` is the shared suite. Add `caseBucketPlacement` + `caseClusterState` (US-006) so memory + cassandra + tikv get exercised in lockstep.
- TiKV bucket-config keys follow the variable-length byte-stuffing convention (`internal/meta/tikv/keys.md`). Use the existing per-bucket-blob prefix `s/bk/<bucketID>/<configKind>` shape — placement is `configKind="plcm"`. `cluster_state` is NOT bucket-scoped — use a top-level prefix `s/cs/<clusterID>`.
- Worker registration: every worker has a per-worker `cmd/strata/workers/<name>.go` calling `workers.Register(workers.Worker{Name, Build, SkipLease})` from `init()`. For rebalance: `SkipLease: false` (single leader cluster-wide on `rebalance-leader`).
- Worker iteration spans (shipped in complete-tracing cycle): use `internal/otel.StartIteration / EndIteration` for the per-tick parent span (name `worker.rebalance.tick`); attrs include `strata.component="worker"` + `strata.worker="rebalance"` + atomic uint64 `strata.iteration_id`. Sub-op spans (`rebalance.scan_bucket`, `rebalance.move_chunk`) created via `tracer.Start(ctx, "<name>")` under the parent ctx.
- `Dependencies.Tracer` is already `*strataotel.Provider` (registry.go:39) — use `deps.Tracer.Tracer("strata.worker.rebalance")` to get a named Tracer. Nil-tracer safety: if `tp == nil` (unit tests) the iteration helper falls back to `tracenoop`.
- Manifest CAS via `meta.Store.SetObjectStorage(ctx, bucket, key, versionID, expectedClass, newClass, manifest) (applied bool, err error)`. Pattern: pass `expectedClass == currentClass` because rebalance doesn't change class — only the chunk locations inside the manifest. If `applied=false`, discard the new chunks (enqueue them into GC); the concurrent client write wins. Mirrors the lifecycle transition path exactly.
- GC enqueue: `meta.Store.EnqueueGCEntry(ctx, GCEntry{Pool, OID, ...})` already exists. Used by lifecycle and now rebalance — same grace window applies, the gc worker eventually deletes per `STRATA_GC_GRACE`.
- Backwards compat critical: every existing bucket has `Placement == nil`. The router (`placement.PickCluster`) MUST return `""` for empty/nil policy so callers fall back to the pre-existing `$defaultCluster` routing. Zero schema migration, zero behavior change for unconfigured buckets.
- Hash function for stable routing: `fnv32a("<bucketID>/<key>/<chunkIdx>") % sum(weights)` walking a weight-wheel over sorted cluster IDs. Sorted-cluster-id order is critical for determinism — `map` iteration order is random in Go. Use `sort.Strings(slices.Collect(maps.Keys(policy)))`.
- Drain sentinel cache: PUT hot path reads `cluster_state.state` per chunk. Burden Cassandra/TiKV on every PUT is unacceptable. Use in-process cache with 30s TTL (`internal/rebalance/draincache.go` or `internal/data/draincache.go`). Drain is a slow operator action — eventual consistency for 30s is fine.
- Throttle primitives: `golang.org/x/time/rate.Limiter` for byte-rate; `golang.org/x/sync/errgroup` for inflight bound. Shared helpers go in `internal/rebalance/throttle.go`.
- RADOS `MonCommand({"prefix":"df"})` returns JSON with `stats.total_bytes` + `stats.total_used_bytes`. Parse + cache per-cluster (60s TTL OK). S3 has no native cluster-fill semantics — `ClusterStats` for S3 returns `ErrNotSupported`; the worker treats this as "OK to proceed" with one WARN per iteration.
- Admin API audit: every admin write must `s3api.SetAuditOverride(ctx, action, resource, bucket, principal)` inside the handler. `action="admin:PutBucketPlacement"` / `admin:DrainCluster` / `admin:UndrainCluster`. Listing handlers (GET) skip audit via `auditableMethod`.
