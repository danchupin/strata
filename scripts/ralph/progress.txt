### Learnings for future iterations
- Patterns discovered:
  - Long-running admin background jobs follow a fixed shape: meta.AdminJob row + worker_locks lease + goroutine that flips state pending→running→done|error. Reuse `internal/adminapi/buckets_delete.go::startForceEmptyJob` + `runForceEmptyJob` as the template — kind constants live in meta/store.go (`AdminJobKind*`, `AdminJobState*`).
  - When a handler kicks a goroutine that mutates the same struct it then writes to the response, the goroutine MUST receive a *copy* of the pointer (or the handler must build the response *before* spawning). Race-detector will catch the latter eventually but it's faster to be defensive at the boundary.
  - The leader.Locker is a fully working primitive for ad-hoc admin-handler leases — not just worker leader-elections. Acquire with TTL ~10 minutes (force-empty default), release in the goroutine's defer. The same pattern will fit "audit-export run-now", "manifest-rewriter run-now", etc. in later phases.
  - shadcn doesn't ship a Progress component out of the box. A two-div CSS bar (`<div className="h-2 ... bg-muted"><div style={{width: <pct>%}}/></div>`) is enough for slow-poll progress UI; pulling Radix/Progress for a single use-site is not warranted.
- Gotchas:
  - The TiKV pessimistic-txn `LockKeys` lease is held for the txn lifetime; CAS-style early returns (e.g. CreateAdminJob seeing the key already exists) MUST `txn.Rollback()` explicitly before returning the non-error result, otherwise the mock memBackend used by unit tests deadlocks the next caller.
  - Memory `Locker` is in-process. Tests that simulate "another node holds the lease" must pre-`Acquire` with a distinct holder string and Cleanup-Release; otherwise the handler's own subsequent acquire would simply succeed (single-locker, single-process semantics).
  - Cassandra `admin_jobs` schema is new — existing keyspaces upgrade additively via the idempotent CREATE TABLE IF NOT EXISTS in `tableDDL`. No `alterStatements` row needed because the table starts wide.
- Useful context:
  - `internal/serverapp/serverapp.go::buildLocker` returns `nil` if the meta backend exposes no locker. Admin handlers that depend on the lease must `if s.Locker == nil { 503 }` so dev rigs without the locker fail loud rather than dropping the lease silently.
  - The drain loop calls ListObjectVersions first (so versioned buckets surrender every version cleanly) and falls back to ListObjects when the bucket has no version rows. memory.Store mirrors both surfaces; cassandra/tikv use the same code path.
  - `AdminApiError` + `buildAdminError` in `web/src/api/client.ts` is the canonical error shape for admin API wrappers — replace the bespoke `CreateBucketError` (US-001) with the shared shape on the next dialog touch-up; new wrappers (US-003+) should use `AdminApiError` from day one.

### Learnings for future iterations
- Patterns discovered:
  - Per-row mutations on a list page (Disable/Enable access key) use `useMutation` directly — onSuccess invalidates the per-list queryKey + fires a `showToast`; onError surfaces the AdminApiError message in a destructive toast. - Gotchas:
  - meta.Store interface change drives a wave of compile errors in the unit-test files that built `*memory.Store` literals — the diagnostics framework caught these immediately, so fix order is: interface → cassandra → tikv → memory before running any test target. The ImpossibleAssert / InvalidIfaceAssign messages all point at the same root cause (missing method).

  - TiKV pessimistic-txn lease holds for the txn lifetime; CAS-style early returns (e.g. `UpdateIAMAccessKeyDisabled` finding the row missing) MUST call `txn.Rollback()` explicitly before returning the sentinel error — `defer rollbackOnError(txn, &err)` only fires on `err != nil`, but the assignment-to-named-err path means rollback fires correctly here.   - Frontend: `AdminApiError` is an INTERFACE in client.ts, not a class — synthesise via `new Error(msg) as AdminApiError; err.code = ...; err.status = ...`. The TypeScript compiler will reject `new AdminApiError({...})` with TS2693.

### Learnings for future iterations
- Patterns discovered:
  - When a meta.Store method needs an inverse-index lookup (ListPolicyUsers, future "users with role X" or "objects in storage class Y at scale"), wire up the inverse-index TABLE first (cassandra: PK (target, source) partition; tikv: separate prefix; memory: second map) — then expose the scan via a dedicated Store method. Pair the operator-spec'd table with the inverse and write/delete in lockstep so the views never desync. US-010 already shipped the inverse-index plumbing; US-013 just exposed it on the public Store.
  - Go 1.22 net/http mux trailing wildcards are powerful for path segments containing slashes (S3 keys, IAM ARNs, RBAC subjects) BUT cannot have a static suffix after the wildcard — `{arn...}/document` is rejected. When AC names a URL with both, drop the suffix and overload the verb (PUT replaces the document; the policy ARN IS the resource). Document the deviation in the PRD notes — this is a clean REST collapse, not a capability drop. Reuse this trick for any future AC that puts an ARN/key in the middle of a URL.
- Gotchas:
  - The existing inverse-index from US-010 (PolicyUserPrefix in tikv, iam_policy_attachments in cassandra, policyUsers map in memory) was already populated by AttachUserPolicy/DetachUserPolicy across all 3 backends, so US-013's ListPolicyUsers was a *read-only* surface addition — no schema migration. Adding a public Store method on top of an existing inverse index is the cheap path; don't go inventing a new index when one already exists.
  - The contract test at internal/meta/storetest/contract.go tests against ALL 3 backends via the same factory. When extending an existing case (here caseUserPolicyAttach), the new assertions must work for every backend — cassandra integration tests and tikv contract tests pick them up automatically via the same Run(t, factory) entrypoint.
  - PolicyAttachedError vs AdminApiError: when a 409 body carries structured fields beyond `{code, message}`, add a sub-type that extends AdminApiError + parse the extra fields manually in the wrapper (deleteManagedPolicy). Existing buildAdminError discards anything beyond {code, message}, so a generic `throw await buildAdminError(resp, ...)` would lose the user list. Keep the status branch explicit when you need extra fields.
