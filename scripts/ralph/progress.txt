- **Learnings for future iterations:**
  - Makefile `.PHONY` line lives at the top (line 4 onward) — extend the existing block when adding new phony targets; don't sprinkle additional `.PHONY:` lines through the file.
  - `make dev` Ctrl-C semantics work because `docker compose logs -f` is the foreground process — SIGINT propagates to the compose CLI, which simply detaches from the stream; the containers themselves stay running. No special trap handling needed in the recipe.
  - The `down` target already cleans up profile-gated services (`--profile cassandra --profile tracing --profile webhook-trap --profile ci`) so `dev-down: down` is sufficient — no need to duplicate the profile list.
  - `dev` deliberately re-uses the existing `wait-*` chain rather than introducing a new one so the `up-all + wait-*` building blocks stay the single source of truth for "lab ready" semantics.
  - Use `make -n dev` to verify the recipe chain without actually starting the stack — surfaces the full expansion of dependent targets.
