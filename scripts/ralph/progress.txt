## Codebase Patterns
- Supervisor.LeaderEvents() (cap-8 buffered) emits per-worker (name, acquired) on lease acquire/release; non-blocking send drops on consumer stall. Channel allocated lazily on first call, closed by Run() after wg.Wait().
- Heartbeater.SetLeaderFor(worker, owned) mutates Node.LeaderFor under mu; write() copies slice under same mu before WriteHeartbeat. Idempotent re-acquire / release-of-unowned.
- ROADMAP close-flip: new entry under ## Web UI, format `~~**P3 — title.**~~ — **Done.** ... (commit `<sha>`)`. Use `<pending>` placeholder if SHA needed inline; follow-up commit fills it (per CLAUDE.md).
- compose `depends_on` only resolves services in the active profile. To depend on a profile-gated service from a different profile, list BOTH profiles on the dependency (e.g. `pd` carries `profiles: ["tikv", "lab-tikv"]`). Otherwise `compose config -q` rejects with "depends on undefined service".
- nginx `nginx -t` resolves upstream hostnames at parse time; if the upstream block names docker service hosts (e.g. `strata-tikv-a`), the syntax-test container needs `--add-host=<name>:127.0.0.1` to resolve. Real DNS happens via Docker embedded resolver at runtime.

- **Learnings for future iterations:**
  - `nginx -t` against a config that names docker-network hostnames in its `upstream { server X:port; }` block fails resolution at parse time (not at request time). Use `--add-host=NAME:127.0.0.1` flags on the test container to satisfy the resolver. The `resolver` directive doesn't help for upstream blocks (those resolve at config load).
  - `client_max_body_size 0` is required so multipart uploads (which can be hundreds of MB) aren't bounced with 413 at the LB tier. Without this default 1m kicks in for nginx 1.27.
  - `proxy_request_buffering off` + `proxy_buffering off` together let the gateway stream large PUTs straight through the LB instead of staging the whole body in nginx's tmpdir first. Required for SigV4 streaming (chunked-encoded uploads must reach upstream verbatim, not via re-buffer).
  - Default `proxy_http_version 1.0` strips chunked encoding; setting it to 1.1 keeps chunking + keepalive working end-to-end.
