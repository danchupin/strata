## Codebase Patterns
- Prior cycles closed the cluster-state machine through 4-state `{live, draining_readonly, evacuating, removed}` (drain-transparency cycle, merged at 6cb848a). This cycle adds the 5th state `pending` + `weight int (0..100)` field per cluster for default routing. Read `feedback_cycle_end_to_end.md` and the User Journey Walkthrough section of `tasks/prd-cluster-weights.md` before any story — each story serves at least one walkthrough step.
- **5-state machine** transitions:
  - `(no row in meta) → pending` (auto-init at boot for new clusters)
  - `(no row in meta) → live weight=100` (auto-init at boot for clusters with existing `bucket_stats` reference — backwards compat)
  - `pending → live` (POST /admin/v1/clusters/{id}/activate {weight: N})
  - `live → live` (PUT /admin/v1/clusters/{id}/weight {weight: N} — weight adjustment in place)
  - `live → draining_readonly | evacuating` (existing POST /drain — from drain-transparency)
  - `draining_* → live` (existing POST /undrain)
- **Two weight layers — NEVER COMBINE**:
  - `bucket.Placement != nil` → policy = bucket.Placement (cluster.weight IGNORED for this bucket — bucket explicit wins)
  - `bucket.Placement == nil` AND class env `spec.Cluster == ""` → synthesise `{<each-live-cluster>: <its-weight>}` from cluster.weight
  - Class env `@cluster` suffix → spec.Cluster set → bypass synthesis (explicit per-class pin)
  - Order in PickCluster MUST short-circuit BEFORE consulting weights when bucket.Placement is non-nil
- **Boot-time reconcile** in `internal/serverapp/serverapp.go::Run`: walk STRATA_RADOS_CLUSTERS env, for each cluster id not in cluster_state → check `bucket_stats` for any reference to this cluster → if present: state=live weight=100 (existing-live); else: state=pending weight=0 (new-pending). Log one INFO per cluster + one summary line. Idempotent (re-running creates nothing).
- **Cluster-state cache invalidation**: critical that POST /activate + PUT /weight invalidate the in-process cache synchronously before returning 200. Otherwise operator changes take 30s to propagate to routing path — bad UX. Pattern: invalidate via `cache.Invalidate(clusterID)` in handler before responding.
