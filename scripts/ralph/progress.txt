- **Learnings for future iterations:**
  - buckets table is partitioned by NAME, not id. There is NO buckets-by-id query path. Resolving a bucket by uuid means either the bucketNames id->name cache (warm in practice — gateway looks up bucket by name before every object op) + GetBucket(name), or getBucketByID which does a FULL ListBuckets scan. The TTL cache is what keeps this off the hot path.
  - A single in-process store CANNOT produce a discriminating red/green test for the per-bucket-vs-global shard bug: a buggy store uses s.defaultShard consistently for BOTH write and read, so same-process PUT->GET round-trips pass even when buggy. The bug only manifests when WRITER and READER disagree on the count. The integration test opens TWO stores over the same keyspace with different DefaultShardCount to force that disagreement. Remember this shape for US-003's post-reshard point-GET proof too.
  - CompleteReshard flips bucket.ShardCount on ALL backends (memory/tikv/cassandra) without needing the worker to have run or job.Done=true — that's how caseOnlineReshard already drives a count change. Use Start+Complete to get a non-default ShardCount in a test (note: reshard only doubles UP; to get a count LOWER than the process default e.g. 16, the bucket must be CREATED by a store whose DefaultShardCount=16).
  - Cross-replica cache convergence: invalidateShardCache is process-LOCAL. Other gateway replicas converge within shardCacheTTL (10s). Fine for US-001 steady-state; US-003 transitional model must not depend on cross-replica synchronous invalidation.
  - gopls modernize hints (rangeint, waitgroup, interface{}->any) fire all over this file pre-existing; they are not lint failures here. Don't churn them.
