## Codebase Patterns
- `meta.Store` interface in `internal/meta/store.go` is implemented by THREE backends (memory, cassandra, tikv). Adding a method requires implementations on all three + a contract test in `internal/meta/storetest/contract.go::cases`.
- Cassandra schema migrations are additive only: `tableDDL` (idempotent CREATE) and `alterStatements` (idempotent ALTER ADD column). Never destructive.
- Cassandra LWT (`IF EXISTS`/`IF NOT EXISTS`) is REQUIRED for read-after-write coherence on any row that may be queried with quorum after a previous LWT write. Plain UPDATE after LWT INSERT leaves Paxos state stale and `LOCAL_QUORUM` reads can observe pre-update value. Counter increment on `bucket_stats` MUST use `UPDATE … IF EXISTS`.
- TiKV pessimistic txn with EARLY-RETURN paths must call `txn.Rollback()` explicitly. `defer rollbackOnError(txn, &err)` only fires when err != nil. Any non-error early return (e.g. CAS-reject `applied=false, nil`) MUST `txn.Rollback()` first or the LockKeys lease leaks for txn lifetime — deadlocks the in-process memBackend forever.
- Per-bucket / per-user single-blob configs (lifecycle, CORS, policy, public-access-block, ownership-controls, quota) reuse `setBucketBlob` / `getBucketBlob` / `deleteBucketBlob` helpers in both backends instead of fresh CRUD per endpoint.
- Workers register via `workers.Register(workers.Worker{Name, Build})` in `cmd/strata/workers/<name>.go::init()`. Build receives `workers.Dependencies`, returns `workers.Runner`. Supervisor handles leader-election (`<name>-leader` lease) + panic restart with backoff.
- Worker envs read at Build time (no per-worker flags). New worker env knobs follow the `STRATA_<NAME>_*` convention.
- AuditMiddleware in `internal/s3api/access_log.go` stamps audit rows from request context. Admin handlers stamp operator-meaningful audit row via `s3api.SetAuditOverride(ctx, action, resource, bucket, principal)` — Action shape is `admin:<Verb>` (e.g. `admin:PutBucketQuota`).
- OpenAPI contract `internal/adminapi/openapi.yaml` is the canonical Admin-API source. New admin endpoints must update it + handler.
- Existing Prometheus metric: `strata_http_requests_total{bucket,access_key}`. New metrics for quota cycle: `strata_quota_reconcile_drift_bytes{bucket}` (gauge).
- Web UI: React+TS under `web/`. Per-bucket page is `web/src/pages/BucketDetail.tsx`. IAM users page exists; new per-user Billing page lands as `web/src/pages/UserBilling.tsx`. Playwright e2e in `web/e2e/`.
- `BumpBucketStats` is the canonical write path for the live counter — every PUT / DELETE / multipart Complete / lifecycle expire MUST go through it. Negative deltas allowed.
- Quota check happens BEFORE writing data (PUT-validate time). Live source-of-truth is `bucket_stats` (bucket scope) + sum-across-user-buckets (user scope). v1 user-scope check uses `ListBuckets(ctx, owner)` walk; denormalised user_bucket_count is a P3 follow-up.
- `byte_seconds × 24h` rollup is intentionally crude v1 approximation. Document the caveat in `docs/site/content/best-practices/quotas-billing.md`. Continuous integration is a P3 follow-up.
- Out of scope this cycle: invoice ledger / payment integration. Stop at the `usage_aggregates` feed.
- Quota enforcement (US-006) is gateway-side, not backend-side. Helper `internal/s3api/quota.go::checkQuota(ctx, bucket, quotaWriteIntent)` is the single entry point; new write paths (CopyObject, server-side copy, etc.) MUST call it BEFORE any `data.PutChunks`. CreateBucket uses `checkUserBucketQuota` for the MaxBuckets cap.
- Tests that need an `r.ContentLength` populated MUST use `bytes.NewReader([]byte)` (or `*strings.Reader`/`*bytes.Buffer`). A custom `io.Reader` like `byteReader([]byte)` produces `Request.ContentLength == -1` because Go's HTTP client only fills it for length-knowing readers — the quota pre-check (and any other ContentLength-gated logic) silently no-ops.
- Quota wire codec lives in `internal/meta/quota.go` — `EncodeBucketQuota` / `DecodeBucketQuota` + user variants. JSON-shaped so additive fields land without an ALTER. All three backends share it.
- Drift-reconcile pattern (US-007) for any future denormalised-counter consistency check: sample counter AFTER the data walk (not before), compute `drift = walk - counterAfter`, gate on `objects != 0 || |bytes| > max(MinDriftBytes, used*MinDriftRatio)`, correct via the existing atomic `Bump*` path. Concurrent activity during the walk composes safely because the bump is atomic and only adds a delta on top — never overwrites.
- Cross-bucket aggregation requires a per-bucket index for variable secondary axes. Cassandra's `(bucket_id, storage_class)` partition key forces a per-class fan-out for any (bucket-scoped, all-classes) read; rather than `SELECT DISTINCT` cluster-wide (antipattern), maintain a sibling index table keyed `((bucket_id), storage_class)` written alongside the main row. TiKV: the same pattern lands as a presence-row sub-prefix (`uc/<escClass>\x00\x00`) range-scanned per bucket. `usage_aggregates_classes` (cassandra) / `UsageClassIndex*` (tikv) is the canonical shape — reuse it for any future per-bucket variable-axis enumeration.
- Day-keyed aggregate rows MUST normalise the day to UTC midnight at the write boundary, regardless of backend. Cassandra `date` columns require midnight; TiKV's 4-byte BE day-epoch (`day-Unix / 86400`) requires midnight; memory's int64 key requires midnight. Without normalisation, a `Day: time.Now()` write and a `Day: time.Now().Truncate(24*time.Hour)` read miss each other. `normalizeUsageDay` (cassandra) / `dayEpoch` (tikv) / `normalizeDay` (memory) all do this — keep the helper at the API boundary, not inside the loop.
- "Not configured" for quota Get is signalled by `(zero-value, false, nil)`, never a sentinel error — gateway PUT-validate stays branch-on-bool.
- Per-user single-blob configs need a parallel triple keyed on `userName` (string). Cassandra: dedicated table (e.g. `user_quota`). TiKV: dedicated key prefix (e.g. `prefixUserQuota`) + helper like `UserQuotaKey`. Memory: `map[string][]byte` field.

- **Learnings for future iterations:**

Today the fan-out happens on every PUT to any bucket the user owns — if someone owns 1k buckets, that's 1k storage round-trips per PUT, so the denormalisation matters more than the prd's "P3 follow-up" tag suggests.
  - `multipartMinPartSize` defaults to 5 MiB; `s3api.SetMultipartMinPartSizeForTest(1)` lowers it for tests that exercise multipart Complete with tiny bodies. Without it, the Complete handler rejects all-but-last parts < 5 MiB with `EntityTooSmall`.

## 2026-05-10 - US-007
- New `internal/quotareconcile` package: `Worker.Run(ctx)` loops on `STRATA_QUOTA_RECONCILE_INTERVAL` (default 6h), `RunOnce(ctx)` is the single-pass operator. Per pass: `ListBuckets("")` → for each bucket walk via paginated `ListObjects` (latest non-delete-marker only — exactly the shape `bucket_stats` represents) and sum `Size` / object count; sample `GetBucketStats` AFTER the walk; drift = walk - stats; bump only if drift trips the threshold gate.
- Threshold gate `driftExceedsThreshold(drift, objects, used, minBytes, minRatio)`: object drift always trips (counts are exact); byte drift trips when `|drift| > max(minBytes, used * minRatio)`. Defaults: `MinDriftBytes=1 MiB`, `MinDriftRatio=0.005` (0.5 %).
- Correction uses `meta.Store.BumpBucketStats(deltaBytes, deltaObjects)` — already atomic on every backend (memory mutex / cassandra LWT / tikv pessimistic-txn) so concurrent live PUT/DELETE during reconcile composes atomically.
- New Prometheus gauge `strata_quota_reconcile_drift_bytes{bucket}` (positive = stats undercount real data, negative = overcount). Registered via `metrics.Register()`.
- New worker registration `cmd/strata/workers/quota_reconcile.go` (init() Register; SkipLease=false → supervisor manages the `quota-reconcile-leader` lease). Build is env-only per the cmd/strata/workers convention.
