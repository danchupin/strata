## Codebase Patterns
- Worker concurrency knobs are read at Build time in `cmd/strata/workers/<name>.go` from `STRATA_*` envs (sibling of existing `STRATA_GC_INTERVAL` etc); no per-worker flags.
- Bounded errgroup pattern for per-entry parallelism: `eg := new(errgroup.Group); eg.SetLimit(N); eg.Go(func() error { ...; return nil })`. Always return nil from goroutines so per-entry failure does not cancel siblings (log-and-continue). Wrap each goroutine in `defer recover()` â€” supervisor's panic backoff is for top-level panics only.
- Counters under concurrent writes use `atomic.Int64` instead of plain `int`.
- `golang.org/x/sync` lives in `go.mod`; it was indirect â€” bumping to direct via `go mod tidy` after first errgroup import.
- Go 1.25 modernization: `for i := 0; i < n; i++` flagged as `rangeint`; use `for i := range n`. Loop var aliasing (`e := e`) is also flagged (`forvar`); not needed since Go 1.22.
- New strata-admin subcommands wire into `cmd/strata-admin/main.go::run` via the same dispatch shape as `rewrap` (no-subcommand variant); the shared meta-store builder is `buildRewrapMetaStore` in `rewrap.go` â€” bench code reuses it directly. Add `buildBenchDataBackend` (bench-only) that mirrors `serverapp.buildDataBackend` for memory + rados; the rados stub keeps the bench buildable without the `ceph` build tag.
- Bench JSONL contract: `benchResult` struct (cmd/strata-admin/bench_common.go) is the source of truth for stdout JSON + Prometheus push gauge labels. `STRATA_PROM_PUSHGATEWAY` is opt-in; failure to push logs WARN and never fails the bench run.
- Bench harness on macOS+lima: rebuild `strata:ceph` via `make docker-build`, stop `strata-tikv-{a,b}` + `strata-lb-nginx` to free the lab from gc/lifecycle worker contention, then run a one-shot container per concurrency level: `docker run --rm --network docker_default -v docker_strata-ceph-etc:/etc/ceph:ro -e STRATA_META_BACKEND=tikv -e STRATA_DATA_BACKEND=memory -e STRATA_TIKV_PD_ENDPOINTS=pd:2379 -e STRATA_LOG_LEVEL=ERROR --entrypoint /usr/local/bin/strata-admin strata:ceph bench-{gc,lifecycle} --{entries,objects}=N --concurrency=M`. Restart the replicas afterwards.
- gc.Worker.drainCount infinite-loops when `Data.Delete` returns persistent errors (e.g. RADOS ENOENT for synthetic OIDs). Tracked under `## Known latent bugs` in ROADMAP.md. Until fixed, run bench with `STRATA_DATA_BACKEND=memory`.

- bench-lifecycle: `--objects=N` (default 10000), `--concurrency=M`. Creates a unique `lcbench-<uuid>` bucket, seeds N objects with Mtime=-2h + a single-chunk Manifest, attaches a 1-day Expiration rule (AgeUnit=Hour so every object triggers in one tick), times `lifecycle.Worker.RunOnce`, prints same JSON shape. Cleanup: drops lifecycle, deletes survivor objects, deletes bucket, drains region GC queue for bench-prefixed entries.
- Makefile bench-gc + bench-lifecycle targets sweep `BENCH_CONCURRENCY_LEVELS` (default `1 4 16 64 256`) and tee one JSON line per level into `bench-{gc,lifecycle}-results.jsonl`. Targets default to TiKV meta + RADOS data via lab-tikv defaults; override via env. `make clean` now also drops the JSONL artifacts.
- Smoke test `TestApp_BenchGC_RunsAgainstMemory` exits 0 against in-memory backends with `--entries=10 --concurrency=1`, asserts JSONL shape (entries=10, concurrency=1, backends=memory). `TestApp_BenchGC_RejectsBadConcurrency` pins the [1,256] flag-boundary check.
