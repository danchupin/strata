- **Learnings for future iterations:**
  - Hugo extended (with SCSS) is required by hugo-book; the Homebrew formula `hugo` already ships extended. Pin a version (e.g. `0.130.0+`) in the GH Action.
  - hugo-book defaults `BookSection = 'docs'` which only renders pages under `content/docs/`. Set `BookSection = '*'` to render all top-level sections (`get-started/`, `deploy/`, `architecture/`, etc.) in the sidebar — that's the layout this cycle uses.
  - When adding a Bash command that needs to `cd docs/site`, use absolute paths from the repo root — earlier `cd docs/site` calls in the same shell drift the cwd and the next call fails. Or use `cd /repo/root && cd docs/site && hugo ...`.
  - `go vet` / `go test` are not affected by Hugo content changes — but the AC asks for both per story, so run them anyway as a safety net.
  - The hugo-book theme's exampleSite has `enableGitInfo = true` + `BookLastChangeLink`. We disabled `enableGitInfo` for now to keep the build deterministic; revisit when GH Action publish lands so per-page "last modified" dates are accurate.
