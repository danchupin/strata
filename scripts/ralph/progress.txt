## Codebase Patterns

- shadcn/ui pattern: components live under web/src/components/ui/<name>.tsx as
  SOURCE (not deps). Each component is hand-written / copy-pasted from upstream
  shadcn-ui CLI output; we own and tweak freely. Path alias '@/*' → 'src/*' is
  wired in vite.config.ts (resolve.alias) AND tsconfig.app.json (paths).
  utilities helper is `cn()` in web/src/lib/utils.ts (clsx + tailwind-merge).
  Color tokens come from CSS variables in web/src/styles/globals.css (`hsl(var(--token))`).
  When adding new shadcn primitives, install the matching @radix-ui/react-<x>
  dep and copy the upstream component file — do not pull a wrapper library.
- Theme system: ThemeProvider context (web/src/components/theme-provider.tsx)
  stores theme = 'light' | 'dark' | 'system'; resolvedTheme is derived from
  matchMedia('(prefers-color-scheme: dark)'). Effect adds/removes
  document.documentElement.classList.dark. Persistence:
  localStorage['strata.theme']. ThemeToggle dropdown in the top bar uses Sun
  (light/system) + Moon (dark) icons from lucide-react. To use theme tokens in
  components: write `bg-background text-foreground` etc. — Tailwind picks up
  the CSS vars from globals.css.
- web/ is a NEW source tree at the repo root (sibling of cmd/, internal/, docs/).
  React 18 + Vite 5 + TypeScript 5. Phase 1 stack will add Tailwind 3 + shadcn/ui +
  TanStack Query 5 + recharts 2 + react-router-dom 6 + Playwright 1. Package
  manager: pnpm 9 (NOT npm/yarn — smaller node_modules, reproducible install).
  Lockfile checked in: web/pnpm-lock.yaml.
- go:embed pattern: ROOT package `strata` (not cmd/strata/) owns console.go +
  console_test.go. Declares `//go:embed all:web/dist` + `var consoleFS embed.FS`
  and exposes `ConsoleHandler()` with SPA fallback (any /console/<path> not matching
  a static file falls back to index.html so client-side routes survive direct
  navigation / refresh). The `all:` prefix matters — without it Vite's .vite/-
  prefixed assets are silently dropped. cmd/strata-gateway imports this as
  `strataconsole "github.com/danchupin/strata"`.
- /console/ + /admin/v1/* are reserved path prefixes on the gateway port. Both
  register ahead of the catch-all / S3 router in cmd/strata-gateway/main.go (no
  internal/serverapp package — wiring is inline in main.go). reservedBucketNames
  in internal/s3api/validate.go already includes 'console', 'admin', 'metrics' so
  path-style addressing cannot shadow gateway-internal routes.
- /admin/v1/* is the JSON HTTP contract between UI and gateway. Versioned (v1);
  future-breaking changes ship under /admin/v2/*. OpenAPI 3.1 spec lives at
  internal/adminapi/openapi.yaml (US-003).
- Auth: session cookie strata_session=<JWT> with HS256, 24h Max-Age, HttpOnly +
  Secure + SameSite=Strict + Path=/admin. JWT secret read from
  STRATA_CONSOLE_JWT_SECRET (random hex if unset, with WARN — production
  deployments MUST set it explicitly to survive restarts without invalidating
  sessions). Auth middleware on /admin/v1/* falls back to existing SigV4 path for
  clients without the cookie (curl, programmatic admin clients).
- TanStack Query 5 global QueryClient: staleTime 5_000, refetchInterval 5_000,
  refetchOnWindowFocus true, retry 1. web/src/api/client.ts exposes typed
  wrappers — pages call useQuery({queryKey, queryFn}); polling is automatic.
- Heartbeat shape: cluster_nodes table with default_time_to_live=30 on Cassandra
  (alterStatements add), s/cluster_nodes/<node_id> + background sweeper on TiKV,
  in-process map with timestamp expiry on memory. Lockstep across all three meta
  backends. Each Strata replica writes its own row every 10 s.
- Status derivation (cluster overview): healthy if all nodes report /readyz 200
  in last 30 s; degraded if ≥1 fail but quorum holds; unhealthy if quorum lost.
- Polling cadence per page: 5 s for ranges ≤6 h, 30 s for 24h+, 5 min for 7d
  (per-query refetchInterval). Mind chart re-render cost on longer ranges.
- Bundle target: ≤500 KiB gzipped initial. Use React.lazy + Suspense for
  non-critical routes (Buckets, Bucket Detail, Metrics get code-split). Verify
  via `vite build --report` before US-012 close. Current bundle: 46 KiB gzipped.
- recharts 2.x for charts (smaller than chart.js, declarative). Avoid Apache
  ECharts / Plotly — bundle bloat.
- shadcn/ui CLI copies components into web/src/components/ui/ as source — NOT a
  dependency. We own them; tweak freely, no version pin to worry about.
- Dark mode: class='dark' on <html>; CSS variables in globals.css switch values
  per theme. Toggle persists to localStorage['strata.theme'] = 'light' | 'dark' |
  'system'.
- Playwright e2e: chromium-only on CI for speed. webServer config runs
  `make run-memory` so e2e does not need Cassandra / RADOS up.
  STRATA_STATIC_CREDENTIALS=test:test:owner seeds login creds.
- Gateway dev port: make run-memory uses :9999 (NOT :9000 as some PRD ACs

- Makefile: top-level `build` depends on `web-build`. `make web-typecheck` and

- Tests for the cycle: Go side via `go test ./internal/adminapi/...`, Playwright

- Stdlib JWT (HS256): crypto/hmac + sha256 + base64.RawURLEncoding for
  compact serialization; hmac.Equal for constant-time signature compare.
  ~200 lines — skip jwt-go / golang-jwt and the CVE chase.
- Cookie Secure flag: conditional on r.TLS != nil OR X-Forwarded-Proto=https.
  Unconditional Secure blocks dev over plain HTTP because browsers refuse
  to set Secure cookies on insecure origins. Production gateways behind
  TLS terminators still hit the secure branch; dev with `make run-memory`
  keeps working. Path=/admin keeps the cookie off /console/ static asset
  requests — only the API picks it up.
- subtle.ConstantTimeCompare on the secret AND a fake compare on the
  unknown-access-key branch — equalises wall-clock latency so attackers
  can't distinguish "unknown key" from "wrong secret".
- adminapi.authMiddleware resolution order: path bypass list (/auth/login,
  /auth/logout) → cookie JWT path → SigV4 fallback → 401. Whoami requires
  auth so the UI uses it as a session probe.
- BrowserRouter basename='/console' means React Router paths are '/login'
  and '/*' — NOT '/console/login'. Basename prefix is added by the router;
  don't double-prefix in <Link to=...> or <Navigate to=...>.
- React auth state machine: 'unknown' | 'unauthenticated' | 'authenticated'.
  The 'unknown' state during the initial whoami probe prevents a flash of
  the login page on hard-refresh of a deep link.
- AppShell layout pattern: web/src/components/layout/AppShell.tsx is the
  authenticated shell — sticky w-60 / w-14 sidebar (left), sticky top bar,
  flex-1 main with <Outlet />. Wrap inside RequireAuth at the route layout
  level (App.tsx uses `<Route element={<RequireAuth><AppShell/></RequireAuth>}>`
  with index/child routes). Stub pages live under web/src/pages/<Name>.tsx
  and export a single named component <NamePage>. New nav items: add to
  web/src/components/layout/nav.ts (icon from lucide-react) AND register
  the corresponding <Route> in App.tsx — both ends must stay in sync.
- Sidebar collapse: useState + localStorage['strata.sidebar.collapsed']
  ('1' | '0'). Collapsed=true forces icon-only regardless of width.
  useMediaQuery('(max-width: 1023px)') forces collapsed when narrow even

  the sidebar can't be expanded against the breakpoint policy.
- Mobile sidebar (<640px via useMediaQuery): desktop <aside> hidden;

## 2026-05-02 - US-004
- Go side: HS256 JWT session cookie + login/logout/whoami endpoints.
  internal/adminapi/session.go (signSession + verifySession + DecodeSecret +
  GenerateSecret) uses stdlib only — crypto/hmac, crypto/sha256,
  encoding/base64.RawURLEncoding. No jwt-go / golang-jwt dependency.
- internal/adminapi/auth.go: handleLogin reads JSON {access_key, secret_key},
  validates via auth.CredentialsStore + subtle.ConstantTimeCompare on the
  secret; constant-time fake compare on the unknown-access-key branch to
  avoid leaking existence. Sets cookie strata_session=<jwt>, HttpOnly +
  SameSite=Strict + Path=/admin + Max-Age=86400 + Secure conditional on
  r.TLS != nil OR X-Forwarded-Proto=https so dev over plain HTTP works.
  handleLogout sets Max-Age=-1 (clear cookie) regardless of session state.
  handleWhoami requires auth; returns access_key + expires_at (from the
  cookie's JWT exp claim, or 0 when authenticated via SigV4 instead).
- internal/adminapi/server.go authMiddleware rewrite. Resolution per
  request: (1) /admin/v1/auth/login + /auth/logout bypass auth entirely;
  (2) cookie present → verify JWT + lookup subject in CredentialsStore +
  attach AuthInfo; (3) Authorization header present → fall through to
  SigV4 path; (4) anonymous → 401. The cookie path is preferred for
  browser traffic; SigV4 is the fallback for CLI clients.
- adminapi.New signature gained jwtSecret []byte.
  cmd/strata-gateway/main.go loadJWTSecret() reads
  STRATA_CONSOLE_JWT_SECRET (hex preferred; raw bytes fallback when not
  hex) or generates an ephemeral 32-byte secret with a WARN log line —
  production deployments MUST set the env to survive restarts without
  invalidating sessions.
- internal/adminapi/openapi.yaml documents POST /auth/login (security:
  [] override — anonymous), POST /auth/logout (anonymous), GET
  /auth/whoami (cookie/sigv4 secured) + LoginRequest/SessionInfo
  schemas. Set-Cookie header description matches the runtime shape.
- React side: react-router-dom@6.28 added.

- **Learnings for future iterations:**
  - Stdlib JWT is fine. base64.RawURLEncoding (NOT StdEncoding) per
    JWT compact serialization; hmac.Equal for the signature compare
    (constant-time). 200 lines total — no need to import a JWT
    library and chase its CVEs.
  - The cookie's Secure attribute breaks dev-over-plain-HTTP because
    browsers refuse to set Secure cookies on insecure origins.
    Conditional on r.TLS != nil OR X-Forwarded-Proto=https keeps
    `make run-memory` working AND honours the spec in production
    (where the gateway is behind a TLS terminator). Path=/admin
    means the cookie does NOT travel with /console/ static asset
    requests — only the API path picks it up. That's intentional.
  - subtle.ConstantTimeCompare returns int (0 or 1). Add a fake
    compare on the unknown-access-key branch so the unauth branch
    times the same as the bad-secret branch — otherwise a wall-clock
    attacker can distinguish "unknown key" from "wrong secret" by
    response latency. Cheap defence; do it.
  - The auth middleware's path-based bypass list is hard-coded
    (/admin/v1/auth/login, /admin/v1/auth/logout). Adding any future
    anonymous endpoint means updating isAuthBypassPath. Worth
    keeping the list short — most /admin/v1/* MUST require auth.
  - hasPresignedQuery(r) is a fast pre-filter that avoids handing a
    plain anonymous browser request off to the SigV4 middleware
    just to get the same 401 back; it short-circuits to
    writeAuthDenied(ErrMissingSignature) so the JSON error message
    is consistent across cookie + sigv4 absent paths.
  - React state machine pattern: 'unknown' | 'unauthenticated' |
    'authenticated' avoids the "flash of login page" on initial
    load while whoami() is in flight. RequireAuth renders a
    skeleton during the unknown state; this matters when the user
    hard-refreshes a deep-linked page.
  - BrowserRouter basename='/console' means the React Router paths
    are '/login' and '/*' (NOT '/console/login'); the basename
    prefix is added automatically. Don't double-prefix.
  - react-router-dom@6 useNavigate replaces useHistory from v5;
    location.state is plain unknown — cast to your own type when
    reading state.from (see Login.tsx LocationState interface).
  - Bundle gain after US-004: +9.6 KiB gzipped for react-router-dom
    + auth UI. Headroom to 500 KiB ≈400 KiB; recharts 2.x adds the
    most weight downstream (~50 KiB gzipped). Route-level lazy
    loading from US-009 onward keeps the initial paint light.
  - To test the SigV4 fallback in the future, use a real
    Authorization header (the auth middleware does the parsing); for
    the cookie path, mint a JWT directly via signSession() — much
    simpler than constructing SigV4 from scratch.

- **Learnings for future iterations:**
  - React Router v6 layout pattern: <Route element={<Wrapper />}> with
    NO `path` prop wraps its children in <Wrapper>. The wrapper renders
    <Outlet /> in place of the child route's element. AppShell uses
    this — RequireAuth is the auth gate, AppShell is the chrome, child
    routes are pages. Don't put the wrapper inside `element` of every
    page route; the layout-route pattern is cleaner and avoids
    re-mounting AppShell on navigation (which would discard sidebar
    state + cluster-status fetch).
  - useMediaQuery initializer must guard typeof window !== 'undefined'
    even though the app only runs in the browser today — Vite's SSR-ish
    pre-bundling can hit the initializer in a non-window context. Cheap
    insurance.
  - Radix Dialog / Sheet does NOT auto-close on internal navigation
    (Link / NavLink). Pass an explicit onNavigate callback into
    SidebarNav and call setOpen(false) — otherwise the mobile drawer
    stays open after a route change, which feels broken.
  - NavLink className supports both string and ({isActive}) => string;
    using the function form is the standard way to apply active
    styling without writing a useMatch hook.
  - 'NavLink end' prop matters for the index route ('/') — without
    end:true the Overview link stays active on every '/buckets' /
    '/metrics' page because they all start with '/'.
  - localStorage reads in useState initializers must be lazy
    (`useState(readCollapsed)` not `useState(readCollapsed())`) to
    avoid running during SSR if we ever add it. Trivial habit.
  - The sticky sidebar uses `top-0 h-screen` + `sticky` so it follows
    the scroll without layout shift; the top bar uses `sticky top-0`
    + `bg-background/95 backdrop-blur` so content scrolls underneath
    it cleanly. supports-[backdrop-filter] guard skips the blur on
    browsers without it.
  - cluster_name display should fall back to a hardcoded 'Strata'
    when fetch fails — the alternative (blank top bar) looks broken
    on slow links. The actual cluster name swaps in once status
    arrives, no flicker.
  - adminapi.New signature change cascaded into 2 test files. Future
    constructor changes: grep `adminapi.New(` first to find every
    call site; keep the call sites narrow (currently main.go +
    server_test.go + auth_test.go) so the cascade stays small.
