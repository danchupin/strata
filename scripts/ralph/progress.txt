- 4-story cycle closing 1 P2 ROADMAP entry ("Compose default + multi-cluster profiles race for worker leases on shared Cassandra"). - **Root cause** (operator-discovered live): `strata` (default service, knows only `default` RADOS) + `strata-multi` (multi-cluster, profile-gated) on shared Cassandra raced for global `gc-leader-N` / `rebalance-leader-N` / etc leases. - **Fix shape**: drop `strata` (single-cluster) + `strata-features` services. Rename `strata-multi` â†’ `strata` (no profile, always-on, port `9999`, mounts both ceph keyrings, attaches to `default` + `cephb` clusters by default). Single-cluster smoke now via `STRATA_RADOS_CLUSTERS=default:...` env override at runtime, not a separate compose service.
- **Compose semantics reminder**: services with NO `profiles:` key always start; services WITH `profiles: [X]` start ONLY when `--profile X` is passed. There is no fake "default" profile. Don't try to gate `strata` behind a profile + hope bare `up -d` still starts it â€” that pattern is wrong.
