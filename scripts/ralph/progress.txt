- **Learnings for future iterations:**
  - TiKV cluster_state value encoding now 3-segment (`state\x00mode\x00<decimal>`); decode must `bytes.Split` and tolerate 1, 2, OR 3 segments for backward compat. Adding a 4th field repeats the pattern.
  - Cassandra `int` columns scan NULL → 0 cleanly into Go `int`; no need for sql.NullInt32 when the absent column means "default to zero".
  - Boot reconcile lives in `internal/serverapp/cluster_reconcile.go` and is invoked from `serverapp.Run` AFTER drain cache + apiHandler wiring but BEFORE listener start. Fail-soft (logs warn on err) so a transient meta hiccup doesn't block the gateway.
  - Admin handler weight masking: `Weight` on the wire is stored value when state == live, else 0. Stored value still preserves operator intent across drain (drain handler reads currentRow.Weight and passes it through to SetClusterState).
  - `meta.ValidateClusterStateMode` now accepts (pending, "") as a legal pair — pending+mode is rejected via ErrInvalidClusterMode just like (live, "readonly").
