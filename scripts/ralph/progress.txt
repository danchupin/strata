- **Learnings for future iterations:**
  - Audit stamping pattern: every admin handler grabs `owner := auth.FromContext(ctx).Owner` and calls `s3api.SetAuditOverride(ctx, "admin:<Verb>", "<resource>", "-", owner)` BEFORE writing the response. Verb is the Capitalized HTTP action, resource is the operator-facing label (here `gc-config` / `rebalance-config`); bucket field carries `-` for non-bucket resources.
  - Per-package metric counters live in `internal/metrics/metrics.go` and are registered in the single `Register()` call — adding one without appending to the `MustRegister(...)` list silently drops it. The labelset must be deterministic at startup; even labels that "today" are never populated (the gc-config side) should still be reachable so future expansion doesn't change the metric shape.
  - The Heartbeat store interface only exposes `ListNodes(ctx)` — there's no pre-filtered "live nodes" helper. Inline the freshness check (`now.Sub(lastHeartbeat) <= ttl`) with `IsZero()` guard. Default TTL = `2 × HeartbeatInterval` (20 s on default 10 s cadence); fall back to `2 × heartbeat.DefaultInterval` if the server has no HeartbeatInterval configured.
  - `dto.Metric` from `prometheus/client_model/go` is the in-test counter reader: call `counter.Write(&m)` then check `m.Counter.GetValue()`. The existing `readCounter` helper in `clusters_drain_progress_physical_test.go` follows this pattern — name new helpers distinctly (here `readConfigErrorCounter`) to avoid same-package collision.
  - `cmd/strata/workers` and `internal/adminapi` cannot import each other (cmd is the binding layer), but the binding via `serverapp` can mediate by reading from one package and shaping into the other's struct. Keeps the snapshot lock-step with worker constructors without leaking adminapi types into the worker init code.
  - When a handler degrades gracefully (here: ListNodes failure → 200 with replicas_count=0 + counter increment), bake that into a unit test that mocks an `errHeartbeatStore` and asserts both the response shape AND the counter delta — the AC explicitly calls out "endpoint errors counter" so the assertion is load-bearing.
