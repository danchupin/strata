  - **container_name collision gotcha**: the new `strata-cassandra` service can't reuse `container_name: strata-cassandra` (which the OLD cassandra-image service had) — docker would silently accept the YAML but fail at runtime when both services start under `--profile cassandra`. This is the contractual silent no-op behavior the PRD requires.
  - **Validate compose changes early**: `docker compose -f <file> config --services` is a 100ms lint that catches container_name collisions, missing volumes, profile misconfig before any container starts. Run it before any go vet / make build.
  - **Per-replica `STRATA_NODE_ID` + shared `strata-jwt-shared` volume is the round-robin LB contract**: without distinct node IDs the heartbeat rows collide; without the shared volume cross-replica session JWT validation fails (login → next request lands on other replica → 401). The pingcap/tikv image has no stable HTTP healthcheck contract (status server returns plain text on a non-stable contract; the alpine-glibc base ships no curl). The earlier `wait-tikv` TCP probe on 20160 worked but couldn't distinguish "port open, region not assigned yet" from "fully ready" — HTTP probe on PD covers both since PD blocks until raft majority elects.
  - **`make -n <target>` is the contract-level lint** for Makefile changes: dry-runs the target without executing, surfaces unknown targets ("No rule to make target") and substitution errors. Run before any go vet / make build to catch typos in PHONY rewrites.
  - **Orphan files trigger US-005 grep gate failures**: when dropping a Makefile target that references a config file (like lint-nginx-lab-cassandra → strata-lab-cassandra.conf), delete the config file in the same commit.   - **The residue grep gate is the final cycle-closure check, and it catches misses upstream sweeps couldn't anticipate**: US-002/003/004 each swept their own surfaces (Makefile / scripts+CI / docs) but the grep gate found three more silent regressions (`deploy/prometheus/prometheus.yml` had stale `strata-tikv-{a,b}` scrape targets + `--profile lab-tikv` comment; `docs/site/content/architecture/benchmarks/{gc-lifecycle,rebalance}.md` referenced retired container/profile names; `docs/site/content/deploy/{docker-compose,multi-replica}.md` was unmaintained for 2 cycles; `web/e2e/multi-replica.spec.ts` had a NODE_ID_TO_CONTAINER mapping table that re-translated current names back to retired ones). Run a manual `git grep` for the patterns BEFORE the final story to fold the fixes into the right cycle.
  - **Migration notes are legitimate residue exception** — `docs/site/content/architecture/migrations/<name>.md` is operator-facing narrative documenting what was retired. It names retired containers and profiles by design. Adding that path to the smoke's git-pathspec exclusion list (`':!docs/site/content/architecture/migrations'`) is correct; without it the grep gate fails on a doc that's supposed to mention `strata-tikv-c` etc.
  - **`git grep` exit code is 1 on zero matches** (per grep convention), not 0. Capturing matches via `HITS="$(git grep ... || true)"` then checking `-n "$HITS"` is the safe shape — naïve `git grep ... && fail` triggers fail on zero matches (the success state we want). The smoke script's residue gate uses the capture-then-check pattern.
  - **`docker logs --since` accepts a relative duration string like `30s`** — not a Unix timestamp. The smoke captures `BURST_START="$(date +%s)"` before the 20-req burst, then computes `SINCE="$(( NOW - BURST_START + 5 ))s"` (note trailing `s`) so the duration is always slightly larger than the burst window. Without the `+ 5` buffer, fast bursts (<2s) round down to `0s` which means "everything ever" — defeats the bounded-window filter.
  - **Round-robin LB proof via authenticated GETs against `/admin/v1/clusters` rather than `/readyz`**: `/readyz` bypasses the auth middleware AND the access-log middleware in the strata stack. To prove the LB is round-robining + to make per-replica log lines countable, hit an authenticated path. The cookie jar from `login` carries through 20 sequential curl calls; nginx least_conn routes each request to whichever replica has the lower connection count — over 20 requests, expect ~10/10 with worst-case 5/15. Assert `>= 5` per replica.
  - **CLAUDE.md PRD lifecycle rule means deleting `tasks/prd-<name>.md` in the closing commit** — NOT moving to `tasks/archive/`. The Ralph snapshot at `scripts/ralph/archive/<date>-<branch>/` (auto-created by `archive_cycle` in ralph.sh) is the canonical record. `tasks/archive/` is reserved for design intent without a Ralph snapshot (pre-`archive_cycle` legacy or work folded into another cycle).
  - **`docker ps --format '{{.Names}}' | grep -qx '<name>'` is the precise container-presence check** — `-qx` requires whole-line match so `strata-a` doesn't fuzzy-match `strata-a-foo`. Counterintuitively `grep -F` does NOT anchor; use `-x` for line-anchored exact match. Hits this pattern in two places: assert expected containers running + assert retired containers absent.
  - **Scenario opt-in via env (SMOKE_TDL_SCENARIO_C=1)** for scenarios that need a lab-shape mutation: the smoke shouldn't reshape the lab itself (per existing harness convention — operator brings up, smoke probes). Document the env in the script header AND the SKIP message body so an operator running the smoke and seeing "SKIP: C" immediately knows how to opt-in.
  - **Cycle-end discipline pattern**: in the final story, run `git grep` for the cycle's retired-name patterns BEFORE writing the smoke script. The hits inform which files still need updating; they reveal upstream-cycle misses. Treating the grep gate as a write-time gate (run it, fix the matches, then write the smoke that automates the same check) consolidates the sweep with zero loop-back.
