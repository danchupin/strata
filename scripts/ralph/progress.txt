## Codebase Patterns
- Supervisor.LeaderEvents() (cap-8 buffered) emits per-worker (name, acquired) on lease acquire/release; non-blocking send drops on consumer stall. Channel allocated lazily on first call, closed by Run() after wg.Wait().
- Heartbeater.SetLeaderFor(worker, owned) mutates Node.LeaderFor under mu; write() copies slice under same mu before WriteHeartbeat. Idempotent re-acquire / release-of-unowned.
- ROADMAP close-flip: new entry under ## Web UI, format `~~**P3 — title.**~~ — **Done.** ... (commit `<sha>`)`. Use `<pending>` placeholder if SHA needed inline; follow-up commit fills it (per CLAUDE.md).
- compose `depends_on` only resolves services in the active profile. To depend on a profile-gated service from a different profile, list BOTH profiles on the dependency (e.g. `pd` carries `profiles: ["tikv", "lab-tikv"]`). Otherwise `compose config -q` rejects with "depends on undefined service".
- nginx `nginx -t` resolves upstream hostnames at parse time; if the upstream block names docker service hosts (e.g. `strata-tikv-a`), the syntax-test container needs `--add-host=<name>:127.0.0.1` to resolve. Real DNS happens via Docker embedded resolver at runtime.
- File-based atomic-bootstrap pattern (POSIX O_EXCL): `os.OpenFile(path, O_WRONLY|O_CREATE|O_EXCL, 0600)`. Writer-side success → write → close → return. Loser EEXIST → re-read up to 3× with 100 ms backoff to absorb fsync race. No lock-manager / no leader.Session needed when the bootstrap precedes locker construction. Used by `serverapp.bootstrapSharedJWTSecret` for /etc/strata/jwt-shared/secret.
- Heartbeat row eviction asymmetry: cassandra applies `USING TTL 30` (row gone server-side); tikv stamps `ExpiresAt` in the JSON payload and ListNodes filters `now > ExpiresAt` at read time; memory backend GCs lazily on read AND write. All three converge on "row vanishes within 30s of the writer dying", so post-stop assertions like `count(status==healthy)==1` hold uniformly. Lease TTL in `internal/leader.Session` defaults to 30s, RenewPeriod TTL/3 — chip rotation lands within ~30-35s after holder kill.
- `STRATA_STATIC_CREDENTIALS` is comma-separated `accesskey:secret[:owner]` entries. First entry is the conventional admin/login key in dev-stack scripts. Parse with `${VAL%%,*}` then `${ENTRY%%:*}` / `${ENTRY#*:%%:*}` — no awk/cut dependency.
- Admin session cookie: `strata_session`, path=/admin, signed with the JWT secret. In a multi-replica deployment the secret is shared via the strata-jwt-shared volume (US-004), so a cookie issued by replica A is accepted by replica B / the LB.
- Playwright multi-config split: default `playwright.config.ts` carries one inline webServer (memory-mode gateway) for `e2e-ui`. Specs that need an external (compose-managed) stack live behind a separate `playwright.<name>.config.ts` with NO webServer + a project-scoped testMatch; default config testIgnore-s the spec filename so the two never overlap. Run via `pnpm exec playwright test --config=playwright.<name>.config.ts`. Same pattern works for any future "needs real docker" spec.
- CI gating with PR-title token: `if: contains(github.event.pull_request.title, '[multi-replica]') || github.event_name == 'workflow_dispatch'` — keep the default e2e fast budget intact while opt-in PRs (or manual workflow_dispatch from the Actions tab) exercise the heavier scenario. Always pair with `workflow_dispatch:` under top-level `on:` so the manual trigger is reachable.

- **Learnings for future iterations:**
  - POSIX O_EXCL at file-open is the cheapest "first writer wins" primitive that doesn't need a lock manager. Works across processes / containers as long as both share the underlying filesystem (named docker volume satisfies that).
  - Don't try to be clever with tmp-file + rename here: PRD prescribed O_EXCL on the actual path because the loser must see the same path EEXIST and read THAT path. Tmp+rename hides EEXIST under "rename-replaces-existing" semantics on POSIX, defeating the race detection.
  - The 100ms × 3 retry isn't paranoia — write→close happens after O_EXCL create, so a concurrent reader can see an empty/short file in the kernel page cache between the writer's `OpenFile` return and `Close`. The re-read loop converges as soon as the writer finishes (typically <1 ms; 300 ms cap is generous).
  - readJWTSecretFile already handles short / empty files by returning ok=false. Reusing it for both fast-path and post-race read keeps the contract uniform (>= 16 byte length minimum).
  - Splitting loadJWTSecret into loadJWTSecret (env-reading) + loadJWTSecretFrom (pure) made the env-not-touching-file invariant trivially testable. The same pattern should replace any "env + filesystem + fallback" function whose unit tests want to assert side-effect absence.
