- **Learnings for future iterations:**
  - **container_name collision gotcha**: the new `strata-cassandra` service can't reuse `container_name: strata-cassandra` (which the OLD cassandra-image service had) — docker would silently accept the YAML but fail at runtime when both services start under `--profile cassandra`. Flipped cassandra container_name to `strata-cassandra-db` to make room.
  - **Multi-cluster mount path convention**: when adding both clusters, mount each at `/etc/ceph-<id>` (NOT `/etc/ceph`). Gateway reads `STRATA_RADOS_CLUSTERS=default:/etc/ceph-a/...,cephb:/etc/ceph-b/...` so the paths must match exactly. Retired canonical `strata` service had a triple-mount including the legacy `:/etc/ceph:ro` bind for backwards-compat; the new strata-a/b drop it — gateway never reads `/etc/ceph` when STRATA_RADOS_CLUSTERS is set.
  - **`docker compose config` accepts unknown profile flags silently**: passing `--profile retired-name` against a compose file that doesn't reference it produces no warning and no error — the resolved service list is identical to bare-default. This is the contractual silent no-op behavior the PRD requires.
  - **Validate compose changes early**: `docker compose -f <file> config --services` is a 100ms lint that catches container_name collisions, missing volumes, profile misconfig before any container starts. Run it before any go vet / make build.
  - **Per-replica `STRATA_NODE_ID` + shared `strata-jwt-shared` volume is the round-robin LB contract**: without distinct node IDs the heartbeat rows collide; without the shared volume cross-replica session JWT validation fails (login → next request lands on other replica → 401). Both are required; don't simplify one away.

- **Learnings for future iterations:**
  - **`docker compose ps <service>` silently filters profile-gated services**: without the matching `--profile X` flag, the output is empty even if the container is running. The `until [ "$(...)" = "healthy" ]` loop in wait-cassandra hangs forever without the flag. Always add `--profile cassandra` to wait-cassandra (and any future wait targets for profile-gated services). Pattern: `$(COMPOSE) --profile <name> ps --format '{{.Health}}' <service>`.
  - **PD's `/pd/api/v1/health` is the portable HTTP readiness probe** for the TiKV stack. The pingcap/tikv image has no stable HTTP healthcheck contract (status server returns plain text on a non-stable contract; the alpine-glibc base ships no curl). PD's API is stable + curl-friendly from the host, so probe pd:2379 (host-published port) instead of tikv:20160. The earlier `wait-tikv` TCP probe on 20160 worked but couldn't distinguish "port open, region not assigned yet" from "fully ready" — HTTP probe on PD covers both since PD blocks until raft majority elects.
  - **`make -n <target>` is the contract-level lint** for Makefile changes: dry-runs the target without executing, surfaces unknown targets ("No rule to make target") and substitution errors. Run before any go vet / make build to catch typos in PHONY rewrites.
  - **Orphan files trigger US-005 grep gate failures**: when dropping a Makefile target that references a config file (like lint-nginx-lab-cassandra → strata-lab-cassandra.conf), delete the config file in the same commit. Otherwise the file lingers with `strata-cass-` / `strata-lb-nginx-cass` strings and fails the US-005 final grep gate. Same principle applies to retired scripts/CI workflows in US-003.
  - **build-ceph + run-strata still reference the retired `strata` service name** in the existing Makefile — these compose-bound targets MUST be swept whenever a service is renamed. Patterns to grep for when renaming compose services: `$(COMPOSE) up -d <name>`, `$(COMPOSE) build <name>`, `$(COMPOSE) ps <name>`. Easy to miss because make doesn't validate compose service names statically.
