## Codebase Patterns
- RADOS data-plane primitives live in TWO places: always-on shared shape +
  sentinel in main module `internal/data/rados/` (no go-ceph, builds CGO-free),
  real librados impl in the separate `cephimpl/` Go module. Mirror `rados.New`:
  a main-module dispatcher type-asserts `data.Backend` to a capability interface
  and returns `data.ErrRADOSNotCompiled` when it's not the librados backend.
  Keep reusable helpers (rate limiters, parsers) in the main module so cephimpl
  needs no extra deps. Hermetic gate: `GOWORK=off go mod graph | grep -c go-ceph`
  must stay 0.
- Local macOS CANNOT compile cephimpl (cgo + Xcode license + gosigar darwin).
  Verify main-module logic with `CGO_ENABLED=0 GOWORK=off go test ./internal/data/rados/`;
  cephimpl integration legs are CI-only (`make test-rados`). CI (Linux) = truth.
- Chunk OID format is `fmt.Sprintf("%s.%05d", uuid.NewString(), idx)` →
  `<uuid>.<>=5 digits>`. `rados.ParseChunkOID`/`IsChunkOID` is the inverse filter.
- Per-PUT data-tier dimensions ride the data-plane ctx, NEVER the
  `data.Backend.PutChunks(ctx, io.Reader, class)` signature (carries no
  bucket/key/version/mtime). Add a `With*/...FromContext` pair in
  `internal/data/context.go`, read it in the backend `PutChunks`; the gateway
  stamps the ctx via the `dataCtxForPut*` helpers.
- To make a chunk and its meta row agree on `version_id`, the HANDLER must
  pre-decide it (`meta.NewVersionID()`, only for `VersioningEnabled`;
  Disabled/Suspended → `meta.NullVersionID`) and pre-set `Object.VersionID` +
  `Object.Mtime` BEFORE PutChunks — both backends honour a pre-set non-empty
  VersionID. Otherwise version_id is minted by PutObject AFTER PutChunks.
