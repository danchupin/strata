## Codebase Patterns
- **CI verification is CI-bound, not local**: most failures in this cycle (promtool fetch, gosec gate, racecheck flake, compose e2e) do NOT reproduce on a local dev box. Each story's definitive check is a green job on a CI run on the cycle branch (`gh run list --branch=ralph/ci-green-speedup`; `gh run view <id> --json jobs`). Record the run id + per-job conclusion in the progress note. Do NOT mark a story passing on a local pass alone when its AC names a CI job.
- **main is branch-protected**: PR-only, `enforce_admins=true`, 7 required checks (Lint nginx, Web Vite, Docker build, govulncheck×2, license-audit×2). The cycle PR must pass those to merge. US-009 expands the required set only after the new checks are observed green.
- **Auth is secure-by-default now** (`STRATA_AUTH_MODE` default `required`; `off` = dev-only full-access). Any smoke/e2e path that starts a gateway must set `STRATA_AUTH_MODE` explicitly.
- **gosec lockstep**: `.gosec.yml` + `scripts/check-gosec.sh` (GOSEC_EXCLUDE_RULES / EXCLUDE_DIRS) + the `ci.yml` gosec `-exclude=` invocation must stay in lockstep when changing rules/scope.
- **Docker cache**: GitHub Actions Cache `type=gha` is ~10 GB/repo, LRU; use distinct `scope=` per image so the large ceph layers don't evict the bootstrap/Go caches.
