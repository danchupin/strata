- **Learnings for future iterations:**
  - **bash namerefs (`local -n`)** enabling deduplication of the target → (endpoint, ak, sk) case statement that was inlined in workload_put / cleanup_workload / assert_bucket_absent (3 copies). Pattern: `resolve_target "$target" endpoint creds_ak creds_sk` — caller declares the 3 locals before the call. Cleaner than reassigning globals.
  - **warp `get` subcommand emits TWO Report: blocks** — `Report: PUT.` for prepare (seed) phase + `Report: GET.` for drain (measure) phase. Existing `parse_warp_summary` op-anchored awk regex (`^Report: $op\\.`) correctly picks the GET block. The PUT prepare block must NOT be confused with the actual put-{small,medium} workloads.
  - **`--objects=N` for warp get** controls seed object count; `--obj.size` controls each seed object's size. Default 2500 for get-medium (1MiB) = 2.5GB on-disk seed; default 10000 for get-small (1KiB) = 10MB. Choose seed high enough that conc=128 GETs don't all hit the same few cached objects.
  - **Sweep cleanup discipline**: keep one bucket per workload across all conc points; drop bucket once at end (`workload_*_sweep`). Between conc points there's no cleanup beat (warp put generates fresh random keys, warp get reuses --noclear seed). End-of-workload bucket-drop becomes the only cleanup point — for a put-small full sweep at conc=128 × 60s × 3 runs that's ~1M objects to delete via `aws s3 rm --recursive`. Acceptable as a known tax at workload boundaries; US-012 captures actual full-sweep duration.
  - **Mutual-exclusion pattern for flags**: `[[ -n "$A" && -n "$B" ]] && abort "..."` is the simplest way to enforce flag mutex. Surfacing the abort early in the CLI parser path keeps the rest of the dispatch logic clean.
  - **jq pivot trick**: jq `stats($a)` reduction inside `map(...)` produces structured per-target rollups; concatenate strata + rgw branches with `(map(select(.target=="strata") | .p99_ms))` etc. Output the row as a pipe-separated string in jq then `IFS='|' read -r ...` in bash — cleaner than 4 separate jq invocations per row.
  - **Error propagation for stddev** on a ratio Strata/RGW is approximated by sqrt(strata_sd² + rgw_sd²); not strictly correct for a ratio (correct uses ratio × sqrt((strata_sd/strata_mean)² + (rgw_sd/rgw_mean)²)) but the additive-variance form is simpler and the difference is small at typical noise levels. Documented in --report header line. Refine in US-010 doc page if ratio stddev becomes a load-bearing claim.
  - **Strata lab warm-up effect** observed: first run after `make up-all` hits p99 ≈ 200ms+ at conc=4 even for 1KiB; subsequent runs settle. Real bench (60s × 3 runs) absorbs warm-up in the first run; mean ± stddev across 3 runs is the right shape. For 5s smoke runs, warm-up dominates — values like 213ms@c=4 are normal smoke artifacts, not representative of steady-state.
