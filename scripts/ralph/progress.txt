## Codebase Patterns
- Crypto packages live under `internal/crypto/<area>/`. SSE-S3 master key code is in `internal/crypto/master/`.
- Provider interfaces return `(key, keyID, error)`. Callers (gateway main) own fatal-log on error — provider packages stay log-free so they're testable and reusable.
- Sentinel errors (e.g. `master.ErrNoConfig`, `master.ErrInvalidKeyLength`) are exported from the package so callers and tests can `errors.Is` them.
- Use `t.Setenv` / `t.TempDir` for env-var and file-backed tests. For mtime-based hot-reload tests, set the cached mtime to the past via `os.Chtimes` before rewriting so the rewrite is observable on coarse-mtime filesystems.

- **Learnings for future iterations:**
  - PRD's "structured-log error" requirement is satisfied at the call site, not inside the provider. Keep the package log-free; gateway main (US-008/US-014 scope) will adopt slog and fatal on master.FromEnv error.
  - File mtime granularity is coarse on some filesystems (1s on ext4). Forcing the cached mtime into the past with `os.Chtimes` before rewriting is the reliable way to test hot-reload without flake.
  - `STRATA_SSE_MASTER_KEY_ID` is shared by env + file providers — file provider falls back to `file-1` when unset. Future stories that introduce per-provider key IDs (Vault, US-002) will need to thread keyID source through, so revisit if collisions arise.
  - `FromEnv` prefers `STRATA_SSE_MASTER_KEY_FILE` over `STRATA_SSE_MASTER_KEY` when both are set. Document this if operator-facing docs land later.
  - This story does NOT wire the provider into the gateway. US-004 (PutObject/GetObject) is the wiring point; until then `cmd/strata-gateway/main.go` is unchanged.
