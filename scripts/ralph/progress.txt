- **Learnings for future iterations:**
  - `docker compose --profile <name> config --services` surfaces every service active under the profile + the baseline set — useful for validating that a profile-gated addition didn't accidentally trip into the default service list.

The hook expansion uses `eval` so `$SHARDS` is interpolated at run time; the hook string itself contains an escaped `\$SHARDS` so the deferred expansion fires.
