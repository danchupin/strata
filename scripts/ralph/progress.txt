### Learnings for future iterations
- **Config substruct pattern**: every new env knob lives on a substruct under `Config`, with a koanf tag, a default in `config.defaults()`, an envMap entry, validation/clamp in `config.validate()`, a row in `deploy/strata.toml.example`, AND a row in `docs/site/content/reference/env-vars.md`. The drift-lint test `internal/config/env_toml_parity_test.go` catches misses on any of the four — running it post-change is faster than searching for missed sites.
- **Zero-as-disabled is a Go HTTP idiom**: `net/http.Server` treats zero-valued `ReadTimeout`/`WriteTimeout`/`IdleTimeout`/`ReadHeaderTimeout`/`MaxHeaderBytes` as "no timeout / default". US-001 mirrors that semantic for the operator UX (set `0s` → opt out) without inventing a sentinel value.
- **Negative duration parsing**: `time.ParseDuration("-1s")` succeeds with a negative `time.Duration`. Validation MUST explicitly reject negatives because koanf will accept them silently. Mirror this for every new duration env added in later stories.
- **Test pattern for env-driven config**: use `t.Setenv("STRATA_*", "...")` then call `config.Load()`. `t.Setenv` auto-cleans at test end so the helper composes cleanly with subtests. `STRATA_HTTP_WRITE_TIMEOUT=0s` round-trips through koanf as `time.Duration(0)` — no special-casing needed for the "explicit zero" path.
- **Slowloris chaos test shape**: spawn a real `http.Server` via `net.Listen("tcp", "127.0.0.1:0")` + `srv.Serve(ln)`, dial raw with `net.Dial`, dribble bytes, then `io.ReadAll` with `SetReadDeadline`. `io.ReadAll` returns `nil` on EOF (Go convention), surfaces RST as a non-timeout error, and only returns a `net.Error.Timeout()=true` when the server kept the socket open past the slack window. The deadline branch is the only failure mode.
- **`internal/data/rados/cephimpl/backend.go` build-tag diagnostic** (`OperationNoFlag` type mismatch) is unrelated to the default build — it surfaces only under `-tags ceph`. Default `go vet ./...` and `go build ./...` ignore it; CI's e2e job catches the tagged build separately.
- **ROADMAP "Cycle A" entry pattern**: NEW open entry inserted at the top of `## Correctness & consistency` in US-001; US-010 flips it to `~~Done~~` shape with the closing-SHA backfill on a follow-up commit. Mirrors the 12-prior-cycle close protocol — do NOT skip the open entry.

### Learnings for future iterations
- **Go's http2 ConfigureServer cipher gotcha**: when `srv.TLSConfig.CipherSuites` is non-empty, `golang.org/x/net/http2.ConfigureServer` (called automatically by `ListenAndServeTLS`) returns `"http2: TLSConfig.CipherSuites is missing an HTTP/2-required AES_128_GCM_SHA256 cipher"` unless at least one of `TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256` / `TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256` is present. The check is naive — it does NOT recognize TLS 1.3 ciphersuites (TLS_AES_128_GCM_SHA256 etc) as satisfying RFC 7540's intent. Workaround: always include the two TLS 1.2 ECDHE-AES-128-GCM ciphers in any non-nil CipherSuites list, regardless of TLS 1.3-only intent. Alternative: set CipherSuites=nil and rely on Go's defaults.
- **Cert chain via `srv.TLSConfig.Certificates`** is the modern path — call `srv.ListenAndServeTLS("", "")` and Go reads from `Certificates`/`GetCertificate`. The cert/key file path args are only consulted when both fields are nil. This sets US-003 up cleanly (SNI multi-cert dispatch via `GetCertificate` swap).
- **Mozilla SSL Config profiles → Go cipher list mapping**: Modern is "TLS 1.3 only" per Mozilla's actual recipe (Mozilla doesn't pin Go-specific TLS 1.3 cipher IDs — those are always all 3 by RFC 8446). Intermediate is 6 TLS 1.2 ECDHE AEAD ciphers (GCM-128, GCM-256, CHACHA20, each × ECDSA/RSA). Go's `tls` package treats TLS 1.3 cipher IDs as constants that can sit in `CipherSuites` but are never actually negotiated — they're ignored at handshake time per RFC 8446 ("TLS 1.3 ciphersuites are not configurable").
- **`openssl req` ec keygen on macOS**: prefer `openssl req -x509 -newkey ec -pkeyopt ec_paramgen_curve:prime256v1 …` over the bash process-substitution form `-newkey ec:<(openssl ecparam -name prime256v1)`. The substitution form hangs silently on LibreSSL (macOS default). The `-pkeyopt` form works on both LibreSSL + OpenSSL 3.
- **`openssl s_client` is too interactive for unattended smoke** even with `echo Q | openssl s_client …` — LibreSSL on macOS hangs differently. Prefer `curl -sv --cacert <ca> https://host:port/path 2>&1` and grep for `SSL connection using TLSv1\.[23]` / `SSL certificate verify ok`. Bonus: curl also tests `--http2` ALPN negotiation in the same invocation.
- **HTTPS test pattern via `httptest`**: `httptest.NewUnstartedServer(mux)` → `ts.TLS = tlsCfg` → `ts.StartTLS()` lets you inject the exact `tls.Config` you'd build at production boot. The cert pool for the test client comes from reading the same cert PEM file written by `writeSelfSignedCert` helper. Round-trip latency is < 5 ms — fits unit-test budgets.
- **Self-signed cert generation in Go tests**: `ecdsa.GenerateKey(elliptic.P256(), rand.Reader)` + `x509.CreateCertificate` with SerialNumber/Subject/Validity. For 127.0.0.1 / IP-based SAN, set `tmpl.IPAddresses = []net.IP{ip}` (NOT `DNSNames`). `tmpl.IsCA = true` + `BasicConstraintsValid = true` makes the cert self-signed-CA so it can sign itself. PEM-encode the DER bytes with `pem.EncodeToMemory(&pem.Block{Type: "CERTIFICATE", Bytes: der})`.
