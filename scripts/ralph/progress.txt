## Codebase Patterns

(seed entries for this cycle. Replace/remove as US-001..US-005 land.)

- internal/auth/streaming.go (the existing streaming decoder) decodes
  chunk framing via bufio.Reader: ReadString('\n') for size+optional
  ;chunk-signature header, ParseInt(size, 16, 64) for hex length, then
  forwards `size` bytes verbatim to the consumer. Today it does NOT
  recompute the chain — US-002 changes the read loop to a
  buffer-then-validate shape (read full chunk → SHA-256 → expected sig
  via chunkSigner.Next → hmac.Equal → forward on match, error on
  mismatch).
- internal/auth/sigv4.go::deriveSigningKey(secret, date, region,
  service) returns the 32-byte SigV4 derived signing key — the same
  key the outer canonical-request hash used. US-001's chunkSigner
  consumes this key directly. computeSignature is the reference for
  HMAC-SHA256(key, string-to-sign) shape.
- internal/auth/middleware.go:76-77 wraps r.Body with
  newStreamingReader when r.Header.Get('x-amz-content-sha256') ==
  streamingBody constant. US-002 extends this wire point to also pass
  the seed-sig + signing-key + scope into the decoder so the chain
  validator has everything it needs.
- AuthInfo struct in internal/auth/types.go currently exposes
  identity-only. US-002 adds either an unexported field (cycle of
  staleness shorter) or a context.WithValue carrier for the seed
  signature + signing key + scope. Pick one; document in a one-line
  comment so future readers see the choice.
- hmac.Equal([]byte, []byte) is the Go stdlib idiom for constant-time
  authentication-tag comparison. Use it for chunk-signature compare
  (US-001 + US-002). Plain == on hex strings is a timing footgun.
- Streaming-shape rule: peak buffer is ONE chunk (≤16 MiB cap), never
  the whole body. For typical SDK output the per-chunk buffer is
  ≤8 KiB. The "no full-body buffer" invariant is preserved.
- aws-chunked-trailer (US-003) is detected via x-amz-trailer request
  header. ROADMAP "Known latent bugs" already documents the older
  decoder breaking on trailer-format with a confusing error; US-003
  promotes that to a clean 501 NotImplemented response. This is
  detection-only; real trailer-format support is a separate future
  PRD.
- ROADMAP maintenance rule (CLAUDE.md): every commit closing a
  ROADMAP item flips the bullet to ~~strikethrough~~ Done format in
  the same commit, with closing SHA inline (or `(commit pending)`
  with a follow-up SHA edit). Every commit discovering a new gap adds
  a fresh entry under the right severity section in the same commit.
- s3-tests baseline lives in scripts/s3-tests/README.md as an
  append-only history. US-001 captures the cycle-start headline in
  its commit message footer; US-005 re-runs scripts/s3-tests/run.sh
  and asserts the new headline >= the captured baseline. Chunk
  validation MUST NOT regress upload paths — a regression blocks
  cycle close.
- AWS test vectors for the chain are transcribed verbatim into Go
  literals in internal/auth/streaming_test.go. Inline copy so the
  test stays offline-runnable across AWS doc URL rotations. Source:
  https://docs.aws.amazon.com/AmazonS3/latest/API/sigv4-streaming.html
  "Example Calculations".
- No metadata schema changes; no Cassandra/TiKV/memory backend
  touches; no RADOS/S3-backend touches. Pure auth-layer fix scoped
  to internal/auth/* + internal/s3api/sigv4_streaming_test.go +
  ROADMAP.md + docs/security/.
