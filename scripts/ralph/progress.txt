## Codebase Patterns
- Hugo site lives under `docs/site/`. Theme is `hugo-book` as a git submodule at `docs/site/themes/hugo-book/`. Fresh checkouts need `git submodule update --init --recursive`.
- Hugo config: `docs/site/hugo.toml`. `BookSection = '*'` so every top-level content section appears in the sidebar (we don't nest under `content/docs/`).
- Build artifacts (`docs/site/public/`, `docs/site/resources/`, `.hugo_build.lock`) are gitignored. Drop them via `make clean`.
- Makefile targets: `make docs-serve` (dev preview on :1313), `make docs-build` (minified bundle). Both `cd` into `docs/site` first.
- Hugo CLI on Homebrew is hugo-extended (>= 0.128 needed for hugo-book). The future GH Action workflow (US-011) must pin a hugo-extended version.
- **Canonical doc paths** are now under `docs/site/content/<section>/<page>.md`. Content tree mirrors CockroachDB shape: `get-started/`, `deploy/`, `architecture/{backends,benchmarks,migrations}/`, `best-practices/`, `s3-compatibility/`, `reference/`, `developers/`. Tooling refs (CLAUDE.md / ROADMAP.md / code comments) point at the canonical-source path; user-facing copy in README/ROADMAP may use the published-URL form (US-010 finalises that).
- **Cross-link inside Hugo content**: use `{{< ref "/section/page" >}}` shortcode. Hugo errors on missing refs at build time — that's the safety net.
- **Inline-code escape gotcha**: backticked text containing `{{< ... >}}` is still parsed by Hugo. Escape with the comment form `{{</* ... */>}}` to render the literal token (e.g. when documenting the shortcode itself in `developers/_index.md`).
- **Theme-agnostic styling**: the hugo-book theme exposes CSS custom properties `--body-background`, `--body-font-color`, `--color-link`, `--gray-100/200/500` (see `themes/hugo-book/assets/plugins/_themes.scss`). Custom landing/marketing markup should reference those vars so it adapts to every theme variant (light, dark, catppuccin, etc.) without duplicating palettes.
- **Hero/CTA pattern for content pages**: `content/_index.md` uses `unsafe = true` markdown + inline `<style>` + raw `<div>` blocks for the hero + feature grid. Use `{{< relref "/section/" >}}` (NOT `{{< ref >}}`) inside `href="..."` so the rendered URL is properly prefixed by the site `baseURL` subpath (`/strata/...`). `relref` renders a relative link; `ref` renders absolute — the latter breaks under GitHub Pages subpath hosting.
