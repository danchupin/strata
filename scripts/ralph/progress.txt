## Codebase Patterns
- /admin/v1/* now sits behind `s3api.NewAuditMiddleware`. Admin handlers stamp the operator-meaningful audit row via `s3api.SetAuditOverride(ctx, action, resource, bucket, principal)` — bypassing the path-derived `PostBucket` default. Action="" means fall back to default.
- New admin handler? Wire `mux.HandleFunc("METHOD /admin/v1/...", s.handleX)` in `internal/adminapi/server.go::routes()`. Handlers go in `internal/adminapi/<topic>.go`. JSON helpers `writeJSON` / `writeJSONError` are in server.go.
- Admin auth: `auth.FromContext(ctx).Owner` is set by `adminapi.authMiddleware` via session-cookie `Lookup` and is the value to use as the `owner` argument to `meta.Store.CreateBucket`.
- Bucket-name validation: call `s3api.ValidBucketName(name)` (public alias of `validBucketName` in internal/s3api/validate.go); rules include reserved gateway-internal route names (`console`, `admin`, etc.) — duplicate the same regex client-side in dialogs for live inline feedback.
- Versioning enum: `meta.VersioningDisabled` ("Disabled") — the freshly-created default; `meta.VersioningEnabled`; `meta.VersioningSuspended`. `versioningLabel()` in adminapi/buckets.go maps Disabled→"Off" for the operator-facing label. Public dialog accepts only Enabled|Suspended.
- Object-Lock requires `meta.SetBucketObjectLockEnabled` (separate per-bucket flag). Server-side guard: reject `object_lock_enabled=true` unless versioning=Enabled.
- TanStack Query invalidation pattern after a write: `queryClient.invalidateQueries({ queryKey: ['buckets', 'list'] })` — partial key prefix matches all (sort/order/page/pageSize) variants.
- shadcn Dialog primitives in web/src/components/ui/dialog.tsx — wrap content in `<Dialog open onOpenChange><DialogContent><DialogHeader/><form/><DialogFooter/></DialogContent></Dialog>`.
- Toasts: `import { showToast } from '@/lib/toast-store'` — fire-and-forget; ToastProvider already mounted in App.
- adminapi.Server now carries a `leader.Locker` (Config.Locker). serverapp.buildLocker is the single source — call it once and pass the result both to adminapi.Config.Locker and the workers Supervisor (see `internal/serverapp/serverapp.go::Run`). The locker is non-nil for cassandra/tikv/memory backends; admin handlers that take long actions (force-empty, future ones) MUST acquire `bucket-<kind>:<bucket>` against it before kicking a goroutine and release on exit.
- Admin background jobs persist in the new `meta.AdminJob` shape (kind/state/deleted/started_at/updated_at/finished_at). Cassandra: `admin_jobs` table (added in tableDDL); memory: in-process map; tikv: `s/Aj/<id>` row. Pattern: handler creates job in `pending`, kicks goroutine which flips to `running`, drives work, terminal flip to `done|error` + sets FinishedAt. Always pass a *copy* of the job pointer to the goroutine — handler still uses the original to write the 202 response and a shared mutable AdminJob is a race.
- Reusable shared error shape on the React side: `AdminApiError` in `web/src/api/client.ts`. Wrapper helper `buildAdminError(resp, fallback)` parses `{code, message}` + status. Reuse instead of inventing one error type per endpoint.

### Learnings for future iterations
- Patterns discovered:
  - Override-pointer-in-context is the cleanest way to let downstream handlers feed metadata back to outer middleware. The outer puts a `*T` into ctx before Next; the inner mutates the same struct; outer reads after Next returns. Avoids needing trailers / response-writer shims.
  - The admin audit story will repeat across US-002..US-022 — every admin write should stamp `s3api.SetAuditOverride(ctx, "admin:<Verb>", "<resource-shape>", bucket, owner)`. Prefix = "admin:". Bucket-scoped ⇒ resource="bucket:<name>"; object-scoped ⇒ "object:<bucket>/<key>"; iam-scoped ⇒ "iam:<UserName>" or "iam-policy:<arn>".
  - Frontend-side bucket-name validation must mirror the reserved-name set in internal/s3api/validate.go::reservedBucketNames; if the gateway adds another reserved prefix, update the dialog regex too. Helper `nameViolation()` in CreateBucketDialog.tsx is the place.
- Gotchas:
  - `errorResponse` is unexported in internal/adminapi/server.go — tests in the same package can decode it directly; from outside packages you need to define your own. Keep adminapi tests in-package.
  - When wrapping /admin/v1/ with audit middleware, the middleware stamps the override pointer onto the inner ctx. Admin handlers must call `s3api.SetAuditOverride` on `r.Context()` *inside* the handler — not before returning the response — because the middleware reads the override after Next.ServeHTTP returns.
  - shadcn `<Dialog onOpenChange>` fires both on ESC and on the X button — so a single state setter is enough; no separate close handler needed.
  - `auth.WithAuth(ctx, &AuthInfo{Owner: ...})` is how unit tests can inject an authenticated identity when bypassing authMiddleware via `s.routes()` directly.

### Learnings for future iterations
- Patterns discovered:
  - Long-running admin background jobs follow a fixed shape: meta.AdminJob row + worker_locks lease + goroutine that flips state pending→running→done|error. Reuse `internal/adminapi/buckets_delete.go::startForceEmptyJob` + `runForceEmptyJob` as the template — kind constants live in meta/store.go (`AdminJobKind*`, `AdminJobState*`).
  - When a handler kicks a goroutine that mutates the same struct it then writes to the response, the goroutine MUST receive a *copy* of the pointer (or the handler must build the response *before* spawning). Race-detector will catch the latter eventually but it's faster to be defensive at the boundary.
  - The leader.Locker is a fully working primitive for ad-hoc admin-handler leases — not just worker leader-elections. Acquire with TTL ~10 minutes (force-empty default), release in the goroutine's defer. The same pattern will fit "audit-export run-now", "manifest-rewriter run-now", etc. in later phases.
  - shadcn doesn't ship a Progress component out of the box. A two-div CSS bar (`<div className="h-2 ... bg-muted"><div style={{width: <pct>%}}/></div>`) is enough for slow-poll progress UI; pulling Radix/Progress for a single use-site is not warranted.
- Gotchas:
  - The TiKV pessimistic-txn `LockKeys` lease is held for the txn lifetime; CAS-style early returns (e.g. CreateAdminJob seeing the key already exists) MUST `txn.Rollback()` explicitly before returning the non-error result, otherwise the mock memBackend used by unit tests deadlocks the next caller.
  - Memory `Locker` is in-process. Tests that simulate "another node holds the lease" must pre-`Acquire` with a distinct holder string and Cleanup-Release; otherwise the handler's own subsequent acquire would simply succeed (single-locker, single-process semantics).
  - Cassandra `admin_jobs` schema is new — existing keyspaces upgrade additively via the idempotent CREATE TABLE IF NOT EXISTS in `tableDDL`. No `alterStatements` row needed because the table starts wide.
- Useful context:
  - `internal/serverapp/serverapp.go::buildLocker` returns `nil` if the meta backend exposes no locker. Admin handlers that depend on the lease must `if s.Locker == nil { 503 }` so dev rigs without the locker fail loud rather than dropping the lease silently.
  - The drain loop calls ListObjectVersions first (so versioned buckets surrender every version cleanly) and falls back to ListObjects when the bucket has no version rows. memory.Store mirrors both surfaces; cassandra/tikv use the same code path.
  - `AdminApiError` + `buildAdminError` in `web/src/api/client.ts` is the canonical error shape for admin API wrappers — replace the bespoke `CreateBucketError` (US-001) with the shared shape on the next dialog touch-up; new wrappers (US-003+) should use `AdminApiError` from day one.
