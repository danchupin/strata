### Browser EventSource pattern (NEW for Phase 3)
- `EventSource` is browser-native — no third-party lib needed. New helper hook `web/src/hooks/useEventSource.ts` wraps reconnect with exponential backoff (1s → 30s cap). Sends new query string on filter change → close + reopen.
- Filter applied SERVER-SIDE — the SSE handler reads action/principal/bucket query params and the broadcaster filters per subscriber. Wire-payload stays small.
- `:keep-alive\n\n` ping every 25s defeats idle-connection proxies (nginx, AWS ALB).

### Cassandra/meta gotchas (still apply)
- Both `internal/meta/memory` and `internal/meta/cassandra` and `internal/meta/tikv` implement `meta.Store` in lockstep. Adding `ListSlowQueries` (US-003) means updating all three + the storetest contract (`internal/meta/storetest/contract.go`).
- LWT (`IF EXISTS` / `IF NOT EXISTS`) required for read-after-write coherence on the same row. Phase 3 is read-only so this rarely bites here, but US-009's observer-label change is metric-emission only — no schema migration.
- Schema migrations are additive — `internal/meta/cassandra/schema.go::tableDDL` (idempotent CREATE) + `alterStatements` (idempotent ADD COLUMN). No destructive migration ever.
