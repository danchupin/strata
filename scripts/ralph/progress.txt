- **Learnings for future iterations:**
  - **warp v(dev) 2026-05-21 quirks**: (a) `--analyze.out=FILE` is a no-op — file never written; the text summary is on stdout only. Script captures stdout to summary_file. (b) `--benchdata=PATH` appends `.json.zst` suffix; actual file is `PATH.json.zst`. (c) Stdout block shape: `Report: PUT. Concurrency: N. Ran: Xs` + ` * Average: A MiB/s, B obj/s` + ` * Reqs: Avg: ms, 50%: ms, 90%: ms, 99%: ms, Fastest: ms, Slowest: ms, StdDev: ms` (NOT `Operation: PUT` + `99th percentile` like older warp docs suggest). Parser anchors on `^Report: <op>.` + extracts `50%:` / `99%:` directly. p95 is linear interpolant between p50 and p99 since warp's stdout omits it.
  - **`set -euo pipefail` + `grep -m1 -oE | awk` inside `< <(parse_warp_summary ...)` is a footgun**: missing matches (e.g. warp omits `Errors:` line when zero errors) make `grep` return 1, pipefail kills the parse subshell, read sees nothing, emit_row writes empty JSON fields. Fix: `set +eo pipefail` at top of parser fn so missing-match is tolerated; defaults applied via `: "${var:=0.0}"`.
  - **`if ! warp ...; then ...` + `set -e` is tricky**: even though `if` neutralizes set -e for the test command, downstream pipe failures inside the body or in subsequent commands still kill the shell. For warp, explicit `set +e; warp ...; rc=$?; set -e` is cleaner — surfaces rc for logging without unexpected exits.
  - **Strata accepts hyphens in bucket names** — `bench-put-small-strata` worked; PRD's bucket-isolation-per-workload pattern is safe.
  - **TiKV PD `cluster is not bootstrapped`** in strata logs after partial volume wipe — wipe ALL of `strata-{pd,tikv}-data` together or expect PD-region-not-bootstrapped errors that surface to S3 clients as `<InternalError>` 500s. Confirmed: clean `docker volume rm $(docker volume ls -q | grep docker_strata-)` then `make up-all` recovered cleanly.
