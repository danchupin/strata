## Codebase Patterns

- shadcn/ui pattern: components live under web/src/components/ui/<name>.tsx as
  SOURCE (not deps). Each component is hand-written / copy-pasted from upstream
  shadcn-ui CLI output; we own and tweak freely. Path alias '@/*' → 'src/*' is
  wired in vite.config.ts (resolve.alias) AND tsconfig.app.json (paths).
  utilities helper is `cn()` in web/src/lib/utils.ts (clsx + tailwind-merge).
  Color tokens come from CSS variables in web/src/styles/globals.css (`hsl(var(--token))`).
  When adding new shadcn primitives, install the matching @radix-ui/react-<x>
  dep and copy the upstream component file — do not pull a wrapper library.
- Theme system: ThemeProvider context (web/src/components/theme-provider.tsx)
  stores theme = 'light' | 'dark' | 'system'; resolvedTheme is derived from
  matchMedia('(prefers-color-scheme: dark)'). Effect adds/removes
  document.documentElement.classList.dark. Persistence:
  localStorage['strata.theme']. ThemeToggle dropdown in the top bar uses Sun
  (light/system) + Moon (dark) icons from lucide-react. To use theme tokens in
  components: write `bg-background text-foreground` etc. — Tailwind picks up
  the CSS vars from globals.css.
- web/ is a NEW source tree at the repo root (sibling of cmd/, internal/, docs/).
  React 18 + Vite 5 + TypeScript 5. Phase 1 stack will add Tailwind 3 + shadcn/ui +
  TanStack Query 5 + recharts 2 + react-router-dom 6 + Playwright 1. Package
  manager: pnpm 9 (NOT npm/yarn — smaller node_modules, reproducible install).
  Lockfile checked in: web/pnpm-lock.yaml.
- go:embed pattern: ROOT package `strata` (not cmd/strata/) owns console.go +
  console_test.go. Declares `//go:embed all:web/dist` + `var consoleFS embed.FS`
  and exposes `ConsoleHandler()` with SPA fallback (any /console/<path> not matching
  a static file falls back to index.html so client-side routes survive direct
  navigation / refresh). The `all:` prefix matters — without it Vite's .vite/-
  prefixed assets are silently dropped. cmd/strata-gateway imports this as
  `strataconsole "github.com/danchupin/strata"`.
- /console/ + /admin/v1/* are reserved path prefixes on the gateway port. Both
  register ahead of the catch-all / S3 router in cmd/strata-gateway/main.go (no
  internal/serverapp package — wiring is inline in main.go). reservedBucketNames
  in internal/s3api/validate.go already includes 'console', 'admin', 'metrics' so
  path-style addressing cannot shadow gateway-internal routes.
- /admin/v1/* is the JSON HTTP contract between UI and gateway. Versioned (v1);
  future-breaking changes ship under /admin/v2/*. OpenAPI 3.1 spec lives at
  internal/adminapi/openapi.yaml (US-003).
- Auth: session cookie strata_session=<JWT> with HS256, 24h Max-Age, HttpOnly +
  Secure + SameSite=Strict + Path=/admin. JWT secret read from
  STRATA_CONSOLE_JWT_SECRET (random hex if unset, with WARN — production
  deployments MUST set it explicitly to survive restarts without invalidating
  sessions). Auth middleware on /admin/v1/* falls back to existing SigV4 path for
  clients without the cookie (curl, programmatic admin clients).
- TanStack Query 5 global QueryClient: staleTime 5_000, refetchInterval 5_000,
  refetchOnWindowFocus true, retry 1. web/src/api/client.ts exposes typed
  wrappers — pages call useQuery({queryKey, queryFn}); polling is automatic.
- Heartbeat shape: cluster_nodes table with default_time_to_live=30 on Cassandra
  (alterStatements add), s/cluster_nodes/<node_id> + background sweeper on TiKV,
  in-process map with timestamp expiry on memory. Lockstep across all three meta
  backends. Each Strata replica writes its own row every 10 s.
- Status derivation (cluster overview): healthy if all nodes report /readyz 200
  in last 30 s; degraded if ≥1 fail but quorum holds; unhealthy if quorum lost.
- Polling cadence per page: 5 s for ranges ≤6 h, 30 s for 24h+, 5 min for 7d
  (per-query refetchInterval). Mind chart re-render cost on longer ranges.
- Bundle target: ≤500 KiB gzipped initial. Use React.lazy + Suspense for
  non-critical routes (Buckets, Bucket Detail, Metrics get code-split). Verify
  via `vite build --report` before US-012 close. Current bundle: 46 KiB gzipped.
- recharts 2.x for charts (smaller than chart.js, declarative). Avoid Apache
  ECharts / Plotly — bundle bloat.
- shadcn/ui CLI copies components into web/src/components/ui/ as source — NOT a

- Dark mode: class='dark' on <html>; CSS variables in globals.css switch values

- Stdlib JWT (HS256): crypto/hmac + sha256 + base64.RawURLEncoding for
  compact serialization; hmac.Equal for constant-time signature compare.
  ~200 lines — skip jwt-go / golang-jwt and the CVE chase.
- Cookie Secure flag: conditional on r.TLS != nil OR X-Forwarded-Proto=https.
  Unconditional Secure blocks dev over plain HTTP because browsers refuse
  to set Secure cookies on insecure origins. Production gateways behind
  TLS terminators still hit the secure branch; dev with `make run-memory`
  keeps working. Path=/admin keeps the cookie off /console/ static asset
  requests — only the API picks it up.
- subtle.ConstantTimeCompare on the secret AND a fake compare on the
  unknown-access-key branch — equalises wall-clock latency so attackers
  can't distinguish "unknown key" from "wrong secret".
- adminapi.authMiddleware resolution order: path bypass list (/auth/login,
  /auth/logout) → cookie JWT path → SigV4 fallback → 401. Whoami requires
  auth so the UI uses it as a session probe.
- BrowserRouter basename='/console' means React Router paths are '/login'
  and '/*' — NOT '/console/login'. Basename prefix is added by the router;
  don't double-prefix in <Link to=...> or <Navigate to=...>.
- React auth state machine: 'unknown' | 'unauthenticated' | 'authenticated'.
  The 'unknown' state during the initial whoami probe prevents a flash of
  the login page on hard-refresh of a deep link.

  age: ≤10 s → healthy, ≤30 s → degraded, >30 s → unhealthy.

- Prometheus client: internal/promclient (stdlib net/http, no
  prometheus/client_golang/api dep). New(baseURL) — empty URL is legal and
  Available()=false. Both Query (instant vector) + QueryRange (matrix) return
  ErrUnavailable when BaseURL=="" OR upstream non-2xx. Wraps errors with
  ErrUnavailable so callers `errors.Is(err, promclient.ErrUnavailable)` to
  branch into "metrics unavailable" UI without distinguishing root cause.
  Hard timeout 5 s per request — admin widget polls don't get to wedge a
  whole gateway shutdown.
- Top-N widget contract: BucketsTopResponse + ConsumersTopResponse both
  carry a top-level metrics_available bool in addition to the rows. UI uses
  this flag (NOT empty-rows heuristic) to decide whether to render '—' in
  the value cell + a "Metrics unavailable — set STRATA_PROMETHEUS_URL"
  warning under the card title. For buckets/top: size_bytes + object_count
  come from meta.Store walks and remain populated even when prom is down;
  only request_count_24h degrades. For consumers/top: the entire response
  is prom-driven, so prom-down ⇒ rows=[] + metrics_available=false. PromQL
  shapes: `sum by (bucket) (increase(strata_http_requests_total[24h]))` for
  buckets; same with by (access_key) for consumer requests; analogous
  strata_http_bytes_total for consumer bytes. Note: the gateway's existing
  metrics counter is `strata_http_requests_total` with labels (method,
  code) — adding (bucket, access_key) labels is Phase 2 work tracked
  separately. Today the queries return empty vectors, so rows arrive empty
  with metrics_available=true (the warning does NOT fire — it only fires
  on Available()=false / Query error). That's intentional: empty-but-available
  is "no traffic in the last 24 h", which is different from "we don't know".
- bucketSizeAndCount caps per-bucket scans at 50_000 objects (50 pages of
  1000) so pathological buckets don't wedge the home-page widget. The
  widget is "top N", not authoritative inventory — Phase 2 will introduce a
  proper bucketstats package with per-bucket counters maintained on
  PutObject/DeleteObject paths so this synchronous walk goes away.
- Consumer row's `user` field is joined from CredentialsStore.Lookup —

- Auth + TanStack Query: AuthProvider holds whoami() inside a
  useQuery({queryKey: queryKeys.auth.whoami, retry: 0,
  refetchInterval: false, staleTime: 60_000, meta:{silent:true}}). Login
  and logout call qc.setQueryData(queryKeys.auth.whoami, session/null)
  so the auth state stays in sync without an extra whoami round-trip.
  Logout additionally calls qc.removeQueries() to drop per-session
  cached data before re-seeding null whoami; otherwise next user signing
  in would briefly see the previous user's bucket list.
- Page error UX with TanStack Query: only render an inline error banner
  when there is no usable cached data (`!q.data && q.error`). Transient
  refetch failures surface via the global toast (with a Retry button
  that calls queryClient.invalidateQueries). The page must NOT

- vite/client types live in web/src/vite-env.d.ts (`/// <reference types="vite/client" />`).
  Without it `import.meta.env` doesn't resolve in tsc. Required as soon
  as any source file reads `import.meta.env.DEV`.
- Metrics dashboard contract (US-009): GET /admin/v1/metrics/timeseries
  takes (metric, range, step?). Supported metric values:
  request_rate, latency_p50, latency_p95, latency_p99, error_rate,
  bytes_in, bytes_out. Each call returns ONE MetricSeries (server names
  it after the public alias — 'p50', 'p95', 'p99' for the three
  latency variants — so the React layer can layer them in one chart by
  calling the endpoint three times). Range parser supports
  time.ParseDuration plus a 'd' suffix (stdlib doesn't accept '7d', so
  parseDurationParam shims '7d'→'168h'). Default step per range:
  15m→30s, 1h→1m, 6h→5m, 24h→15m, 7d→1h. Same metrics_available bool
  contract as buckets/top + consumers/top: 200 + {series:[],
  metrics_available:false} when STRATA_PROMETHEUS_URL is unset OR
  upstream non-2xx — UI then renders the "Metrics unavailable —
  set STRATA_PROMETHEUS_URL" inline message.
- PromQL window placeholder pattern: metricSpecs entries hold the
  expression with a __WINDOW__ literal that is replaced at request
  time with promDurationString(step*4). step*4 keeps the rate window
  wider than the step so adjacent points share at least one full
  scrape — avoids flatlining when the step is very fine. The
  formatter promDurationString picks the largest whole-unit
  representation (h/m/s) so PromQL sees "1m" not "60s" — purely
  cosmetic for query logs but easier to debug.
- Lazy-loaded routes pattern: heavy chart libraries (recharts ~110 KiB
  gzipped) live behind React.lazy + Suspense in App.tsx. The Metrics
  route imports `lazy(() => import('@/pages/Metrics').then((m) =>
  ({ default: m.MetricsPage })))` so the recharts chunk is loaded
  only on /metrics navigation. Initial home-page bundle stays under
  the 250 KiB raw / 120 KiB gzipped headroom budget. Apply the same
  pattern for any future heavy page (object browser hex-viewer,
  trace UI, etc.).
- Per-range refetchInterval: useTimeseries hook overrides the global
  5_000 ms default with a per-range value (5s for ≤6h, 30s for 24h,
  5min for 7d). Set staleTime = refetchInterval/2 so a tab switch
  triggers a refresh roughly halfway through the visible period.
- recharts patterns: ResponsiveContainer wraps every chart inside a
  fixed-height div (h-56). XAxis dataKey='ts' type='number'
  scale='time' domain=['dataMin','dataMax'] formatter picks
  human-friendly tick labels per range (HH:MM:SS for ≤1h,
  HH:MM for 6h/24h, MMM DD for 7d). YAxis tickFormatter does the
  unit conversion (formatBytes scales to KiB/MiB/GiB; formatPercent
  multiplies by 100; formatSeconds picks µs/ms/s). Always pass
  isAnimationActive={false} to recharts series — animation flickers on
  every refetch and is distracting on a polling dashboard.
- mergePoints helper: layered charts (latency, bytes) merge multiple
  per-metric responses into a single ChartPoint[] keyed by ts so
  recharts can render them as overlaid Lines on the same X axis.
  Because each query returns its own series, the React side keys the
  resulting row by the caller-supplied 'key' (p50/p95/p99 etc.) —
  the server's series.name is informational.
- Metric counter coverage today: gateway emits
  strata_http_requests_total{method,code} +
  strata_http_request_duration_seconds (histogram). request_rate +
  latency + error_rate widgets work today. bytes_in/bytes_out widgets
  query strata_http_bytes_in_total / strata_http_bytes_out_total
  which are NOT yet emitted (Phase 2 work tracked alongside the
  bucket/access_key labels). Querying a missing metric returns an
  empty matrix → empty chart with metrics_available=true. That's
  intentional — empty-but-available is "no traffic", different from
  "we don't know".

- **Learnings for future iterations:**
  - PromQL `increase()` over a 24h range correctly handles counter
    resets (process restarts) — prefer it to a hand-rolled
    `delta(counter[24h])` which can show negative deltas across
    restarts. The widget rounds the float result to int64 via `int64(s.Value)`
    — fine for "tens of thousands of requests" but be aware that

  - adminapi.New is now a Config-struct constructor. NEVER reorder
    Config fields — call sites are positional in spirit (composite
    literal) but Go composite literals tolerate field reorder. Adding
    a field is non-breaking; renaming or removing is.

    meta package as a dependency; the table DDL still ships in
    internal/meta/cassandra/schema.go because that's the central
    bootstrap path. Two-package split is intentional — keep it.

    can race with TTL eviction in some cluster topologies and silently
    drop the row. Always pass empty slice (nilToEmpty helper). This
    bit me on a previous Cassandra service.

    ("we know we're up so status=healthy") because then a Cassandra
    table outage looks healthy. Honest "unhealthy" + the empty-state

    then setState only if !cancelled. The cleanup return clears both
    cancelled and the interval. Don't try to cancel in-flight fetches
    via AbortController for Phase 1 — adds bytes for marginal benefit.
    US-008 swaps the whole thing for useQuery anyway.

    enum → rank (0=unhealthy, 1=degraded, 2=healthy, 3=unknown). Pure

    that's the right time to introduce one.

    with bg-emerald-500/15 etc. layers cleanly over the shadcn outline
    variant. The status colour palette is encoded inline in

    a fourth surface uses it (US-007 widgets, US-009 metrics), extract
    to shared utility.

- **Learnings for future iterations:**
  - QueryCache.onError is the right place to centralise error UX.
    Per-component try/catch loops are the wrong place — once you have
    multiple polling queries, you'll re-implement the same error toast
    3+ times. Make the global handler opt-in via meta.silent=true for
    queries that shouldn't toast (auth probes, anything where 401 is
    a normal control-flow signal).
  - Throttle the global error toast per-query-key. Without it, a
    wedged endpoint at 5 s refetchInterval × retry=1 will spam two
    toasts every 10 s — instant user rage. 30 s throttle window is a
    reasonable balance (still tells the user something is wrong, but
    doesn't drown them).
  - Toast store as a module-level useSyncExternalStore (not a React
    context). Reason: the QueryCache onError handler runs OUTSIDE the
    React tree, and needs to dispatch toasts. A context-based toast
    store would force the queryClient to live inside React and have
    a hook to access showToast — circular. Module-level store
    side-steps this cleanly.
  - meta.label on every useQuery. Without it the global error toast
    shows queryKey JSON in the title — opaque to the user. Keep the
    label short ('cluster status', 'top buckets (size)'), it'll
    appear in the toast title 'Failed to load <label>'.
  - Aggregate API client at web/src/api/client.ts. Re-exports per-
    resource files + adds wrappers for endpoints not yet implemented
    so pages can write `import { fetchBucket } from '@/api/client'`
    today and the wrapper just lights up when the backend lands. Saves
    a round of import-path churn per story.
  - vite-env.d.ts triple-slash reference is mandatory the first time
    any source file reads import.meta.env.DEV — without it tsc throws
    'Property env does not exist on type ImportMeta'. Add it once
    early in the project rather than per-story.
  - AuthProvider migration to useQuery is subtle: the whoami probe
    must NOT trigger the global error toast on 401 (which is the
    normal "you're logged out" signal). Set meta.silent=true. retry=0
    so a 401 doesn't double-fetch. refetchInterval=false because the
    global default of 5 s would be wasteful for an auth probe.
  - login/logout: use qc.setQueryData(queryKeys.auth.whoami, value)
    to seed the cache directly instead of refetching whoami. Avoids
    a flash of 'Checking session…' (RequireAuth) between login
    success and the next redirect. Logout MUST call qc.removeQueries()
    before re-seeding null whoami, otherwise the next user signing in
    will briefly see the prior user's bucket/consumer data when the
    UI mounts before the new whoami completes.
  - Page error UX: only show inline error banner when there is no
    cached data. Once useQuery has cached data, transient refetch
    errors are surfaced via the global toast — keeping the existing
    UI mounted is the whole point. Branching on
    `!q.data && q.error instanceof Error` keeps both signals: the
    blank-page-on-first-load failure case AND the
    background-refetch-failed-but-keep-the-table-visible case.
  - Devtools lazy-load: `import.meta.env.DEV ? lazy(() =>

    treats the dynamic import behind the false branch as dead code
    and drops the entire devtools bundle from prod. Verified via
    `vite build` — JS bundle stayed 115 KiB gzipped (devtools alone

  - Co-locate queryKeys: web/src/lib/query.ts owns the central
    queryKeys map. Pages import { queryKeys } and call
    queryKeys.cluster.status (constant array) or queryKeys.buckets.top(by, limit)
    (factory function). Centralising stops accidental key drift —
    mutations and invalidateQueries see the same key as the original
    useQuery.

- **Learnings for future iterations:**
  - PromQL rejects "7d" as a range literal in some versions, but
    accepts "7d" as a query duration arg. The handler does its
    OWN parseDurationParam (with 'd' shim) so the user-facing range
    param matches the spec wording, then converts to a Prometheus-
    safe step using promDurationString.
  - histogram_quantile inside `sum by (le) (rate(...[W]))` is the
    standard pattern. Without the `sum by (le)` wrapper, the
    quantile is computed per-series-set which yields nonsense
    when the histogram is fanned across multiple labels (method).
  - error_rate via `sum(rate(...{code=~"5.."})) / clamp_min(sum(rate(...)),1)`
    avoids divide-by-zero when there's no traffic in the window.
    Returns 0 instead of NaN. recharts will silently drop NaN
    points but the JSON shape becomes ugly (NaN is not valid
    JSON; encoders emit "null" or fail) — the clamp_min guard
    is load-bearing.
  - Lazy-loading via React.lazy + Suspense at the route level
    cuts the home-page bundle by the size of recharts. Don't
    code-split inside a single page (e.g. lazy-loading a chart
    component) — Suspense boundaries are easier to reason about
    at the route boundary.
  - rangeOptions is exported via lookup-by-key so the component
    only stores the string key in state. This keeps the URL-
    serialisable shape simple for a future "remember last range"
    feature without rewriting the state machine.
  - mergePoints folds multiple responses by timestamp into a
    single ChartPoint[]. Recharts renders gaps when a series
    is missing a value at a given ts — connectNulls=true on
    the Line series eliminates the visual gap (we want a
    continuous line even if p99 is missing one tick).
  - Step query param is encoded as plain seconds by promclient
    (e.g. "900" not "15m"); the step query string format is
    flexible upstream, but tests should match what the client
    actually sends. capturedStep="900" not "15m" in tests.
  - Tests that capture upstream Prom calls should use a
    non-greedy match strategy: assert on substring presence
    (Contains "rate(strata_http_requests_total[") rather than
    full equality — the rate window changes with step which
    changes with range, so equality is too brittle.
  - Recharts isAnimationActive=false everywhere on a polling
    dashboard. Default animation is jarring when each refetch
    re-animates from 0.
  - recharts Type=number + scale=time + domain=['dataMin',
    'dataMax'] gives accurate time-spaced X axis when data
    points are not evenly spaced — without it recharts spaces
    them by index.
