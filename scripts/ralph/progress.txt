## Codebase Patterns
- AuditMiddleware (`internal/s3api/audit.go`) wraps state-changing requests; GET/HEAD/OPTIONS skipped. Add publishers via `Publisher AuditPublisher` field after construction (publish only on `EnqueueAudit` success).
- adminapi routes register via `mux.HandleFunc("VERB /admin/v1/...", handler)` in `Server.routes()` (Go 1.22 mux). Trailing-wildcard `{key...}` must be the last segment.

## 2026-05-04 — US-005 — OTel trace ring buffer + endpoint
- New `internal/otel/ringbuf` package — `RingBuffer` implements `sdktrace.SpanProcessor` (no-op `OnStart`/`Shutdown`/`ForceFlush`; `OnEnd` records every span). Bytes-budgeted LRU eviction (default 4 MiB; `STRATA_OTEL_RINGBUF_BYTES` overrides). Per-trace span cap 256 (`PerTraceSpanCap`); excess spans dropped with one WARN per trace via injected `*slog.Logger`. Two indices: `traces` keyed by `trace.TraceID` and `byReq` keyed by `request_id` span attribute. Public `GetByRequestID` / `GetByTraceID` return the wire-shape `Trace` doc.
- Wired into `otel.Init` via `STRATA_OTEL_RINGBUF` (default `on`). When OTLP endpoint is set we install BOTH the existing tail-sampler+OTLP pipeline AND the ring buffer in parallel so failing spans still export and the in-process retention always has data. Refactored `Init(ctx)` → `Init(ctx, InitOptions{Logger, RingbufMetrics})` to thread the slog + Prometheus deps without env coupling. `Provider.Ringbuf()` accessor returns the active `*ringbuf.RingBuffer` for adminapi.
- HTTP middleware now stamps `request_id` on the root span via `r.Header.Get("X-Request-Id")` after `next.ServeHTTP` returns — by then the inner `logging.Middleware` has set the header on the request, and the deferred `span.End()` fires after the attribute is set. Hardcoded header name to avoid an `otel → logging` import edge.
- New admin route `GET /admin/v1/diagnostics/trace/{requestID}` (`internal/adminapi/diagnostics_trace.go`). Resolves by request_id first, falls back to raw trace-id (operator-paste shape). 404 NotFound when the trace has aged out, 503 RingbufUnavailable when the ring buffer is disabled. Audit override `admin:GetTrace`.
- New Prometheus pair `strata_otel_ringbuf_traces` (gauge) + `strata_otel_ringbuf_evicted_total` (counter). Adapter `metrics.OTelRingbufObserver` keeps prometheus out of `internal/otel/ringbuf` (same pattern as `AuditStreamObserver`).
- serverapp wiring: `strataotel.Init(ctx, InitOptions{Logger, RingbufMetrics: metrics.OTelRingbufObserver{}})`; `adminapi.Config.TraceRingbuf = tracerProvider.Ringbuf()`.
- Tests: `internal/otel/ringbuf/ringbuf_test.go` (bytes-budget eviction order, per-trace span cap, request-id index, attr type round-trip, root-span detection, invalid SpanContext drop, no-op SpanProcessor methods); `internal/adminapi/diagnostics_trace_test.go` (200 known, 404 unknown, 503 disabled, trace-id fallback). Existing `otel_test` no-endpoint tests now `t.Setenv(EnvRingbuf, "off")` to keep their noop-provider intent.
- Files changed: `internal/otel/ringbuf/{ringbuf,ringbuf_test}.go` (new), `internal/otel/{otel,middleware,otel_test}.go`, `internal/adminapi/{server,diagnostics_trace,diagnostics_trace_test}.go`, `internal/metrics/metrics.go`, `internal/serverapp/serverapp.go`.
- Quality: `go build ./...` ✓, `go vet ./...` ✓, `go vet -tags integration ./...` ✓, `go test ./...` ✓.
- **Learnings for future iterations:**
  - Span processors in OTel SDK 1.x register as a list; the SDK fans `OnEnd` to each in order. Installing the tail-sampler-wrapped BSP and the ring buffer as separate processors means the ring buffer sees ALL spans regardless of sample ratio (which is the point — operators want the last 10 minutes retained even when ratio=0.01).
  - `tracetest.SpanStubs.Snapshots()` is the way to fabricate `sdktrace.ReadOnlySpan` in tests without driving a real TracerProvider — the SDK's `ReadOnlySpan` interface is unexported, so direct construction is a non-starter. Use stubs to drive `OnEnd` for unit tests of any custom SpanProcessor.
  - `attribute.Value` discriminates type via `.Type()` / `.AsString()` / `.AsInt64()` / `.AsBool()` etc. Build a switch over `attribute.STRING|BOOL|INT64|FLOAT64|*SLICE` and fall back to `.Emit()` for the slice/array types you don't care to round-trip. JSON encoding of `map[string]any` then matches what the JS waterfall expects.
  - Adding ringbuf changed `otel.Init`'s signature from `Init(ctx)` to `Init(ctx, InitOptions{...})`. Only one production caller (`internal/serverapp`) and the env-driven test file. Future env-derived options should ride on `InitOptions`, not new env-only globals — keeps the dependency injection seam visible at the wiring layer.
  - Ring-buffer `request_id` indexing depends on the otel HTTP middleware running OUTSIDE the logging middleware, because the inner `logging.Middleware` is what stamps `r.Header["X-Request-Id"]`. After `next.ServeHTTP` returns the header is set, then we copy it to the span attr, then the deferred `span.End()` fires → ring-buffer's `OnEnd` reads the attr and populates `byReq`.
  - For tests of any new admin handler that needs a ring buffer, do not initialize via `otel.InitWithConfig` — too much SDK boilerplate. Construct `ringbuf.New()` directly, build stubs via `tracetest.SpanStubs{...}.Snapshots()`, call `rb.OnEnd(...)`, then assign `s.TraceRingbuf = rb`.
