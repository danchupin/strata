## 2026-05-15 — US-004 Admin force-empty enqueues chunks into GC queue
- `internal/adminapi/buckets_delete.go::forceEmptyDrainPage` now captures the `*meta.Object` returned by `DeleteObject` (both ListObjectVersions path AND ListObjects fallback) and hands it to a new helper `Server.enqueueRemovedChunks(ctx, job, removed)` immediately after each successful delete. Helper resolves region from `s.Region` (fallback "default" matching s3api / lifecycle convention), early-exits on `removed==nil || removed.Manifest==nil || len(removed.Manifest.Chunks)==0` (delete markers + MPU residue), copies the chunk slice before enqueue (defensive — Manifest may be reused by callers), and best-effort calls `s.Meta.EnqueueChunkDeletion(ctx, region, chunks)`. Enqueue errors are logged via `s.Logger.Printf` but never abort the drain: the meta DELETE already committed, so a transient GC queue failure must be repairable via a follow-up force-empty pass (which now picks up the stranded chunks because the meta rows are gone but the data backend still has them — the worst case is operator-driven retry, not data loss).

- **Learnings for future iterations:**
  - `Meta.DeleteObject` returns the fully-populated `*meta.Object` (Key, VersionID, Manifest with .Chunks slice). All other deletion paths (s3api/server.go, lifecycle/worker.go) capture this and enqueue chunks — admin force-empty was the lone leak site. Pattern: any new code path that calls `Meta.DeleteObject` MUST capture the returned object and route its chunks to GC, mirroring `internal/s3api/server.go:99-105` / `internal/lifecycle/worker.go:403-409`.
  - Region resolution convention across enqueue sites: `region := <Server>.Region; if region == "" { region = "default" }`. Keep the fallback in lockstep — gc workers query `ListGCEntries(ctx, region, ...)` so a typo'd default region routes entries into a queue partition the worker never inspects.

Log + continue; operator retries via re-running force-empty pick up the survivors because their meta rows are gone.
  - In-memory backend's `ListGCEntries` accepts a `before` cutoff — pass `time.Now().Add(time.Hour)` to capture freshly-enqueued rows. Cassandra+TiKV honour the same contract.
  - `Logger: log.Default()` in `New()` ensures `s.Logger.Printf` is never nil — no nil-guard needed in handler code.
