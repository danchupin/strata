## Codebase Patterns
- 4-story cycle closing 1 P2 ROADMAP entry ("Compose default + multi-cluster profiles race for worker leases on shared Cassandra"). Structural fix not band-aid: collapse the parallel single-cluster + multi-cluster strata services to ONE multi-cluster service. Race impossible by construction (only one strata).
- **Root cause** (operator-discovered live): `strata` (default service, knows only `default` RADOS) + `strata-multi` (multi-cluster, profile-gated) on shared Cassandra raced for global `gc-leader-N` / `rebalance-leader-N` / etc leases. When `strata` won and a GC entry for `cephb` got popped, deletion failed forever (no cephb connection) → queue piled up → `deregister_ready` stuck false → drain wedged.
- **Fix shape**: drop `strata` (single-cluster) + `strata-features` services. Rename `strata-multi` → `strata` (no profile, always-on, port `9999`, mounts both ceph keyrings, attaches to `default` + `cephb` clusters by default). Single-cluster smoke now via `STRATA_RADOS_CLUSTERS=default:...` env override at runtime, not a separate compose service.
- **Compose semantics reminder**: services with NO `profiles:` key always start; services WITH `profiles: [X]` start ONLY when `--profile X` is passed. There is no fake "default" profile. Don't try to gate `strata` behind a profile + hope bare `up -d` still starts it — that pattern is wrong.
- **lab-cassandra-3 profile** (US-003): mirror of `lab-tikv-3` for Cassandra-backed multi-replica HA validation. 3 named services `strata-cass-a/b/c` behind `profiles: ["lab-cassandra-3"]`, distinct host ports `10001`/`10002`/`10003`, fronted by `strata-lb-nginx-cass` on `:10000` (mirror of `strata-lb-nginx` from `lab-tikv-3`). Default `strata` service stays running alongside; operator stops it before lab harness via `docker compose stop strata`.
