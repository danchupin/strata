## Codebase Patterns
- `meta.Store` interface in `internal/meta/store.go` is implemented by THREE backends (memory, cassandra, tikv). Adding a method requires implementations on all three + a contract test in `internal/meta/storetest/contract.go::cases`.
- Cassandra schema migrations are additive only: `tableDDL` (idempotent CREATE) and `alterStatements` (idempotent ALTER ADD column). Never destructive.
- Cassandra LWT (`IF EXISTS`/`IF NOT EXISTS`) is REQUIRED for read-after-write coherence on any row that may be queried with quorum after a previous LWT write. Plain UPDATE after LWT INSERT leaves Paxos state stale and `LOCAL_QUORUM` reads can observe pre-update value. Counter increment on `bucket_stats` MUST use `UPDATE ‚Ä¶ IF EXISTS`.
- TiKV pessimistic txn with EARLY-RETURN paths must call `txn.Rollback()` explicitly. `defer rollbackOnError(txn, &err)` only fires when err != nil. Any non-error early return (e.g. CAS-reject `applied=false, nil`) MUST `txn.Rollback()` first or the LockKeys lease leaks for txn lifetime ‚Äî deadlocks the in-process memBackend forever.
- Per-bucket / per-user single-blob configs (lifecycle, CORS, policy, public-access-block, ownership-controls, quota) reuse `setBucketBlob` / `getBucketBlob` / `deleteBucketBlob` helpers in both backends instead of fresh CRUD per endpoint.
- Workers register via `workers.Register(workers.Worker{Name, Build})` in `cmd/strata/workers/<name>.go::init()`. Build receives `workers.Dependencies`, returns `workers.Runner`. Supervisor handles leader-election (`<name>-leader` lease) + panic restart with backoff.
- Worker envs read at Build time (no per-worker flags). New worker env knobs follow the `STRATA_<NAME>_*` convention.
- AuditMiddleware in `internal/s3api/access_log.go` stamps audit rows from request context. Admin handlers stamp operator-meaningful audit row via `s3api.SetAuditOverride(ctx, action, resource, bucket, principal)` ‚Äî Action shape is `admin:<Verb>` (e.g. `admin:PutBucketQuota`).
- OpenAPI contract `internal/adminapi/openapi.yaml` is the canonical Admin-API source. New admin endpoints must update it + handler.
- Existing Prometheus metric: `strata_http_requests_total{bucket,access_key}`. New metrics for quota cycle: `strata_quota_reconcile_drift_bytes{bucket}` (gauge).
- Web UI: React+TS under `web/`. Per-bucket page is `web/src/pages/BucketDetail.tsx`. IAM users page exists; new per-user Billing page lands as `web/src/pages/UserBilling.tsx`. Playwright e2e in `web/e2e/`.
- `BumpBucketStats` is the canonical write path for the live counter ‚Äî every PUT / DELETE / multipart Complete / lifecycle expire MUST go through it. Negative deltas allowed.
- Quota check happens BEFORE writing data (PUT-validate time). Live source-of-truth is `bucket_stats` (bucket scope) + sum-across-user-buckets (user scope). v1 user-scope check uses `ListBuckets(ctx, owner)` walk; denormalised user_bucket_count is a P3 follow-up.
- `byte_seconds √ó 24h` rollup is intentionally crude v1 approximation. Document the caveat in `docs/site/content/best-practices/quotas-billing.md`. Continuous integration is a P3 follow-up.
- Out of scope this cycle: invoice ledger / payment integration. Stop at the `usage_aggregates` feed.
- Quota wire codec lives in `internal/meta/quota.go` ‚Äî `EncodeBucketQuota` / `DecodeBucketQuota` + user variants. JSON-shaped so additive fields land without an ALTER. All three backends share it.
- "Not configured" for quota Get is signalled by `(zero-value, false, nil)`, never a sentinel error ‚Äî gateway PUT-validate stays branch-on-bool.
- Per-user single-blob configs need a parallel triple keyed on `userName` (string). Cassandra: dedicated table (e.g. `user_quota`). TiKV: dedicated key prefix (e.g. `prefixUserQuota`) + helper like `UserQuotaKey`. Memory: `map[string][]byte` field.

- **Learnings for future iterations:**
  - Cassandra has no `SET col = col + ?` for non-counter bigint columns ‚Äî counters can't compose with regular columns or LWT. The PRD wording "UPDATE ‚Ä¶ SET used_bytes = used_bytes + ? IF EXISTS" is shorthand for the read-modify-CAS loop with `IF used_bytes = ? AND used_objects = ?`. Counter tables are not viable because counters cannot live inside an LWT and concurrent bumps lose read-after-write coherence.
  - In-memory hot-path wiring must subtract the prior row's size on unversioned PUT overwrite (and on Suspended-mode null replacement). Otherwise the counter monotonically grows even when the live object set's byte total stays flat. Same invariant applies to CompleteMultipartUpload's unversioned-overwrite path.
  - `DeleteObject` skips the counter decrement for delete markers ‚Äî markers occupy no bytes and the row count is already absorbed into the marker (which lays on top of versioned objects without removing them). Forgetting this gives spurious -1 decrements on versioned-DELETE that creates a marker.
  - TiKV pessimistic-txn for read-modify-write on a single key is the canonical shape: `Begin(true) ‚Üí LockKeys ‚Üí Get ‚Üí decode ‚Üí Set ‚Üí Commit`. `defer rollbackOnError(txn, &err)` covers the early-return paths because BumpBucketStats has no CAS-reject branch.
  - Cross-backend lockstep landings remain the right default: memory + cassandra + tikv impls land together with a single contract test case. US-005 collapses to "verify integration suite green + wire hot paths on cassandra+tikv".
