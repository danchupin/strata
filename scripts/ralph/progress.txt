## Codebase Patterns
- meta.Store interface in `internal/meta/store.go` is implemented by THREE backends (memory, cassandra, tikv) + embedded shells (e.g. `serverapp_test.go::fakeMeta` embeds `*memory.Store`). Adding a method requires implementations on all three real backends; embedded shells inherit automatically.
- Logical-vs-runtime shard split: writers stamp `meta.GCShardID(oid) = fnv32a(oid) % meta.GCShardCount` (1024). Readers compute belonging via `entry.ShardID % shardCount == myShard`. Lets operators rescale runtime shards without re-keying.
- Contract suite is registered in `internal/meta/storetest/contract.go` `cases` slice. Add new cases there + a `caseXxx` func — every backend with a `storetest.Run(t, factory)` wrapper picks them up.
- `go vet ./...` is the typecheck gate; `go test ./...` runs unit tests; `go test -race` runs race detector. Integration tests need Docker / `-tags integration`.
- Cassandra env-driven knobs follow the `EnvFooBar`/`FooBarFromEnv()` pair pattern (see `observer.go::SlowMSFromEnv`, `gc.go::GCDualWriteFromEnv`). `Open()` reads the env at construction when the matching `Options` field is nil, so the option pointer cleanly distinguishes "explicitly set" from "fall back to env".
- Dual-write cutover pattern (US-002 cassandra `gc_entries_v2`): writers fan-out under env-gated dual-write flag, reads prefer the new table and top up from the legacy table when the result is short, ack deletes both sides only while dual-write is on. Reusable shape for any future additive schema migration that needs an in-place cutover.
- `cmd/strata/workers.Worker.SkipLease` (added US-004): when true, supervisor skips the per-worker `<name>-leader` lease and runs Build → Run directly. Use for workers that own their leader-election internally (gc fan-out → per-shard `gc-leader-<i>` leases). Supervisor still wraps Run with panic recovery + backoff. Workers must wire their lease transitions through `Dependencies.EmitLeader(name, acquired)` so heartbeat `leader_for=<worker>` chips still flip.
- `strata_worker_panic_total` (US-004): now `[]string{"worker", "shard"}`. Non-sharded workers pass `shard="-"`; gc fan-out passes `strconv.Itoa(shardID)`. When adding a new sharded worker, follow the same shape.
- Per-bucket lease + distribution gate (US-005, lifecycle): `lifecycle.Worker.Locker` + `ReplicaInfo func() (count, id int)`. Distribution: `fnv32a(bucketID) % count == id` filter, then non-blocking `Locker.Acquire("lifecycle-leader-<bucketID>", ...)`. ReplicaInfo returning id<0 skips the whole cycle (no defensible stake); count<=0 disables the gate. Legacy callers (admin tick, bench) leave both nil → process every bucket sequentially. Production wiring on `cmd/strata/workers/lifecycle.go::lifecycleReplicaInfo(STRATA_GC_SHARDS)` reads `workers.GCFanOut.HeldShards()`. Reusable shape for any future per-resource lease distribution (e.g. inventory, replication).

- **Learnings for future iterations:**
  - In-process `memory.Locker` does NOT model fair leader-election the way Cassandra LWT / TiKV pessimistic-txn lockers do — once a replica wins a lease its renew loop holds it indefinitely. Tests against the in-process locker must restrict themselves to the disjointness invariant (no double-grant) rather than fairness (every replica busy). Production fairness is a property of the underlying lease scheduler, not of FanOut.
  - When wrapping per-worker panic recovery inside FanOut, the supervisor's outer panic recovery still applies — but FanOut handles the interesting per-shard case so the supervisor only sees an unrecoverable FanOut.Run failure.   - `Dependencies.EmitLeader` is wired by `Supervisor.Run` into `s.emitLeader` after caller-supplied overrides. SkipLease workers should call `deps.EmitLeader(name, acquired)` from their internal leader-election callback so heartbeat chips still flip. The first call to `Supervisor.LeaderEvents()` allocates the channel; for SkipLease workers, no acquire/release events fire at the supervisor level, so the worker MUST emit them itself.
  - Adding a label to a `prometheus.CounterVec` is a metric-shape change for downstream dashboards. `strata_worker_panic_total` went from `["worker"]` to `["worker", "shard"]` in this commit; non-shard workers pass `"-"`. The Grafana panel queries (none currently reference shard) gracefully degrade to a single time-series per worker since the new label is constant for non-fan-out workers.
  - The `gc.FanOut.HeldShards()` accessor is the integration point for US-005 — lifecycle reads it to derive its replica ID. The `GCFanOut` package-level pointer in `cmd/strata/workers/gc.go` is the cross-worker bridge. When wiring lifecycle, lifecycle's Build closure reads `workers.GCFanOut` (not nil after gc registration runs) and computes `myReplicaID = workers.GCFanOut.HeldShards()[0]` per cycle.
