## Codebase Patterns
- `meta.Store` interface in `internal/meta/store.go` is implemented by THREE backends (memory, cassandra, tikv). Adding a method requires implementations on all three + a contract test in `internal/meta/storetest/contract.go::cases`.
- Quota wire codec lives in `internal/meta/quota.go` (`EncodeBucketQuota` / `DecodeBucketQuota` + user variants). All three backends share it — JSON keeps additive fields forward-compatible.
- Per-bucket single-blob configs go through `setBucketBlob` / `getBucketBlob` / `deleteBucketBlob` helpers on each backend. Memory adds a `map[uuid.UUID][]byte` field; Cassandra adds a `bucket_<kind>` table to `tableDDL`; TiKV adds a `BlobX` constant in `keys.go` + a single shared key prefix.
- Per-user single-blob configs need a parallel triple keyed on `userName` (string). Cassandra: dedicated table (e.g. `user_quota`); TiKV: dedicated key prefix (e.g. `prefixUserQuota`) + helper like `UserQuotaKey`.
- "Not configured" is signalled by `(zero-value, false, nil)`, never a sentinel error — the gateway PUT-validate path stays branch-on-bool.
- Contract tests register in `Run()`'s `cases` slice. Memory runs unconditionally; Cassandra/TiKV run under `-tags integration`.
