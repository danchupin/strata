## Codebase Patterns

(seed entries for this cycle. Replace/remove as US-001..US-022 land.)

- Branch policy (memory/feedback_ralph_branch_policy.md): every cycle
  ends with merge back into main; new cycle starts from main. This
  cycle starts from main = modern-complete + binary-consolidation +
  tikv-meta-backend + web-ui-foundation + s3-over-s3-backend
  (Phase 1 web UI shipped via e27cf21; s3-over-s3 backend via
  12ec0f1). Cycle end (US-022) merges ralph/web-ui-admin back into
  main + pushes origin main.
- Phase 1 surface available on main: /console/ (embedded React+TS),
  /admin/v1/* JSON API (read-only handlers), HS256 JWT session-cookie
  auth (24 h, HttpOnly+SameSite=Strict+Path=/admin) + SigV4 fallback,
  internal/adminapi + internal/heartbeat packages, web/src/pages/*
  (Login, Overview, Buckets, BucketDetail, Metrics, Consumers,
  Settings stub). Phase 2 ADDS write endpoints under /admin/v1/* and
  new pages (IAM, Multipart, AuditLog, Settings, Buckets-with-tabs).
- internal/adminapi/server.go: every new handler registered in routes()
  via mux.HandleFunc("METHOD /admin/v1/<path>", s.handleX). Errors
  via writeJSONError(w, status, code, message). Auth path is
  authMiddleware -> sigv4 fallback -> session cookie verify; leave
  the chain alone, just register handlers.
- meta.Store admin surface ALL exists for the bucket-config triple
  (Get / Set / Delete) for: Lifecycle, CORS, Policy, PublicAccessBlock,
  OwnershipControls, Encryption, ObjectLockConfig, Notification,
  Website, Replication, Logging, Tagging, ACL+Grants, Inventory.
  IAM (CreateIAMUser / DeleteIAMUser / ListIAMUsers /
  CreateIAMAccessKey / DeleteIAMAccessKey / UpdateIAMAccessKeyDisabled
  / ListIAMAccessKeysByUser) all on main. ManagedPolicy +
  AttachUserPolicy SHAPES ARE NEW (US-010 ships them in lockstep
  across cassandra + memory + tikv with storetest contract entries).
- Cassandra schema migrations are additive: alterStatements in
  internal/meta/cassandra/schema.go via idempotent ALTER TABLE ADD
  swallowed by isColumnAlreadyExists. New tables go in tableDDL
  (idempotent CREATE TABLE IF NOT EXISTS). Memory + tikv mirror.
- Audit middleware (internal/s3api/AuditMiddleware) wraps both S3 +
  /admin/v1/* requests. Every Phase 2 write endpoint emits one
  admin:<Action> row. Resource pattern: bucket:<name> for bucket
  ops, object:<bucket>/<key> for object ops, iam:<UserName> /
  iam-key:<AccessKey> / iam-policy:<Arn> for IAM ops.
- Monaco editor: install monaco-editor + @monaco-editor/react. Lazy-
  load via React.lazy + dynamic import inside the editor pages
  (Lifecycle JSON tab, CORS JSON tab, Policy editor, ManagedPolicy
  editor). Verify via vite build --report that monaco lives in its
  own chunk under web/dist/assets/ and the home/metrics/buckets-
  list bundles stay <=500 KiB gzipped.
- Browser-side multipart upload (US-015): operator's session-cookie
  identity is checked at presign-mint time; the URL is signed with a
  server-held credential (not the operator's personal AK/SK). Web
  Worker (web/src/workers/upload.ts) drives the part PUTs in
  parallel so the UI thread stays responsive.
- TanStack Query 5 invalidate pattern: every successful write calls
  queryClient.invalidateQueries({queryKey: ['buckets']}) (or the
  matching key) so the next read reflects the write. NO optimistic
  UI; LWT round-trips are slow enough that fake-success leads to
  confusing rollbacks.
- Form state: react-hook-form (already in package.json) for every
  form. Zod schemas mirror Go validators where practical.
- shadcn/ui: extend the Phase 1 shadcn components rather than add
  new libraries. Toast, Dialog, Select, Tabs, Table, Form, Sheet,
  DropdownMenu, Skeleton, Card all exist under web/src/components/
  ui/. Generate new shadcn components via pnpm exec shadcn add
  <component> when needed.
- BackendPresign toggle (US-020): meta.Store.SetBucketBackendPresign
  + meta.Bucket.BackendPresign already on main from the s3-over-s3
  merge. UI just calls PUT and reads from /admin/v1/cluster/status to
  decide grey-out (cluster.data_backend != 's3').
- ROADMAP maintenance rule (CLAUDE.md): every commit closing a
  ROADMAP item flips the bullet to ~~strikethrough~~ Done format in
  the same commit, with closing SHA inline (or `(commit pending)`
  with a follow-up SHA edit). New gaps: add a fresh entry under the
  right severity section in the same commit.
