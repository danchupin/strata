## Codebase Patterns
- OTel tracer provider lives at `internal/otel/otel.go::Provider`. `tp.Tracer(name)` factory returns named tracers — convention `strata.<component>` (e.g. `strata.data.rados`, `strata.meta.cassandra`). New tracers this cycle: `strata.meta.tikv`, `strata.data.s3`, `strata.worker.<name>`.
- US-001 introduces shared constants `internal/otel.AttrComponentGateway` + `AttrComponentWorker` (attribute.KeyValue). All gateway-side observers (HTTP middleware + cassandra observer + RADOS observer) stamp `strata.component=gateway` via the gateway constant. Worker iteration spans stamp `strata.component=worker` via the worker constant.
- HTTP middleware (`internal/otel/middleware.go`) already emits per-request spans + traceparent extract from headers. US-001 adds `span.SetAttributes(strataotel.AttrComponentGateway)` right after `tracer.Start`.
- Cassandra meta observer (`internal/meta/cassandra/observer.go::SlowQueryObserver`) wraps `gocql.QueryObserver` interface. Emits `meta.cassandra.<table>.<op>` child spans via `tracer.Start` retroactively timed `(q.Start, q.End)`. Failing queries set span status Error. US-001 stamps gateway component on each emitted span.
- TiKV has no gocql-style observer hook — must wrap at Store-method level via functional decorator `observer.WrapOp(ctx, op, table, fn)`. Span name `meta.tikv.<table>.<op>`. Attributes: `db.system="tikv"`, `db.operation`, `db.table`, `strata.component="gateway"`. US-002 builds this.
- RADOS data observer (`internal/data/rados/observer.go::ObserveOp`) wraps `goceph` ops; tracer arg passed through. Pattern reference for span naming convention `data.<backend>.<op>`. US-001 stamps gateway component on each emitted span.
- S3 data backend (`internal/data/s3/backend.go`) uses AWS SDK Go v2. Install OTel via `go.opentelemetry.io/contrib/instrumentation/github.com/aws/aws-sdk-go-v2/otelaws` middleware on `awsCfg.APIOptions` BEFORE client construction. Auto-emits spans per SDK call with semconv `rpc.system=aws-api`, `rpc.service=S3`, `rpc.method=<op>`, `http.status_code`, `aws.region`. Custom hook adds `strata.component="gateway"` + `strata.s3_cluster=<id>`. US-003.
- Tail-sampler (`internal/otel/sampler.go`) decides at OnEnd: any span with status=Error OR http.status_code >= 500 always exports regardless of `STRATA_OTEL_SAMPLE_RATIO`. Same path applies to worker spans — failing iterations always exported.
- Ring buffer (`internal/otel/ringbuf/`) is parallel SpanProcessor — retains every span (even sampled-out ones) under bytes-budgeted LRU for `/admin/v1/diagnostics/trace/{requestID}` admin endpoint. Workers contribute to it automatically.
- Worker shape: `cmd/strata/workers/registry.go::Worker{Name, Build, SkipLease}` registered via `init()` in each `cmd/strata/workers/<name>.go`. `Build` receives `workers.Dependencies` (Logger, Meta, Data, Tracer, Locker, Region, EmitLeader) and returns `workers.Runner`. **`Dependencies.Tracer` is ALREADY `*strataotel.Provider` (line 39 of registry.go), wired at `serverapp.go:254`** — workers just need to use `deps.Tracer.Tracer("strata.worker.<name>")` to get a named Tracer. No struct change required.
- Supervisor (`cmd/strata/workers/supervisor.go::Run`) spawns one goroutine per worker, manages leader-election + panic recovery + exponential backoff. No tracer wiring change — supervisor already passes Provider through Dependencies.
- Worker iteration span helper (US-004): new `cmd/strata/workers/iteration_span.go` exposes `StartIteration(ctx, tracer trace.Tracer, name string) (ctx, span)` and `EndIteration(span, err)`. Span name `worker.<name>.tick`. Attributes: `strata.component="worker"` (via `AttrComponentWorker` constant from US-001), `strata.worker=<name>`, `strata.iteration_id=<atomic-uint64-counter>` per worker. atomic.AddUint64 — safe for fan-out workers (gc fan-out per shard).
- Per-iteration only (no long-running parent — SDK has known issues with multi-hour spans). Operator wanting "which iteration belongs to which worker process" filters by `service.instance.id` (Resource attribute) + `strata.worker`.
- Nil-tracer safety: if `tracer == nil` (unit tests without tracer wiring), iteration spans use `tracenoop.NewTracerProvider().Tracer(...)`. No-panic, no behavior change.
