## Codebase Patterns

(seed entries for this cycle. Replace/remove as US-001..US-019 land.)

- data.Backend interface (internal/data/backend.go) is the contract; three
  in-tree implementations now: memory (test), rados (primary), s3 (this
  cycle, US-001+). All four ops are stream-shaped: Put(reader, size),
  Get/GetRange returning io.ReadCloser, Delete. New backend impls live in
  internal/data/<name>/ — mirror the existing s3 package layout.
- Native shape ONLY: one Strata object = one backend S3 object via the
  backend's own multipart upload. Never split a Strata object into N
  4-MiB-chunks-as-S3-objects — request count amplification kills it.
  Manifest.Chunks is empty when BackendRef is set (US-008).
- VersionId is the defensive correctness lever (US-002, US-004, US-008,
  US-010). Capture from SDK response on every PUT/Complete; pass back on
  every Delete. Three-state semantics: '' (backend without versioning, or
  versioning-off — plain delete), 'null' (versioning suspended — versioned
  delete with literal 'null'), <uuid> (versioning enabled — versioned
  delete bypasses delete-marker creation). Never branch on backend
  capability — let the SDK return what it returns and pass through.
- aws-sdk-go-v2 brings ~6 transitive deps. Acceptable for a P1 first-class
  feature. Pin via go.mod; track upstream releases manually (no
  auto-update cron exists).
- Streaming PUT uses feature/s3/manager.NewUploader — single-shot or
  multipart transparently per object size. Memory bound by part-size
  (default 16 MiB) * upload concurrency (default 4) = 64 MiB peak.
- Operator pre-creates the backend bucket. Strata refuses to start if the
  bucket does not exist or is not writable. Strata does NOT own the
  backend bucket lifecycle (auto-create would leak ownership).
- Backend object key format: <bucket-uuid>/<object-uuid> — UUIDv4 first
  4-8 hex chars give AWS S3 enough entropy for automatic prefix
  partitioning, per-bucket grouping is forensic bonus
  (`aws s3 ls s3://<bb>/<bucket-uuid>/`).
- Cross-region operation (Strata in region X, backend bucket in region Y)
  is an operator concern. Document in docs/backends/s3.md (US-017); do
  NOT add startup region-mismatch detection or enforcement.
- Backend versioning on Strata-managed bucket = anti-pattern but supported
  defensively via VersionID. If operator enables versioning on an existing
  bucket Strata wrote to with empty VersionID rows, those rows leak on
  delete (plain DeleteObject creates delete-marker). Document; do not
  policy.
- Lifecycle bidirectional mapping (US-014) is best-effort. Strata's own
  lifecycle worker keeps running for transition classes the backend does
  not support natively; for natively-supported classes (STANDARD_IA /
  GLACIER / DEEP_ARCHIVE), Strata translates to backend lifecycle and
  skips those objects.
- ROADMAP maintenance rule (CLAUDE.md): every commit closing a ROADMAP.md
  item flips the bullet to strikethrough Done format in the SAME commit,
  with closing SHA inline (or `(commit pending)` with a follow-up SHA
  edit). Every commit discovering a new gap adds a new entry under the
  right severity section in the same commit.
- Tests for the S3 backend that need a real MinIO: use the same
  testcontainers-go pattern as internal/meta/cassandra/store_integration_test.go
  + internal/meta/tikv/tikvtest/cluster.go. Build tag `integration`.
  MinIO single-node ~30 s start; bucket created via mc mb in init step.
- US-001 stub uses blank imports in internal/data/s3/deps.go to keep the
  three aws-sdk-go-v2 packages in go.mod / go.sum until the first real
  consumer lands (config → US-005, service/s3 → US-002+, manager → US-002).
  Drop the matching blank-import line as each package gets a non-blank
  reference. Don't rewrite the stub's signature shape (PutChunks /
  GetChunks / Delete / Close) until the data.Backend contract itself
  changes — US-002..US-004 may extend the interface, but US-001 only
  satisfies the existing one. (deps.go removed in US-002 since all three
  packages are now imported directly.)
- s3 package has TWO constructors: New() returns a stub Backend (no
  client, every op returns errors.ErrUnsupported — preserved for the
  US-001 contract), Open(ctx, cfg) builds a live Backend wired through
  aws-sdk-go-v2. New methods (US-002 Put, future US-003 Get/GetRange,
  US-004 Delete) check b.uploader / b.client != nil and surface
  errors.ErrUnsupported on the stub branch. Don't drop New() until US-009
  flips the gateway dispatch fully to Open.
- testcontainers-go MinIO module: github.com/testcontainers/testcontainers-go/modules/minio
  v0.41.0. Runtime: minio.Run(ctx, "minio/minio:latest", minio.WithUsername,
  minio.WithPassword); ConnectionString returns "host:port" without
  scheme — prepend "http://" before passing to s3.Backend Config.Endpoint.
  Bucket must be pre-created (s3 admin client → CreateBucket); Strata
  refuses to auto-create per US-005 design.
- HeapInuse-based memory bound assertion: sample HeapInuse during the
  upload via a 20 ms ticker goroutine, take the max, assert peak delta
  < body size. With prod defaults (16 MiB * 4 = 64 MiB pool) HeapInuse
  drifts up to ~100 MiB on docker-localhost uploads (Go's GC sawtooth
  + sync.Pool retention of recently-used part buffers), masking the
  bound — use a tightened test config (PartSize=5 MiB, Concurrency=2 →
  10 MiB pool) to keep the assertion provable. Don't rely on HeapAlloc
  delta — same problem (sync.Pool keeps the buffers live).
- data.ErrNotFound (internal/data/errors.go) is the package-wide
  sentinel for "object missing". Backend implementations wrap with %w
  (`fmt.Errorf("...: %w", data.ErrNotFound)`); callers use errors.Is.
  Map SDK-typed errors via errors.As on the concrete type
  (*s3types.NoSuchKey for the s3 backend) — not via string code match.
- aws-sdk-go-v2 field-name irregularity: PutObjectOutput.VersionId
  (lowercase d) but manager.UploadOutput.VersionID (capital D). Same
  thing semantically; trips compile if you assume one shape. Check
  with `go doc <type>` before referencing the field.
- s3.Backend's per-object delete is named DeleteObject (not Delete) —
  the interface method `Delete(ctx, m *Manifest)` already occupies the
  Delete name and is the chunk-based RADOS-style delete. Same reason
  Put/Get coexist with PutChunks/GetChunks. Future US-008+ may rename
  the interface method to DeleteChunks; until then the s3-backend
  per-object surface is DeleteObject(ctx, oid, versionID) and
  DeleteBatch(ctx, refs).
- Tests that need to count HTTP requests on the s3 client: inject a
  *http.Client wrapping a counting RoundTripper via Config.HTTPClient
  → awsconfig.WithHTTPClient. Reset the counter after seed PUTs so
  only the operation under test is observed.
- Test bucket versioning: enable AFTER createBucket via
  client.PutBucketVersioning(VersioningConfiguration{Status:
  s3types.BucketVersioningStatusEnabled}). PutObjectOutput.VersionId
  is empty until versioning is Enabled; admin.ListObjectVersions
  returns Versions + DeleteMarkers separately for assertion.
- internal/config/config.go: every new env var is wired by adding to (1)
  the relevant per-section koanf-tagged struct, (2) defaults() if a
  default exists, (3) envMap (explicit env→koanf-path mapping — no
  auto-mangling), and (4) validate() if it's required or has a
  whitelist. validate() is called once at end of Load() so misconfig
  fails before the binary starts serving — the operator finds out at
  startup, not at first request.
- s3.Backend boot-time writability probe: PutObject + DeleteObject on
  ProbeKey (".strata-readyz-canary") runs inside Open() unless
  Config.SkipProbe = true. Probe captures PutObjectOutput.VersionId and
  passes it to DeleteObject so versioning-enabled backends are left
  exactly as found (no canary version slot, no delete-marker). Tests
  that don't want the probe round-trip flip SkipProbe; tests that need
  the bucket to exist must call createBucket before Open.
- Koanf env-var loading is testable via t.Setenv — Load() reads from
  os.Getenv at each call, so per-test env override works without
  affecting other tests in the same process.
- SDK retry config: awsconfig.WithRetryMode(aws.RetryModeAdaptive) +
  awsconfig.WithRetryMaxAttempts(N) is the v2-canonical way to enable
  adaptive retries with a custom cap. Default retryable codes (5xx +
  429 + connection errors) match PRD US-006 — no need for a custom
  IsErrorRetryable. RetryMode constants live in github.com/aws/aws-
  sdk-go-v2/aws.
- SDK retry log lines are emitted at Debug classification (see
  aws-sdk-go-v2 v1.41.5 aws/retry/middleware.go:219). To honour
  US-006's "WARN" requirement without a full custom retryer, wrap
  o.Logger with a thin filter that re-emits any "retrying request"
  format-prefixed line through slog.Warn. The line carries
  service+operation+attempt; delay is NOT in this format string —
  AC partial on the delay detail.
- Per-op timeout pattern: opCtx + uploadCtx helper methods on Backend
  centralise the WithTimeout/WithCancel choice. Get/GetRange must
  NOT defer cancel — body read happens AFTER the SDK call returns
  and is bound to the same ctx; use cancelOnCloseReader so Body.Close
  releases the context (and the per-op timer). Side effect: opTimeout
  bounds the entire Get lifecycle including body streaming, so
  operators with slow-link large GETs should bump
  STRATA_S3_BACKEND_OP_TIMEOUT_SECS.
- Synthetic SDK responses for retry tests: an http.RoundTripper that
  replays a fixed []responseFn sequence is enough to exercise the
  SDK retry middleware without MinIO. Return raw 503 with a SlowDown
  XML body; the SDK's standard retryable-status set covers it.
  Default adaptive backoff schedules ~3 s between attempts, so a
  three-attempt test runs in ~6 s wall-clock.
- SDK middleware observation point for backend metrics: Initialize step
  (Before) brackets the entire op including serialize → retry → send →
  deserialize. middleware.GetOperationName(ctx) returns the smithy op
  name (e.g. "PutObject") set by the SDK at request bootstrap. After the
  op returns, retry.GetAttemptResults(metadata) returns the per-attempt
  result slice — len(results) - 1 == retry count. Wire via
  o.APIOptions = append(o.APIOptions, instrumentStack) on s3.Options.
- Prometheus collector registration in libraries: keep the package-level
  CounterVec / HistogramVec usable without registration (Inc / Observe
  works regardless), gate prometheus.MustRegister behind sync.Once via
  an exported RegisterMetrics(), and call RegisterMetrics() from each
  live constructor (Open). This keeps the rados-only path from
  registering s3 collectors while making the s3 path register on first
  Open without cmd-side wiring.
- prometheus/client_golang/prometheus/testutil.ToFloat64(c) reads a
  single CounterVec leaf without resetting collector state — the right
  primitive for delta assertions across operations in a single test
  process. testutil pulls in github.com/kylelemons/godebug as an
  indirect dep; expect go mod tidy to add it.

- New env vars STRATA_S3_BACKEND_MAX_RETRIES (s3_backend.max_retries
  int) and STRATA_S3_BACKEND_OP_TIMEOUT_SECS (s3_backend.op_timeout
  _secs int) in internal/config; validate() rejects negatives.
  Documented in README.md env-var table.

  with an in-process http.RoundTripper returning SlowDown, SlowDown,
  200; asserts Put succeeds and transport is invoked exactly 3 times.
- Files: internal/data/s3/{backend.go,config.go,backend_test.go},
  internal/config/{config.go,config_test.go}, README.md, go.mod, go.sum.
- **Learnings:**
  - aws-sdk-go-v2's retry middleware logs at Debug, not Warn. Aligning
    with operator-facing log conventions requires a thin Logger
    wrapper that filters on the "retrying request" format prefix —
    full attempt-aware middleware is overkill for the AC.
  - Get/GetRange cannot defer cancel(): the SDK's response.Body is
    bound to the request ctx, so cancel-on-return aborts the
    streaming read. Transferring cancel ownership to a wrapping
    ReadCloser fixes it without requiring caller cooperation.
  - Per-op timeout for streaming Gets is necessarily a body-read
    ceiling, not a request-setup ceiling. Document via the env-var
    description so operators size it for the worst-case body
    duration, not the headers RTT.
  - The SDK's default retryable-status set already matches PRD US-006
    (5xx, 429, network) and excludes 4xx auth/not-found — no custom
    IsErrorRetryable required.

- **Learnings:**
  - Probe ordering matters on versioning-enabled buckets: a naive
    PutObject + plain DeleteObject leaves a delete-marker behind for
    the canary key. Capture VersionId from PutObjectOutput and pass
    it to DeleteObject so probe is idempotent and leaves zero state
    on every backend type (non-versioned, suspended, enabled).
  - Koanf envMap keeps the env-var → struct-path mapping explicit (no
    auto-mangling) so adding STRATA_S3_BACKEND_* required only
    appending eight envMap entries. The pattern is to mirror the
    naming convention of the section already in the file
    (STRATA_RADOS_*, STRATA_CASSANDRA_*).
  - SkipProbe is the right escape hatch for the unit-test path that
    needs Open() to succeed without any network round-trip. The probe
    cost (one PUT + one DELETE) is fine for boot, but expensive (and
    flaky) for table-driven unit tests that exercise the constructor
    in isolation.
  - validate() of half-set creds (one of AccessKey/SecretKey set, the
    other empty) is the easy operator footgun — without the check,
    the SDK silently falls back to the default credential chain
    instead of using the partially-configured static creds, leading
    to "permission denied" at runtime that's hard to debug. Fail
    fast at config-load time.
