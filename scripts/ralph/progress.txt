## Codebase Patterns
- CI go version: actions/setup-go go-version '1.25', go.mod `go 1.25.10`. go 1.25 REJECTS `go install pkg@version` when the TARGET module's go.mod has replace/exclude directives (1.24.x tolerated them). Bit promtool — never assume a `go install …@latest` CI step that "used to work" still does after a go-toolchain bump.
- For 3rd-party tools whose module won't `go install` (prometheus/promtool), fetch the pinned prebuilt release tarball into /usr/local/bin (on PATH for ubuntu runners) with a bounded curl retry loop.
- Stories here are CI-infra; "Tests pass" mostly means the workflow YAML parses + the touched make target runs green locally. Full CI green is verified by pushing the branch (record run id).
- DIAGNOSE CI-only failures from the actual run log, not local repro: `gh run list --branch main`, `gh run view <id>`, `gh run view --job <jobid> --log`. Many failures (gosec cephimpl, etc.) DON'T reproduce locally because the dev's tool versions differ from the SHA-pinned CI ones.
- go.work unifies main + cephimpl. Any CI tool that file-walks `./...` (gosec) descends into cephimpl, whose backend.go imports go-ceph cgo (needs librados, absent off the ceph image) -> package load failure. Such tools MUST `-exclude-dir=internal/data/rados/cephimpl`. `go list ./...` does NOT include it (module boundary), so the discrepancy is tool-specific.
- gosec lockstep trio: `.github/workflows/ci.yml` (both MEDIUM+LOW steps) + `scripts/check-gosec.sh` (GOSEC_EXCLUDE_RULES + GOSEC_EXCLUDE_DIRS) + `.gosec.yml` (rationale). Change one -> change all three.
- Pipe-to-tail masks the upstream exit code (`$?` = tail's). Capture tool exit with `cmd >/tmp/log 2>&1; echo $?`.
- ResponseWriter wrappers (e.g. s3api auditWriter capturing status) MUST forward `http.Flusher` or they silently 500 any downstream SSE/streaming handler (`w.(http.Flusher)` fails). Unit tests with httptest.ResponseRecorder won't catch it (recorder implements Flusher) — only a wired httptest.NewServer / real `go run` server does. Same caveat for http.Hijacker if raw-conn handlers exist.
- Frontend e2e (web/e2e/*.spec.ts) boots a real `go run ./cmd/strata server` (memory backend, AUTH_MODE=off) on :9999 with embedded console via playwright webServer. Backend-only Go fixes are picked up on next recompile; web/dist need not be rebuilt. Run a single flow with `node_modules/.bin/playwright test <file>.spec.ts -g "<name>"`.
