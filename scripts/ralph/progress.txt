## Codebase Patterns
- Worker concurrency knobs are read at Build time in `cmd/strata/workers/<name>.go` from `STRATA_*` envs (sibling of existing `STRATA_GC_INTERVAL` etc); no per-worker flags.
- Bounded errgroup pattern for per-entry parallelism: `eg := new(errgroup.Group); eg.SetLimit(N); eg.Go(func() error { ...; return nil })`. Always return nil from goroutines so per-entry failure does not cancel siblings (log-and-continue). Wrap each goroutine in `defer recover()` — supervisor's panic backoff is for top-level panics only.
- Counters under concurrent writes use `atomic.Int64` instead of plain `int`.
- `golang.org/x/sync` lives in `go.mod`; it was indirect — bumping to direct via `go mod tidy` after first errgroup import.
- Go 1.25 modernization: `for i := 0; i < n; i++` flagged as `rangeint`; use `for i := range n`. Loop var aliasing (`e := e`) is also flagged (`forvar`); not needed since Go 1.22.
