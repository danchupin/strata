### Learnings for future iterations
- **Go's http2 ConfigureServer cipher gotcha**: when `srv.TLSConfig.CipherSuites` is non-empty, `golang.org/x/net/http2.ConfigureServer` (called automatically by `ListenAndServeTLS`) returns `"http2: TLSConfig.CipherSuites is missing an HTTP/2-required AES_128_GCM_SHA256 cipher"` unless at least one of `TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256` / `TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256` is present. The check is naive â€” it does NOT recognize TLS 1.3 ciphersuites (TLS_AES_128_GCM_SHA256 etc) as satisfying RFC 7540's intent. Workaround: always include the two TLS 1.2 ECDHE-AES-128-GCM ciphers in any non-nil CipherSuites list, regardless of TLS 1.3-only intent. Alternative: set CipherSuites=nil and rely on Go's defaults.
- **Cert chain via `srv.TLSConfig.Certificates`** is the modern path â€” call `srv.ListenAndServeTLS("", "")` and Go reads from `Certificates`/`GetCertificate`. The cert/key file path args are only consulted when both fields are nil. This sets US-003 up cleanly (SNI multi-cert dispatch via `GetCertificate` swap).
- **Mozilla SSL Config profiles â†’ Go cipher list mapping**: Modern is "TLS 1.3 only" per Mozilla's actual recipe (Mozilla doesn't pin Go-specific TLS 1.3 cipher IDs â€” those are always all 3 by RFC 8446). Intermediate is 6 TLS 1.2 ECDHE AEAD ciphers (GCM-128, GCM-256, CHACHA20, each Ã— ECDSA/RSA). Go's `tls` package treats TLS 1.3 cipher IDs as constants that can sit in `CipherSuites` but are never actually negotiated â€” they're ignored at handshake time per RFC 8446 ("TLS 1.3 ciphersuites are not configurable").
- **`openssl req` ec keygen on macOS**: prefer `openssl req -x509 -newkey ec -pkeyopt ec_paramgen_curve:prime256v1 â€¦` over the bash process-substitution form `-newkey ec:<(openssl ecparam -name prime256v1)`. The substitution form hangs silently on LibreSSL (macOS default). The `-pkeyopt` form works on both LibreSSL + OpenSSL 3.
- **`openssl s_client` is too interactive for unattended smoke** even with `echo Q | openssl s_client â€¦` â€” LibreSSL on macOS hangs differently. Prefer `curl -sv --cacert <ca> https://host:port/path 2>&1` and grep for `SSL connection using TLSv1\.[23]` / `SSL certificate verify ok`. Bonus: curl also tests `--http2` ALPN negotiation in the same invocation.
- **HTTPS test pattern via `httptest`**: `httptest.NewUnstartedServer(mux)` â†’ `ts.TLS = tlsCfg` â†’ `ts.StartTLS()` lets you inject the exact `tls.Config` you'd build at production boot. The cert pool for the test client comes from reading the same cert PEM file written by `writeSelfSignedCert` helper. Round-trip latency is < 5 ms â€” fits unit-test budgets.
- **Self-signed cert generation in Go tests**: `ecdsa.GenerateKey(elliptic.P256(), rand.Reader)` + `x509.CreateCertificate` with SerialNumber/Subject/Validity. For 127.0.0.1 / IP-based SAN, set `tmpl.IPAddresses = []net.IP{ip}` (NOT `DNSNames`). `tmpl.IsCA = true` + `BasicConstraintsValid = true` makes the cert self-signed-CA so it can sign itself. ### Learnings for future iterations
- **`httptest.StartTLS` clobbers `tls.Config.Certificates`** when the slice is empty (Go 1.25 stdlib `net/http/httptest/server.go:167`): `if len(s.TLS.Certificates) == 0 { s.TLS.Certificates = []tls.Certificate{cert} }`. This silently overwrites GetCertificate-only configs because the server-side selection at `crypto/tls.Config.getCertificate` returns from `Certificates` when `clientHello.ServerName == ""`. IP-literal connections (e.g. `https://127.0.0.1:port`) emit no SNI per RFC 6066. Workaround for tests that need to assert GetCertificate-backed dispatch: bind a raw `net.Listen("tcp","127.0.0.1:0")` + `http.Server.ServeTLS(ln,"","")` and never go through `httptest`.
- **`atomic.Pointer[T]` for hot-reloadable read paths**: every TLS handshake does a single `Pointer.Load()` returning an immutable snapshot â€” zero contention with reload-side writers. The snapshot stays alive for any in-flight handshakes referencing it; Go GC reclaims when the last reference drops. Reload-side cost: build new snapshot (off the read path) + `Pointer.Store(new)`. Avoid RWMutex for read-heavy hot paths where the write rate is low.
- **fsnotify on k8s ConfigMap/Secret mounts**: never watch the file basename, always watch the parent directory. Kubelet stores data in a versioned `..YYYY_MM_DD_HH_MM_SS.abc/` subdir and exposes a `..data` symlink to the current version. Rotation creates a new versioned dir and atomically renames `..data_tmp` onto `..data` via `os.Rename` (POSIX atomic). `fsnotify.Add(file)` on the outer `tls.crt` symlink fires once on initial-symlink-deletion-via-rename then stops firing because the watch is keyed on inode and the new inode is different. Watch the parent dir for RENAME/CREATE events on the `..data` basename (kubelet projects this name unconditionally) AND on outer symlink basenames. Always pair fsnotify with a periodic re-stat ticker (default 60s) because the watch dies silently on some filesystems (overlayfs, gVisor, etc).
- **fsnotify v1.9.0 is already in `go.sum`** as a transitive dep (via `knadh/koanf` â†’ `knadh/koanf/providers/file`). Direct use just requires promoting it from `// indirect` to a top-level require in `go.mod`. Do NOT run `go mod tidy` in this workspace â€” `cephimpl` is workspace-resolved and tidy attempts `latest` query against the public module proxy which fails. Manual go.mod edit is the path.

### Learnings for future iterations
- **`*http.Transport.Clone()` is the safe primitive for transport overrides**: `http.DefaultTransport.(*http.Transport).Clone()` returns a fresh transport with the SDK-friendly defaults (DialContext, ForceAttemptHTTP2, MaxIdleConnsâ€¦) pre-set. Mutating `transport.TLSClientConfig` on the clone never bleeds into the default transport shared by other tests / unrelated SDK clients in the same process. Same pattern works for `aws-sdk-go-v2`'s `aws.Config.HTTPClient` swap because the SDK wraps whatever `*http.Client` you hand it without re-cloning the transport.
- **`httptest.NewTLSServer` leaf-cert-as-CA trick** is reusable across backends â€” leaves are self-signed (Issuer==Subject) so writing the leaf DER to a PEM file gives you a valid trust anchor without poking at the unexported test-internal CA. Same code path used by US-004 Cassandra TLS integration test; lifted into the s3 tls_test.go as `writeServerCertAsPEM`. Eliminates the need for any cert-generation library inside the s3 test surface.
- **Per-cluster vs global TLS resolution must be "winner-takes-all"**: tempting to merge per-cluster + global on a per-field basis (per-cluster ca_file but global skip_verify, etc), but that creates surprise semantics when one knob is omitted (operator forgets to set `skip_verify: false` on a per-cluster override, inherits a global `skip_verify: true` they didn't expect). Decision: any single per-cluster TLS key replaces the global block ENTIRELY. `ClusterTLS.HasAny()` discriminates zero vs partial-zero â€” `nil` and `&ClusterTLS{}` both fall through to global; any field set on the per-cluster block locks in the per-cluster bundle.
- **GaugeVec label changes are a breaking change for existing series**: extending `strata_backend_tls_skip_verify` from `{backend}` to `{backend, cluster}` means any existing PromQL `strata_backend_tls_skip_verify{backend="cassandra"}` query that previously matched will no longer match unless `cluster=""` is added â€” Prometheus treats different label sets as different series. Pre-launch hard-cutover is acceptable per project policy; for post-launch backends the equivalent would be a new gauge name (`strata_backend_tls_skip_verify_v2`) deployed alongside the old one until dashboards migrate. Going forward, design label sets to anticipate multi-cluster from the outset (`cluster=""` for single-instance backends is cheaper than adding a label later).
- **Per-cluster `tls` field on `S3ClusterSpec` uses `*ClusterTLS` not `ClusterTLS`**: distinguishes "operator opted out of TLS for this cluster explicitly" (impossible â€” there is no opt-out, only opt-in) from "operator did not configure the field, fall back to global". `json:",omitempty"` on a pointer marshals as missing when nil â€” keeps the legacy `STRATA_S3_CLUSTERS` JSON shape unchanged for clusters that don't need a per-cluster override. The `HasAny()` helper bridges the "operator explicitly set `tls: {}` empty" edge case to fall through to global (any single field empty on the value path is treated the same as missing).
- **PEM parse + cert/key load happen at `connFor` boot path, not at first request**: surfacing PEM-decode / x509 parse / mismatched cert/key errors as gateway-boot errors gives operators a fast feedback loop. The penalty for deferring to first SDK call is a generic HTTP 500 on the first data-plane PUT, plus a leaking goroutine that retries. Mirror this pattern for any future backend mTLS wiring â€” load cert materials in the construction path, not the hot path.
- **`crypto/tls.Config` `Certificates` slice fully replaces client-cert chain**: setting `Certificates = []tls.Certificate{cert}` overrides any prior chain; appending is rarely what you want. `RootCAs` is the trust anchor pool (server-side cert validation); `Certificates` is the local client cert chain. Both fields are independent â€” populate either / both / neither.
