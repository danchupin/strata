- **Learnings for future iterations:**
  - **`omitempty` + empty-string is the right shape for "this field optional per-backend".** A Go field of type `string` with `omitempty` serialises to absent for the memory backend (Cluster="") but populated for rados/s3. UI must handle both — "absent" key and "" value should both render as "—". Don't drift to a `*string` pointer; it forces every consumer through a nil check for no gain.
  - **OpenAPI updates do NOT need a path entry.** When the storage/data path predates the schema-driven workflow, adding just the schema is correct — the AC says "schema definition", not "endpoint". Future stories that touch the storage endpoint should land the `/storage/data` path on the spec.
  - **CI ceph build will exercise the refactored helper too.** Because `health_helper.go` has no build tag, both default and `-tags ceph` builds compile the same code; ceph tag only adds health.go on top. No risk of divergent semantics between unit-test and prod.
  - **For US-002 next iteration**: the helper now returns sorted (cluster, pool, ns) tuples — the UI's "Cluster column on Pools table, sort ascending with empty last" can rely on that ordering. Storage page in React shouldn't re-sort the wire response.
