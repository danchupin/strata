## Codebase Patterns
- ROADMAP.md is canonical project-state list. Closing items: flip bullet to `~~**P<n> — <title>.**~~ — **Done.** <summary>. (commit `<sha>`)`. New gaps: add bullet under P1/P2/P3/Known latent bugs in same commit.
- Project root CLAUDE.md = project memory (auto-loaded by Claude Code). scripts/ralph/CLAUDE.md = Ralph task prompt. Knowledge goes in root CLAUDE.md only.
- Docs-only stories: `go vet ./...` is sufficient typecheck.
- CLI binaries in this repo use stdlib `flag`, not Cobra (see cmd/strata-admin pattern: `app{out,err,args}` struct + `run(ctx) int|error`, subcommand dispatch via switch on `args[0]`). Match this style for new CLIs to keep deps small and tests trivial.
- Build version comes from `runtime/debug.ReadBuildInfo().Settings` via `vcs.revision` (no -ldflags pipeline today). Fall back to literal `"unknown"` when running outside a git checkout.
- Gateway entrypoint lives in `internal/serverapp.Run(ctx, cfg, logger) error`. Both `cmd/strata server` and the transitional `cmd/strata-gateway` delegate to it — when changing gateway wiring, edit serverapp.go (helpers buildDataBackend/buildMetaStore/buildHealthHandler/auditRetention/vhostPatterns/awsKMSClientFactory live there, with tests in serverapp_test.go).
- Cross-cutting `strata server` flags promote to `STRATA_*` env vars via `applyServerFlagOverrides` BEFORE `config.Load` runs. New cross-cutting flags follow the same pattern (set env, never mutate the cfg struct directly).
- Pre-existing local smoke gap: `BOGUS x-amz-storage-class` returns 200 instead of 400 (smoke.sh exits 1 at that step). Reproduces on legacy strata-gateway too — not introduced by binary consolidation.
- Worker registry: `cmd/strata/workers.Register({Name, Build})` from per-worker `init()`. Build receives `workers.Dependencies` (Logger, Meta, Data, Tracer, Locker, Region) and returns a `workers.Runner`. Supervisor handles leader election (`<name>-leader`), panic recovery (`strata_worker_panic_total{worker}`), exponential backoff (1s→5s→30s→2m), and stable-after reset (5m). Lease loss restarts immediately (no backoff). US-005..US-012 each add one `cmd/strata/workers/<name>.go` registration.
- Test seam pattern for Supervisor: `Backoff []time.Duration` (flat = `[]time.Duration{0,0,0,0}`), `Sleep Sleeper` (no-op `func(ctx, d){}`), `LeaderTTL/LeaderRenew/AcquireRetry` short. For lease-loss tests, use a `fakeLocker` with controllable `expire(name)` rather than `memory.NewLocker` — the memory Locker's TTL races with renew so you can never reliably preempt.
- Per-worker env reads happen inside Build (NOT via cfg.GC.* / cfg.Lifecycle.*). `cmd/strata/workers/gc.go::buildGC` reads `STRATA_GC_INTERVAL` / `STRATA_GC_GRACE` / `STRATA_GC_BATCH_SIZE` directly with literal defaults. The shared `durationFromEnv` / `intFromEnv` helpers in gc.go are reusable for sibling workers.
- Supervisor wiring lives in `internal/serverapp.Run(ctx, cfg, logger, []workers.Worker)`: builds locker via `buildLocker(cfg, metaStore)` (cassandra LWT or memory locker), spawns Supervisor in a goroutine alongside HTTP, awaits both `serverErr` and `workerErr` on shutdown. `cmd/strata-gateway/main.go` passes `nil` workers (gateway-only); `cmd/strata/server.go` passes `selected` from `workers.Resolve`.
- Each worker migration also touches `deploy/docker/Dockerfile` (replace `go build ./cmd/strata-<name>` with `go build ./cmd/strata` once + COPY the unified binary) and `deploy/docker/docker-compose.yml` (entrypoint flips to `["/usr/local/bin/strata", "server", "--workers=<name>"]`). US-015/US-016 finalise compose collapse — for now the per-worker container survives, just runs the unified binary.
- `strata server --help` "Known workers" section now renders per-worker env-var docs from `knownWorkers []knownWorker{name, envs}` in cmd/strata/server.go. When migrating a worker, populate its `envs` slice; tests in main_test.go assert at least one env var per migrated worker shows up in --help.
- Workers that depend on env-driven external config (notify needs STRATA_NOTIFY_TARGETS) return an error from Build when the config is missing. Supervisor logs ERROR + retries on backoff (1s→5s→30s→2m). Gateway and sibling workers continue serving — same behaviour as a transient panic. AWS SDK wiring (sqs client factory) lives in cmd/strata/workers/<worker>.go alongside the Build constructor.
- Test registry isolation: `cmd/strata/workers/main_test.go` snapshots the production registry in TestMain and exposes `restoreInitial()`. Registry/supervisor tests must `t.Cleanup(restoreInitial)` (NOT `Reset`) so files alphabetically after `registry_test.go` (replicator_test, supervisor_test, ...) still see registered workers when their `Lookup` runs.

  - Test files alphabetically after `registry_test.go` need the snapshot/restore TestMain pattern. Future workers (audit-export, manifest-rewriter, …) sort after registry but before supervisor — they will hit the same trap unless they use `restoreInitial`. Pattern is now in place; future tests just call it.
