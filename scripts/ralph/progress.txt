  - Strata does NOT pin region/service server-side. validateHeader derives the signing key from parsed.Region/Service (taken from the Authorization Credential scope), so a self-consistent any-region request PASSES. The adversarial "wrong region/service" is therefore a TAMPERED scope (header claims region B) vs the original signature (signed for region A) → key mismatch → ErrSignatureInvalid. Do not expect a dedicated "region mismatch" sentinel.
  - validateHeader order: parse → require X-Amz-Date (missing → ErrMalformedAuth, NOT ErrSignatureInvalid) → parse date (bad → ErrMalformedAuth) → skew check (ErrClockSkew, fires BEFORE signature so a correctly-signed-but-old request is ErrClockSkew not ErrSignatureInvalid) → Lookup cred → compute+compare sig (ErrSignatureInvalid) → session-token check.
  - bodyHash defaults to unsignedBody when X-Amz-Content-Sha256 absent. Deleting the (signed) header changes BOTH the canonical header value AND the payload hash → mismatch.
  - sentinel→S3-code mapping lives in s3api.WriteAuthDenied (internal/s3api/errors.go:180), a switch on err.Error() STRING. auth tests can't import s3api (cycle) — mirror the map locally (wantS3Code) and keep it in lockstep. "signature does not match"→SignatureDoesNotMatch, "request time outside permitted window"→RequestTimeTooSkewed, malformed/default→AccessDenied.

- **BUG FOUND + FIXED** in conditional.go: checkConditional evaluated If-Unmodified-Since unconditionally — If-Match:<match> + If-Unmodified-Since:<past> returned 412 instead of 200. RFC 7232 §6 + AWS docs: If-Match suppresses If-Unmodified-Since (match wins → serve). - Report: R2 closed, handler matrix conditional.go thin→covered (41 covered/1 thin/1 uncovered). ROADMAP: P2 Done entry added.
- **Learnings for future iterations:**
  - conditional.go GET/HEAD path is ONE helper (checkConditional, server.go:1205) shared by GET+body=false HEAD; PUT-side is separate inline (server.go:928), multipart-Complete separate (multipart.go:499), copy separate (copy_object.go:225). 4 distinct precondition sites — do NOT unify (multipart If-Match-on-missing diverges to 404, single-PUT keeps 412).
  - RFC 7232 §6 precedence is two PAIRS: (If-Match ⊳ If-Unmodified-Since) and (If-None-Match ⊳ If-Modified-Since). The AC's "If-Match over If-Modified-Since" phrasing is loose; the real suppression pairs are match/unmod and none-match/mod. Each higher-precedence header's presence must gate its partner via else.
  - ETag from PUT response header is the quoted strong form clients send back in If-Match — capture it, don't recompute MD5. putCondObject helper in conditional_test.go.
  - If-Modified/Unmodified-Since granularity is seconds; use fixed far-past ("Mon, 02 Jan 2006...") + far-future ("Fri, 01 Jan 2100...") dates for deterministic mtime comparison vs put-time now.

  - Presigned lifetime gate is ONE check: now.Before(reqTime-15m) || now.After(reqTime+expires*sec) → ErrClockSkew (BOTH expired AND future-dated map to ErrClockSkew → RequestTimeTooSkewed, NOT a dedicated "expired" sentinel). No separate signature step for the window — skew fires before sig compare, same as header path.
  - Streaming chain is order-binding: emitting correctly-signed chunks in swapped wire order fails because the reader's prevSig (seeded) no longer matches the first wire chunk's chained sig → ErrSignatureInvalid. Distinct sentinels by failure mode: bad/missing/out-of-order sig → ErrSignatureInvalid; truncated DATA section (declared size > bytes present) → io.ErrUnexpectedEOF from io.ReadFull (returned raw, NOT wrapped). Missing terminator (complete data, no 0-chunk) → ErrSignatureInvalid (readChunk EOF path).
