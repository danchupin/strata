## Codebase Patterns
- `docs/` today is a flat collection of operator-facing markdown — no Hugo, no navigation. This cycle introduces `docs/site/` as a Hugo project with a sectioned `content/` tree. `docs/*.md` files are MOVED (`git mv`) into the new tree, not duplicated, so there is one canonical source.
- Hugo theme installed as a Git submodule under `docs/site/themes/<theme>/` — `hugo-book` is the recommended default (mature, technical-docs first). Pin Hugo extended version in the GH Action so local + CI builds match.
- Sidebar ordering uses `weight` field in page frontmatter (Hugo convention). Section index pages are `_index.md` per directory.
- Cross-references between docs pages use Hugo's `{{< ref >}}` shortcode (or relative paths) — plain `[link](docs/foo.md)` does NOT work after the move.
- `make docs-serve` (new target this cycle) → `cd docs/site && hugo server -D` on `:1313` for local preview. `make docs-build` → static `hugo --minify` for CI smoke.
- GH Action `.github/workflows/docs.yml` publishes via `peaceiris/actions-gh-pages@v3` to the `gh-pages` branch. Triggers: push to main + `workflow_dispatch`. Never runs on PRs.
