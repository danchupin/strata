### US-003 â€” nginx LB
- nginx 1.27-alpine ships `nginx -t` for config syntax check. CI-friendly: `docker run --rm -v <conf>:/etc/nginx/conf.d/default.conf nginx:1.27-alpine nginx -t`.
- **Streaming-aware settings (CRITICAL)** â€” without these, multipart UploadPart on parts >100 MiB get buffered to nginx temp dir and either fill disk or time out at the default `proxy_read_timeout 60s`. Required:
  - `proxy_request_buffering off` â€” pipe upload body chunked to upstream rather than buffering.
  - `proxy_buffering off` â€” stream GET response back to client without buffering.
  - `client_max_body_size 0` â€” disable upload cap (rely on gateway's own enforcement; AWS S3 spec allows 5 GiB single PUT, 5 GiB per multipart part).
  - `proxy_read_timeout 300s` + `proxy_send_timeout 300s` â€” large uploads.
  - `proxy_http_version 1.1` â€” persistent connections + chunked encoding.
- least_conn upstream load balancing (not round-robin) â€” long uploads do not pin one replica.
- Headers preserved: `Host` ($host), `X-Forwarded-For` ($proxy_add_x_forwarded_for), `X-Forwarded-Proto` ($scheme).
