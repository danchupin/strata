- **Learnings for future iterations:**
  - **Backward-compat hybrid pattern**: When a worker Build switches from env-only to Cfg-driven, the easiest test-preserving path is `workerCfg(deps)` that returns deps.Cfg if set, else calls config.Load() in fallback. Existing `t.Setenv` tests keep working without touching the test surface — Load() honors env precedence. New production code path always sets deps.Cfg via serverapp wiring.
  - **Clamp logic belongs in config.validate**, not in each worker Build. Pre-US-002 had GC clamping in workers/gc.go (clampShards) AND rebalance clamping in workers/rebalance.go (clampInt/clampDuration). Centralising in config keeps TOML + env paths symmetric and removes ~6 redundant helper variants. Build sites just read clamped values out of cfg.
  - **`*FromEnv` helpers in shared packages should accept the raw string too**: `notify.RouterFromEnv` was the canonical example. Adding `RouterFromSpec(raw, opts...)` next to it gives cfg-driven callers a clean entry point without breaking the env-driven test suite. Mirror the pattern for any future `XXXFromEnv` (e.g. `kms.FromEnv` next cycle).
  - **`koanf` nested-section TOML key**: `cfg.Workers.GC.Interval` maps to `workers.gc.interval`. Field path matches TOML path via koanf tags on each nested struct. Don't try to coalesce into a single root-level `[gc]` section — koanf would silently bind to whichever path it sees first, and the regression test `TestLoadEmptyEnvDoesNotOverrideTOMLValue` only catches the empty-env case.
  - **`STRATA_GC_INTERVAL` shorthand → nested `workers.gc.interval`**: PRD US-002 explicitly overrides the option 2A naming convention (which would have produced `gc.interval`) by nesting under `workers.*`. Document why: every worker knob lives under `workers.*` so the operator can grep one section instead of scattered top-level keys. The convention 2A still applies to non-worker knobs (auth.*, kms.*, otel.*, etc).
  - **Hard cutover on env keys**: existing operators using `STRATA_GC_INTERVAL` are unaffected (env still wins via koanf precedence). Only TOML files that previously used `[gc] interval = "30s"` need to migrate to `[workers.gc] interval = "30s"`. Update the regression test in lockstep to avoid a silent test → flaky regression.
  - **`bucketstats` env vars remain** outside the worker section (read directly in `internal/serverapp/serverapp.go::bucketStatsTopN` + `bucketStatsInterval`). They're not workers — they're a side-loop on the gateway process. Maps cleanly to `bucket_stats.*` in US-005.
