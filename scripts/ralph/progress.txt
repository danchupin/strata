This keeps flag > env > TOML > defaults working even when the flag conceptually maps to a non-env-named koanf field. Cleaner than threading flag values through Load explicitly.
  - **`Config.Logging.Format` is forward-compat**: only `"json"` (default) and `"text"` are wired. Adding `"console"` later doesn't break the surface — `NewWithLevel` switches via strings.EqualFold. Pattern: leave the field free-form; switch in one place; document supported values in the .toml.example comment.
  - **AuditLog.Retention zero-fallback in helper**: `auditRetention(cfg)` returns `s3api.DefaultAuditRetention` when cfg.AuditLog.Retention == 0 instead of asserting non-zero post-clamp. The clamp helper skips zero values (treats them as "default"), so a TOML file that omits `audit_log.retention` lands a zero in cfg.AuditLog.Retention; the fallback restores the production default. Same shape as the worker-fallback `orDuration` helpers in US-002.
  - **`STRATA_AUDIT_RETENTION` `<N>d` env parsing was historically duplicated** in 3 places: `s3api.ParseAuditRetention` (audit handler), `cmd/strata/workers/rebalance.parseAuditRetention` (drain.complete audit row), `internal/serverapp.auditRetention` (gateway TTL). US-004 consolidates the env-side parse into `config.ParseAuditRetention` (used by the env callback inside Load); the s3api version stays for backward-compat callers but is no longer in the gateway hot path. The workers/rebalance copy was deleted.

- **Learnings for future iterations:**
  - **Provider package pattern for cfg-driven boot**: when a leaf package owns its own env-driven `FromEnv`, the cleanest cfg-driven sibling is `FromConfig(Self.Config{...})` where Self.Config mirrors the env field shape one-to-one. The provider package does NOT import `internal/config`; serverapp owns a tiny `xxxConfigFromAppConfig(cfg)` projection helper. Established for `kms.FromConfig` (US-003); now `master.FromConfig` (US-005). Future provider extractions: same shape.
  - **cephimpl tuning knobs flow via rados.Config fields, not via cephimpl-specific config**: adding fields to `internal/data/rados/config.go::Config` keeps the cephimpl entry point's signature stable (`cephimpl.New(rados.Config)`) and lets serverapp populate them once at boot from `cfg.RADOS.*`. Zero values fall back to the existing `*FromEnv` helpers so direct callers (bench harnesses, ceph-tagged tests) keep working without code change.
  - **Env-override fallback in `Or` helpers**: `cassandra.SlowMSOr(int) int` returns the cfg int when >0, else `SlowMSFromEnv()`. Same shape as `bucketStatsTopN(cfg)` and `heartbeat.NodeIDOr(string)` — the cfg layer is authoritative when set, otherwise consult the legacy env path. Keeps tests that `t.Setenv("STRATA_X", ...)` working without rewriting the test surface AND lets TOML-only boot stay self-contained.
  - **Convention 2A overrides for legacy multi-word env names**: `STRATA_BUCKETSTATS_*` env names predate the convention (no underscore between "bucket" and "stats"). TOML section name is the cleaner `bucket_stats.*` (per US-001 learning on multi-word sections). The envMap is the source of truth — declare the override explicitly; the audit-script heuristic gets it wrong and the gap-list flags it (which is correct).
  - **Manifest.Format enum validation** belongs in `config.validateMisc()`, NOT in `data.SetManifestFormat` (which already errors on bad input). The double check is fine — config catches misconfiguration at boot before any data path runs, with a clear error message keyed by the TOML key path (`manifest.format`), and data.SetManifestFormat keeps its self-contained guard for direct callers.
  - **VHost.Pattern empty vs explicit-empty distinction is lost** by koanf — both unset env + unset TOML and `STRATA_VHOST_PATTERN=""` (explicit empty) land as `cfg.VHost.Pattern == ""`. Default is now applied via the `defaults()` seed (`Pattern: "*.s3.local"`) so unset → default; the legacy `LookupEnv ok=false` distinction is gone. The "-" sentinel still disables vhost extraction. Hard cutover OK per [Pre-launch no deploys] memory — explicit-empty was a no-op anyway (`s3api.ParseVHostPatterns("")` returns nil).

Delete the helper; pass `cfg.BucketStats.Interval` to the Sampler directly. Same pattern applies whenever a helper's only job was env-string-to-typed-value conversion.

Don't delete the entry point — make it a thin wrapper around the new cfg-aware sibling. Same pattern as `logging.Setup` (env-driven) coexisting with `logging.SetupWithLevel(level, format)` (cfg-driven) from US-004.
