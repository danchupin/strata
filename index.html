        Strata Documentation

        Strata#

        S3-compatible object gateway, written in Go. Metadata in Cassandra, ScyllaDB, or TiKV. Data as 4 MiB chunks in RADOS or any S3 bucket. Drop-in replacement for Ceph RGW — without the bucket-index ceiling.

        ⚠️ Alpha software. Pre-launch; no production deploys yet. APIs and schemas may change without notice.

        Where to start#

        Get Started — Install Strata, verify it is healthy, and put your first object in under five minutes.

        Concepts — What Strata is, the S3 surface it speaks, how multi-cluster routing and drain work, what the workers do.

        Deploy — Production deployment guides — single-node, Docker Compose, multi-replica, Kubernetes.

        Operate — Day-2 workflows — drain a cluster, monitor, scale, back up, plan capacity.

        Best Practices — Tuning guides for placement, tracing, GC, lifecycle, quotas, billing, compliance, multi-cluster.

        Reference — Env vars, Admin API surface, S3 API surface, interactive OpenAPI viewer.

        S3 Compatibility — Supported / unsupported S3 surface — read this first if you are evaluating Strata as a Ceph RGW replacement.

        Architecture — Layer-by-layer architecture — auth, router, meta-store, data-backend, workers, plus deep dives for the critical flows.
