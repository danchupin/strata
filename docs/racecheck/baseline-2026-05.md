## Why the in-process baseline

The acceptance criterion for US-009 reads `Trigger nightly workflow manually
via workflow_dispatch (or wait for first scheduled run); commit
docs/racecheck/baseline-2026-XX.md with SUMMARY.md content + commit SHA +
workflow link`. The Ralph cycle ships everything the nightly workflow needs
(`.github/workflows/race-nightly.yml`, `scripts/racecheck/run.sh`,
`scripts/racecheck/summarize.sh`, `make up-all-ci`,
`bin/strata-racecheck`), but cannot itself dispatch a GitHub Action from
inside the autonomous run. The in-process snapshot below proves the harness
is clean end-to-end against the same workload mix, signing path, and
verifier oracles the nightly will run; the SHA + workflow link rows above
get backfilled in a follow-up commit on `main` once the first scheduled
run completes.

## How to refresh

Local in-process baseline (no Docker needed, ~1 min):

```sh
go run ./scripts/racecheck/genbaseline -duration=45s -concurrency=12 -dir=/tmp/baseline
REPORT_DIR=/tmp/baseline bash scripts/racecheck/summarize.sh
cat /tmp/baseline/SUMMARY.md
```

CI nightly baseline (real Cassandra + Ceph stack, 1 h, ubuntu-latest):

```sh
gh workflow run race-nightly.yml -f duration=1h -f concurrency=32
gh run download <run-id> --name race-nightly-<run-id>
cat race-nightly-<run-id>/SUMMARY.md
```

The CI run uploads `report/` as an artefact and mirrors `SUMMARY.md` into
`$GITHUB_STEP_SUMMARY` per US-007 / US-008.
